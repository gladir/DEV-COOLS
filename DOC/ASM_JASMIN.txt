==============================================================================
  1. Introduction
==============================================================================

  Jasmin est un langage assembleur pour la machine virtuelle Java (JVM).
  Il fournit une representation textuelle lisible du bytecode Java, ce
  qui permet d ecrire, d etudier et de comprendre les instructions de
  bas niveau executees par la JVM. Le nom "Jasmin" vient de "Java
  Assembler Interface".

  Contrairement aux assembleurs traditionnels qui ciblent un processeur
  physique (8086, 6502, Z80...), Jasmin cible une machine virtuelle a
  pile (stack-based). La JVM ne possede pas de registres generaux comme
  un processeur reel : toutes les operations se font par l intermediaire
  d une pile d operandes et de variables locales numerotees.

  L assembleur Jasmin a ete cree par Jonathan Meyer en 1996 et est
  devenu un outil de reference pour l enseignement du bytecode Java,
  la generation de code par des compilateurs, et l experimentation
  avec la JVM.

  Le projet DEV-COOLS contient le compilateur CCJASMIN.PAS (6334 lignes)
  qui traduit du code C en assembleur Jasmin pour la JVM. Ce document
  decrit le fonctionnement du langage assembleur Jasmin, ses directives,
  ses opcodes et son utilisation.

==============================================================================
  2. La machine virtuelle Java (JVM)
==============================================================================

  2.1 Architecture generale
  -------------------------

  La JVM est une machine virtuelle a pile (stack machine). Cela signifie
  que les operandes sont empiles sur une pile avant d etre consommes par
  les instructions. Par exemple, pour additionner deux entiers :

    bipush 3          ; empiler la valeur 3
    bipush 5          ; empiler la valeur 5
    iadd              ; depiler 3 et 5, empiler le resultat 8

  Ce modele est fondamentalement different des processeurs a registres
  comme le 8086 ou le 80386 ou les operandes sont dans des registres
  nommes (AX, BX, CX...).

  2.2 Zones memoire de la JVM
  ---------------------------

  La JVM organise la memoire en plusieurs zones :

    - Pile d operandes (Operand Stack) : pile LIFO utilisee par les
      instructions pour manipuler les valeurs. Chaque methode possede
      sa propre pile d operandes.

    - Variables locales (Local Variables) : tableau indexe de 0 a N
      contenant les parametres de la methode et les variables locales.
      Dans une methode d instance, la variable 0 contient la reference
      "this". Dans une methode statique comme main, la variable 0
      contient le premier parametre.

    - Tas (Heap) : memoire partagee pour les objets et tableaux alloues
      dynamiquement avec "new". Le ramasse-miettes (garbage collector)
      gere automatiquement la liberation de cette memoire.

    - Zone de methodes (Method Area) : contient le code des methodes,
      les constantes, et les metadonnees des classes chargees.

    - Bassin de constantes (Constant Pool) : table de constantes pour
      chaque classe, contenant les chaines de caracteres, les noms de
      classes, les signatures de methodes, et les valeurs numeriques.

  2.3 Types de donnees de la JVM
  ------------------------------

  La JVM reconnait les types primitifs suivants :

    Type JVM     Taille    Description
    ---------    ------    ----------------------------------
    byte         1 octet   Entier signe 8 bits (-128 a 127)
    short        2 octets  Entier signe 16 bits
    int          4 octets  Entier signe 32 bits
    long         8 octets  Entier signe 64 bits
    float        4 octets  Virgule flottante IEEE 754 simple
    double       8 octets  Virgule flottante IEEE 754 double
    char         2 octets  Caractere Unicode 16 bits
    boolean      1 octet   Valeur booleenne (0 ou 1)

  Et les types references :

    - References d objets (instances de classes)
    - References de tableaux (arrays)
    - null (reference nulle)

  Important : sur la pile d operandes, les types byte, short, char et
  boolean sont tous traites comme des int (32 bits). Les types long et
  double occupent deux emplacements sur la pile.

==============================================================================
  3. Structure d un fichier Jasmin
==============================================================================

  Un fichier Jasmin porte typiquement l extension .j et contient la
  definition d une seule classe Java. La structure minimale est :

    .class public NomDeLaClasse
    .super java/lang/Object

    ; Constructeur par defaut
    .method public <init>()V
        aload_0
        invokespecial java/lang/Object/<init>()V
        return
    .end method

    ; Methode principale
    .method public static main([Ljava/lang/String;)V
        .limit stack 10
        .limit locals 10
        ; instructions ici
        return
    .end method

  Chaque fichier doit contenir :

    1. La declaration de la classe (.class)
    2. La declaration de la classe parente (.super)
    3. Un ou plusieurs blocs de methodes (.method / .end method)

  Les commentaires commencent par un point-virgule (;) et s etendent
  jusqu a la fin de la ligne.

  Les etiquettes (labels) sont des identifiants suivis de deux-points :

    Label1:
        bipush 42
        goto Label1

==============================================================================
  4. Les directives Jasmin
==============================================================================

  4.1 Directives de classe
  ------------------------

  .class <acces> <nom>
      Declare le nom de la classe et ses modificateurs d acces.
      Exemples :
        .class public Main
        .class public final MaClasse
        .class abstract MaClasseAbstraite

      Modificateurs d acces :
        public       - accessible depuis n importe ou
        final        - ne peut pas etre etendue (heritee)
        abstract     - ne peut pas etre instanciee directement
        interface    - declare une interface

  .super <nom>
      Specifie la classe parente (superclasse).
      Toute classe doit avoir une superclasse (sauf java/lang/Object).
      Exemple :
        .super java/lang/Object

  .implements <interface>
      Declare qu une classe implemente une interface.
      Exemple :
        .implements java/io/Serializable

  .source <fichier>
      Indique le fichier source d origine (optionnel, pour le debogage).
      Exemple :
        .source Main.java

  4.2 Directives de champs (variables d instance / de classe)
  -----------------------------------------------------------

  .field <acces> <nom> <descripteur> [= <valeur>]
      Declare un champ (variable membre) de la classe.
      Exemples :
        .field public compteur I
        .field private static PI D = 3.14159
        .field protected nom Ljava/lang/String;

      Modificateurs d acces pour les champs :
        public       - accessible depuis n importe ou
        private      - accessible uniquement dans la classe
        protected    - accessible dans la classe et ses sous-classes
        static       - champ de classe (partage entre toutes les instances)
        final        - valeur constante, ne peut pas etre modifiee
        volatile     - acces atomique en environnement multi-thread

  4.3 Directives de methodes
  --------------------------

  .method <acces> <nom><descripteur>
      Debut de la definition d une methode.
      Exemples :
        .method public static main([Ljava/lang/String;)V
        .method public getValeur()I
        .method private calculer(II)I

      Modificateurs d acces pour les methodes :
        public       - accessible depuis n importe ou
        private      - accessible uniquement dans la classe
        protected    - accessible dans la classe et ses sous-classes
        static       - methode de classe (pas besoin d instance)
        final        - ne peut pas etre redefinie
        abstract     - pas de corps, doit etre implementee
        synchronized - acces synchronise en multi-thread
        native       - implementee en code natif (JNI)

  .end method
      Fin de la definition d une methode.

  .limit stack <N>
      Declare la profondeur maximale de la pile d operandes pour cette
      methode. La JVM verifie que cette limite n est jamais depassee.
      Exemple :
        .limit stack 100

  .limit locals <N>
      Declare le nombre maximal de variables locales utilisees par la
      methode. Inclut les parametres et la reference "this" pour les
      methodes d instance.
      Exemple :
        .limit locals 100

  .line <N>
      Associe un numero de ligne du source aux instructions suivantes.
      Utilise par le debogueur pour localiser l instruction courante.
      Exemple :
        .line 42

  .throws <classe>
      Declare qu une methode peut lancer une exception.
      Exemple :
        .throws java/io/IOException

  .catch <classe> from <debut> to <fin> using <gestionnaire>
      Declare un bloc try-catch pour la gestion des exceptions.
      Exemple :
        .catch java/lang/Exception from Label1 to Label2 using Handler

==============================================================================
  5. Les descripteurs de types JVM
==============================================================================

  Les descripteurs de types sont des codes compacts utilises dans les
  signatures de methodes et les declarations de champs. Chaque type
  Java possede un descripteur :

  5.1 Types primitifs
  -------------------

    Descripteur   Type Java     Description
    -----------   ---------     --------------------------
    B             byte          Entier signe 8 bits
    C             char          Caractere Unicode 16 bits
    D             double        Flottant 64 bits
    F             float         Flottant 32 bits
    I             int           Entier signe 32 bits
    J             long          Entier signe 64 bits
    S             short         Entier signe 16 bits
    Z             boolean       Booleen (true/false)
    V             void          Aucune valeur (retour)

  5.2 Types references
  --------------------

    Les classes sont representees par L<nom>;

      Ljava/lang/String;        -> java.lang.String
      Ljava/lang/Object;        -> java.lang.Object
      Ljava/io/PrintStream;     -> java.io.PrintStream
      Ljava/util/ArrayList;     -> java.util.ArrayList

    Les separateurs de paquets utilisent / au lieu de .

  5.3 Types tableaux
  ------------------

    Les tableaux sont prefixes par [ pour chaque dimension :

      [I                        -> int[]
      [D                        -> double[]
      [Ljava/lang/String;       -> String[]
      [[I                       -> int[][]
      [[[B                      -> byte[][][]

  5.4 Signatures de methodes
  --------------------------

    Les signatures de methodes sont de la forme :
      (types_parametres)type_retour

    Exemples :
      ()V                       -> void methode()
      (I)V                      -> void methode(int)
      (II)I                     -> int methode(int, int)
      (Ljava/lang/String;)V     -> void methode(String)
      ([Ljava/lang/String;)V    -> void methode(String[])
      (ILjava/lang/String;)Z    -> boolean methode(int, String)
      (DDF)D                    -> double methode(double, double, float)
      ()Ljava/lang/String;      -> String methode()

  7.1 Arithmetique entiere (int)
  ------------------------------

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    iadd            60     Addition : a + b
    isub            64     Soustraction : a - b
    imul            68     Multiplication : a * b

  8. Les opcodes de conversion de types
==============================================================================

  Ces instructions convertissent une valeur du sommet de la pile d un
  type a un autre.

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    i2l             85     int vers long
    i2f             86     int vers float
    i2d             87     int vers double
    l2i             88     long vers int
    l2f             89     long vers float
    l2d             8A     long vers double
    f2i             8B     float vers int
    f2l             8C     float vers long
    f2d             8D     float vers double
    d2i             8E     double vers int
    d2l             8F     double vers long
    d2f             90     double vers float
    i2b             91     int vers byte (troncature signee)
    i2c             92     int vers char (troncature non signee)
    i2s             93     int vers short (troncature signee)

  Note : les conversions d elargissement (widening) comme int->long
  sont toujours sans perte. Les conversions de retrecissement
  (narrowing) comme long->int peuvent perdre de l information.

==============================================================================
  9. Les opcodes de controle de flux
==============================================================================

  9.1 Branchements inconditionnels
  --------------------------------

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    goto <label>    A7     Branchement inconditionnel vers <label>
    goto_w <label>  C8     Branchement inconditionnel (offset large)
    jsr <label>     A8     Saut vers sous-routine, empile adresse retour
    jsr_w <label>   C9     Saut vers sous-routine (offset large)
    ret <var>       A9     Retour de sous-routine (adresse dans <var>)

  9.2 Branchements conditionnels sur un int
  ------------------------------------------

    Ces instructions depilent un int et branchent selon sa valeur :

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    ifeq <label>    99     Brancher si valeur == 0
    ifne <label>    9A     Brancher si valeur != 0
    iflt <label>    9B     Brancher si valeur < 0
    ifge <label>    9C     Brancher si valeur >= 0
    ifgt <label>    9D     Brancher si valeur > 0
    ifle <label>    9E     Brancher si valeur <= 0

  9.3 Branchements conditionnels sur deux int
  --------------------------------------------

    Ces instructions depilent deux int et comparent :

    Opcode            Hex    Description
    ----------------- ----   ------------------------------------------
    if_icmpeq <label> 9F     Brancher si a == b
    if_icmpne <label> A0     Brancher si a != b
    if_icmplt <label> A1     Brancher si a < b
    if_icmpge <label> A2     Brancher si a >= b
    if_icmpgt <label> A3     Brancher si a > b
    if_icmple <label> A4     Brancher si a <= b

  9.4 Branchements conditionnels sur les references
  --------------------------------------------------

    Opcode            Hex    Description
    ----------------- ----   ------------------------------------------
    ifnull <label>    C6     Brancher si reference == null
    ifnonnull <label> C7     Brancher si reference != null
    if_acmpeq <label> A5     Brancher si ref_a == ref_b
    if_acmpne <label> A6     Brancher si ref_a != ref_b

  9.5 Instructions de choix multiple (switch)
  --------------------------------------------

    Opcode                  Hex    Description
    ----------------------- ----   -----------------------------------
    tableswitch             AA     Switch avec index contigus
    lookupswitch            AB     Switch avec valeurs dispersees

    Exemple de tableswitch :
      tableswitch 0 2        ; valeurs de 0 a 2
          Label0             ; case 0
          Label1             ; case 1
          Label2             ; case 2
          Default            ; default

    Exemple de lookupswitch :
      lookupswitch
          1 : Label1         ; case 1
          5 : Label5         ; case 5
          100 : Label100     ; case 100
          default : Default  ; default

  9.6 Retour de methode
  ---------------------

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    ireturn         AC     Retourner un int
    lreturn         AD     Retourner un long
    freturn         AE     Retourner un float
    dreturn         AF     Retourner un double
    areturn         B0     Retourner une reference
    return          B1     Retourner void (aucune valeur)

==============================================================================
  10. Les opcodes d invocation de methodes
==============================================================================

  10.1 Types d invocation
  -----------------------

    Opcode                     Hex    Description
    -------------------------- ----   --------------------------------
    invokevirtual <methode>    B6     Appeler methode d instance
    invokespecial <methode>    B7     Appeler constructeur ou super
    invokestatic <methode>     B8     Appeler methode statique
    invokeinterface <meth> <n> B9     Appeler methode d interface
    invokedynamic <meth>       BA     Appel dynamique (Java 7+)

  10.2 Format des references de methodes
  ---------------------------------------

    Les methodes sont referencees par leur chemin complet :
      classe/methode(descripteur)type_retour

    Exemples :

      ; Appeler System.out.println(String)
      invokevirtual java/io/PrintStream/println(Ljava/lang/String;)V

      ; Appeler le constructeur de Object
      invokespecial java/lang/Object/<init>()V

      ; Appeler Integer.parseInt(String)
      invokestatic java/lang/Integer/parseInt(Ljava/lang/String;)I

      ; Appeler Math.max(int, int)
      invokestatic java/lang/Math/max(II)I

  10.3 Acces aux champs
  ---------------------

    Opcode                  Hex    Description
    ----------------------- ----   -----------------------------------
    getstatic <champ>       B2     Lire champ statique
    putstatic <champ>       B3     Ecrire champ statique
    getfield <champ>        B4     Lire champ d instance
    putfield <champ>        B5     Ecrire champ d instance

    Exemples :

      ; Charger System.out (un PrintStream)
      getstatic java/lang/System/out Ljava/io/PrintStream;

      ; Lire un champ d instance
      getfield MaClasse/compteur I

==============================================================================
  11. Les opcodes de manipulation de la pile
==============================================================================

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    pop             57     Retirer le sommet de la pile (1 mot)
    pop2            58     Retirer 2 mots du sommet
    dup             59     Dupliquer le sommet de la pile
    dup_x1          5A     Dupliquer et inserer sous le 2e element
    dup_x2          5B     Dupliquer et inserer sous le 3e element
    dup2            5C     Dupliquer les 2 mots du sommet
    dup2_x1         5D     Dupliquer 2 mots et inserer sous le 3e
    dup2_x2         5E     Dupliquer 2 mots et inserer sous le 5e
    swap            5F     Echanger les 2 elements du sommet
    nop             00     Ne rien faire (aucune operation)

  Exemple d utilisation de dup pour affecter et utiliser une valeur :

    bipush 42             ; empiler 42
    dup                   ; dupliquer : pile = 42, 42
    istore 1              ; stocker dans var 1 : pile = 42
    ; la valeur 42 est encore sur la pile pour une utilisation ulterieure

==============================================================================
  12. Les opcodes de creation d objets et tableaux
==============================================================================

  12.1 Creation d objets
  ----------------------

    Opcode                  Hex    Description
    ----------------------- ----   -----------------------------------
    new <classe>            BB     Creer un nouvel objet (non initialise)
    instanceof <classe>     C1     Verifier le type d un objet
    checkcast <classe>      C0     Conversion de type avec verification

    Exemple de creation d objet :
      new java/lang/StringBuilder       ; creer objet
      dup                               ; dupliquer la reference
      invokespecial java/lang/StringBuilder/<init>()V  ; constructeur

  12.2 Creation de tableaux
  -------------------------

    Opcode                  Hex    Description
    ----------------------- ----   -----------------------------------
    newarray <type>         BC     Creer tableau de type primitif
    anewarray <classe>      BD     Creer tableau de references
    multianewarray <t> <d>  C5     Creer tableau multidimensionnel
    arraylength             BE     Obtenir la longueur d un tableau

    Codes de type pour newarray :
      4  = boolean
      5  = char
      6  = float
      7  = double
      8  = byte
      9  = short
      10 = int
      11 = long

    Exemple :
      bipush 10                ; taille = 10
      newarray int             ; creer int[10]
      astore 1                 ; stocker la reference dans var 1

==============================================================================
  13. Les opcodes de comparaison
==============================================================================

  Ces instructions comparent des valeurs long, float ou double et
  empilent un int (-1, 0, ou 1) representant le resultat :

    Opcode          Hex    Description
    --------------- ----   ------------------------------------------
    lcmp            94     Comparer deux long
    fcmpl           95     Comparer deux float (NaN -> -1)
    fcmpg           96     Comparer deux float (NaN -> 1)
    dcmpl           97     Comparer deux double (NaN -> -1)
    dcmpg           98     Comparer deux double (NaN -> 1)

  Resultat empile :
    -1 si a < b
     0 si a == b
     1 si a > b

  Exemple : comparer deux long et brancher si egal :
    lload 0               ; charger premier long
    lload 2               ; charger second long
    lcmp                  ; comparer -> -1, 0, ou 1
    ifeq SontEgaux        ; brancher si resultat == 0

==============================================================================
  14. Exemples complets
==============================================================================

  14.1 Hello World
  ----------------

    ; HelloWorld.j - Affiche "Hello, World!" sur la console

    .class public HelloWorld
    .super java/lang/Object

    .method public <init>()V
        aload_0
        invokespecial java/lang/Object/<init>()V
        return
    .end method

    .method public static main([Ljava/lang/String;)V
        .limit stack 2
        .limit locals 1

        getstatic java/lang/System/out Ljava/io/PrintStream;
        ldc "Hello, World!"
        invokevirtual java/io/PrintStream/println(Ljava/lang/String;)V

        return
    .end method

  14.2 Calcul arithmetique
  ------------------------

    ; Calcule (3 + 5) * 2 et affiche le resultat

    .class public Calcul
    .super java/lang/Object

    .method public <init>()V
        aload_0
        invokespecial java/lang/Object/<init>()V
        return
    .end method

    .method public static main([Ljava/lang/String;)V
        .limit stack 3
        .limit locals 2

        bipush 3              ; empiler 3
        bipush 5              ; empiler 5
        iadd                  ; 3 + 5 = 8
        iconst_2              ; empiler 2
        imul                  ; 8 * 2 = 16
        istore 1              ; stocker dans variable locale 1

        ; Afficher le resultat
        getstatic java/lang/System/out Ljava/io/PrintStream;
        iload 1
        invokevirtual java/io/PrintStream/println(I)V

        return
    .end method

  14.3 Boucle avec compteur
  -------------------------

    ; Affiche les nombres de 1 a 10

    .class public Boucle
    .super java/lang/Object

    .method public <init>()V
        aload_0
        invokespecial java/lang/Object/<init>()V
        return
    .end method

    .method public static main([Ljava/lang/String;)V
        .limit stack 3
        .limit locals 2

        iconst_1              ; i = 1
        istore 1

    Debut:
        iload 1               ; charger i
        bipush 11             ; charger 11
        if_icmpge Fin         ; si i >= 11, sortir

        ; Afficher i
        getstatic java/lang/System/out Ljava/io/PrintStream;
        iload 1
        invokevirtual java/io/PrintStream/println(I)V

        iinc 1 1              ; i++
        goto Debut

    Fin:
        return
    .end method

  14.4 Condition if/else
  ----------------------

    ; Verifie si un nombre est positif, negatif ou zero

    .class public Condition
    .super java/lang/Object

    .method public <init>()V
        aload_0
        invokespecial java/lang/Object/<init>()V
        return
    .end method

    .method public static main([Ljava/lang/String;)V
        .limit stack 3
        .limit locals 2

        bipush 42             ; valeur a tester
        istore 1

        iload 1
        ifgt Positif          ; si > 0, aller a Positif
        iload 1
        iflt Negatif          ; si < 0, aller a Negatif

        ; Cas zero
        getstatic java/lang/System/out Ljava/io/PrintStream;
        ldc "Zero"
        invokevirtual java/io/PrintStream/println(Ljava/lang/String;)V
        goto Suite

    Positif:
        getstatic java/lang/System/out Ljava/io/PrintStream;
        ldc "Positif"
        invokevirtual java/io/PrintStream/println(Ljava/lang/String;)V
        goto Suite

    Negatif:
        getstatic java/lang/System/out Ljava/io/PrintStream;
        ldc "Negatif"
        invokevirtual java/io/PrintStream/println(Ljava/lang/String;)V

    Suite:
        return
    .end method

  14.5 Appel de methode statique
  ------------------------------

    ; Definit une methode factorielle et l appelle

    .class public Factorielle
    .super java/lang/Object

    .method public <init>()V
        aload_0
        invokespecial java/lang/Object/<init>()V
        return
    .end method

    .method public static fact(I)I
        .limit stack 4
        .limit locals 2

        iload 0               ; charger n
        iconst_1
        if_icmple CasBase     ; si n <= 1, retourner 1

        iload 0               ; empiler n
        iload 0               ; empiler n
        iconst_1
        isub                  ; n - 1
        invokestatic Factorielle/fact(I)I  ; fact(n-1)
        imul                  ; n * fact(n-1)
        ireturn

    CasBase:
        iconst_1
        ireturn
    .end method

    .method public static main([Ljava/lang/String;)V
        .limit stack 3
        .limit locals 1

        getstatic java/lang/System/out Ljava/io/PrintStream;
        bipush 10
        invokestatic Factorielle/fact(I)I
        invokevirtual java/io/PrintStream/println(I)V

        return
    .end method

==============================================================================
  15. Utilisation avec CCJASMIN.PAS
==============================================================================

  15.1 Presentation du compilateur
  --------------------------------

  CCJASMIN.PAS est un compilateur C vers Jasmin JVM ecrit en Pascal
  (Free Pascal / Turbo Pascal). Il fait partie du projet DEV-COOLS
  et compte environ 6334 lignes de code source.

  Il traduit un sous-ensemble du langage C en instructions Jasmin
  executables sur la machine virtuelle Java.

  15.2 Fonctionnalites supportees
  --------------------------------

  Types C supportes et leur correspondance JVM :

    Type C          Type JVM       Descripteur
    -----------     ----------     -----------
    char            byte/int       B / I
    short           short/int      S / I
    int             int            I
    long            long           J
    float           float          F
    double          double         D
    void            void           V

  Constructions du langage C supportees :

    - Declarations de variables (locales et globales)
    - Affectations simples et composees (=, +=, -=, *=, /=, %=)
    - Operations arithmetiques (+, -, *, /, %)
    - Operations logiques (&&, ||, !)
    - Operations bit a bit (&, |, ^, ~, <<, >>)
    - Structures conditionnelles (if, else, switch/case)
    - Boucles (while, for, do-while)
    - Fonctions (declaration, definition, appels)
    - Tableaux a une dimension
    - Instructions break, continue, goto, return
    - Preprocesseur (#include, #define, #if, #ifdef)

  15.3 Generation du code
  -----------------------

  Le compilateur CCJASMIN.PAS genere un fichier .j contenant :

    1. L en-tete de classe (.class public Main / .super java/lang/Object)
    2. Le constructeur par defaut (<init>)
    3. La methode main avec .limit stack 100 et .limit locals 100
    4. Les instructions Jasmin correspondant au code C

  Les variables locales C sont allouees dans les variables locales JVM
  (numerotees a partir de 1 dans main, la variable 0 etant args).

  Les expressions sont evaluees sur la pile d operandes en utilisant
  les instructions de chargement (iload, bipush, ldc) et les
  instructions arithmetiques (iadd, isub, imul, idiv, irem).

  Les structures de controle utilisent des etiquettes numerotees
  (L0, L1, L2...) et les instructions de branchement conditionnel
  (ifeq, ifne, iflt, ifgt, if_icmpeq, goto...).

  15.4 Niveaux d optimisation
  ---------------------------

  Le compilateur supporte plusieurs niveaux d optimisation :

    Niveau 0 : Aucune optimisation
    Niveau 1 : Optimisations de base
    Niveau 2 : Optimisations avancees
    Niveau 3 : Optimisations agressives

  Les techniques d optimisation incluent :

    - Analyse peephole (remplacement de motifs d instructions)
    - Elimination du code mort
    - Repliement de constantes (constant folding)
    - Reduction de force (strength reduction)
    - Allocation de registres (mapping vers variables locales)
    - Optimisation des sauts (elimination de sauts redondants)

==============================================================================
  16. Assemblage et execution
==============================================================================

  16.1 Chaine de compilation
  --------------------------

  Pour compiler et executer un programme Jasmin :

    1. Ecrire le fichier source (MonProgramme.j)
    2. Assembler avec Jasmin :
         java -jar jasmin.jar MonProgramme.j
       Cela produit MonProgramme.class
    3. Executer avec la JVM :
         java MonProgramme

  16.2 Avec CCJASMIN.PAS
  -----------------------

  Pour utiliser le compilateur CCJASMIN.PAS :

    1. Compiler CCJASMIN.PAS avec Free Pascal :
         fpc -Mtp CCJASMIN.PAS -oCCJASMIN
    2. Compiler un fichier C en Jasmin :
         ./CCJASMIN source.c
       Cela produit un fichier .j
    3. Assembler avec Jasmin :
         java -jar jasmin.jar source.j
    4. Executer :
         java Main

  16.3 Verification du bytecode
  -----------------------------

  La JVM effectue une verification du bytecode lors du chargement
  d une classe. Cette verification inclut :

    - Verification de la structure du fichier .class
    - Verification des types sur la pile d operandes
    - Verification que la pile ne deborde pas (stack overflow)
    - Verification que les variables locales sont initialisees
    - Verification que les branchements menent a des instructions valides
    - Verification des acces aux champs et methodes

  Si la verification echoue, la JVM lance une VerifyError et refuse
  d executer le programme. Les directives .limit stack et .limit locals
  doivent etre correctement definies pour eviter ces erreurs.

==============================================================================
  17. Differences avec le bytecode Java brut
==============================================================================

  Jasmin est une representation textuelle du bytecode Java. Les
  differences principales avec le format binaire .class sont :

    Aspect                  Jasmin (.j)            Bytecode (.class)
    ----------------------  ---------------------  ----------------------
    Format                  Texte lisible          Binaire
    Adresses de saut        Etiquettes nommees     Offsets numeriques
    References              Noms complets          Index dans pool
    Commentaires            Oui (;)                Non
    Lisibilite              Elevee                 Tres faible
    Taille                  Plus grande            Compacte
    Modification            Editeur texte          Outil specialise

  Jasmin gere automatiquement :

    - Le calcul des offsets de saut a partir des etiquettes
    - La construction du bassin de constantes (constant pool)
    - L encodage des instructions en octets
    - La generation de l en-tete du fichier .class
    - Le padding des instructions tableswitch/lookupswitch

  Cela rend Jasmin beaucoup plus pratique que la manipulation directe
  du format binaire .class pour l experimentation et l enseignement.
