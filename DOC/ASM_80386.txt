==============================================================================
     ASSEMBLEUR 80386 - Jeu d'instructions et architecture
     Intel 80386 (i386) - Documentation de reference
==============================================================================

  Document de reference sur le processeur Intel 80386 (i386), son
  architecture, son jeu d'instructions, ses modes d'adressage et les
  techniques de programmation en langage assembleur associees.

  Date:       Mars 2026
  Processeur: Intel 80386 (1985), 32 bits
  Norme:      Intel Architecture Software Developer's Manual

==============================================================================
1. INTRODUCTION
==============================================================================

  1.1 Presentation generale
  -------------------------

  Le processeur Intel 80386, commercialise en octobre 1985, est le
  premier microprocesseur x86 entierement 32 bits. Il marque une
  evolution majeure par rapport au 80286 en introduisant :

    - Un bus de donnees de 32 bits
    - Un bus d'adresses de 32 bits (espace adressable de 4 Go)
    - Des registres generaux etendus a 32 bits (EAX, EBX, ECX, EDX,
      ESI, EDI, EBP, ESP)
    - Un mode protege ameliore avec pagination
    - Un mode virtuel 8086 (V86) pour la compatibilite DOS
    - Une unite de gestion memoire (MMU) integree

  Le 80386 est retrocompatible avec le 8086, le 80186 et le 80286.
  Il peut executer le code 16 bits existant sans modification, tout
  en offrant un environnement 32 bits natif bien plus puissant.

  1.2 Variantes du 80386
  ----------------------

  Intel a produit plusieurs variantes du 80386 :

    80386DX  : Version originale, bus externe 32 bits (1985)
    80386SX  : Version economique, bus externe 16 bits (1988)
    80386SL  : Version basse consommation pour portables (1990)
    80386EX  : Version embarquee avec peripheriques integres (1994)

  Toutes ces variantes partagent le meme jeu d'instructions 32 bits.
  Les differences portent sur la largeur du bus externe et les
  fonctionnalites d'economie d'energie.

  1.3 Contexte historique
  -----------------------

  Le 80386 a ete utilise dans les premiers PC 32 bits, notamment :
    - Les premiers systemes sous Windows 3.0 en mode 386 ameliore
    - Les premieres versions de Linux (0.01, 1991)
    - Les systemes OS/2 2.0 d'IBM
    - Les jeux DOS 32 bits via DOS/4GW et DOS/32A
    - Les compilateurs Watcom C, Borland C++ 3.x, DJGPP

  L'assembleur 80386 reste pertinent car :
    - Il constitue la base de l'architecture x86 32 bits (IA-32)
    - Les processeurs modernes (Pentium, Core) restent compatibles
    - Les systemes d'exploitation 32 bits l'utilisent encore
    - Il est la cible de nombreux compilateurs du projet DEV-COOLS
      (ABAPCW32, CLIPPER86 en mode 32 bits, etc.)

  1.4 Modes de fonctionnement
  ---------------------------

  Le 80386 dispose de trois modes de fonctionnement :

    Mode reel (Real Mode) :
      - Compatibilite complete avec le 8086
      - Espace adressable de 1 Mo (20 bits d'adresse)
      - Pas de protection memoire
      - Segments de 64 Ko maximum
      - Mode par defaut au demarrage (RESET)

    Mode protege (Protected Mode) :
      - Acces a l'espace d'adressage complet de 4 Go
      - Protection memoire entre processus
      - 4 niveaux de privilege (anneaux 0 a 3)
      - Pagination optionnelle (pages de 4 Ko)
      - Segments de taille variable (jusqu'a 4 Go)
      - Multitache materiel via le TSS (Task State Segment)

    Mode virtuel 8086 (Virtual 8086 Mode) :
      - Emulation du mode reel dans un environnement protege
      - Permet d'executer des programmes DOS dans une machine
        virtuelle supervisee par le noyau en mode protege
      - Utilise pour la compatibilite DOS sous Windows 3.x et OS/2
      - Active via le drapeau VM (bit 17) du registre EFLAGS

2. ARCHITECTURE ET REGISTRES
==============================================================================

  2.1 Vue d'ensemble de l'architecture
  -------------------------------------

  Le 80386 est un processeur 32 bits a architecture CISC (Complex
  Instruction Set Computer). Son architecture interne comprend :

    - Une unite de bus (Bus Interface Unit) pour les acces memoire
      et E/S sur un bus externe de 32 bits de donnees et 32 bits
      d'adresses
    - Une unite de prefetch de code (Code Prefetch Unit) avec une
      file d'attente de 16 octets
    - Une unite de decodage d'instructions (Instruction Decode Unit)
    - Une unite d'execution (Execution Unit) avec une ALU 32 bits,
      un multiplicateur et un decaleur (barrel shifter)
    - Une unite de segmentation (Segmentation Unit) pour la
      traduction des adresses logiques en adresses lineaires
    - Une unite de pagination (Paging Unit) pour la traduction
      des adresses lineaires en adresses physiques

  Le pipeline du 80386 comporte 6 etapes, ce qui permet un
  recouvrement partiel entre les phases de traitement des
  instructions successives.

  2.2 Registres generaux 32 bits
  ------------------------------

  Le 80386 dispose de 8 registres generaux de 32 bits. Chacun peut
  etre accede en totalite (32 bits), en partie basse 16 bits, et
  pour les 4 premiers, en sous-registres de 8 bits.

  Schema de decomposition (exemple avec EAX) :

    Bits :  31 ............. 16  15 ...... 8  7 ....... 0
            |                 |  |         |  |         |
            |     (non        |  |   AH    |  |   AL    |
            |   accessible    |  |         |  |         |
            |  directement)   |  |         |  |         |
            |                 |  |--- AX --|--|---------|
            |------- EAX -----|--|---------|--|---------|

  Registre  32 bits  16 bits  8 bits haut  8 bits bas  Utilisation
  --------  -------  -------  ----------  ----------  ------------------
  EAX       EAX      AX       AH          AL          Accumulateur
  EBX       EBX      BX       BH          BL          Base (adressage)
  ECX       ECX      CX       CH          CL          Compteur (boucles)
  EDX       EDX      DX       DH          DL          Donnees (E/S, MUL)
  ESI       ESI      SI       -           -           Source (chaines)
  EDI       EDI      DI       -           -           Destination (chaines)
  EBP       EBP      BP       -           -           Pointeur de base
  ESP       ESP      SP       -           -           Pointeur de pile

  Roles conventionnels :

    EAX (Accumulator) :
      - Operande implicite pour MUL, IMUL, DIV, IDIV, CBW, CWD, CDQ
      - Port de retour des fonctions (convention C et stdcall)
      - Operande implicite pour IN/OUT (E/S)
      - Formes courtes d'instructions (1 octet de moins avec AL/AX/EAX)

    EBX (Base) :
      - Registre de base pour l'adressage indirect
      - Souvent utilise pour pointer sur des structures de donnees
      - XLAT utilise DS:[EBX+AL] implicitement

    ECX (Counter) :
      - Compteur implicite pour LOOP, REP, JCXZ/JECXZ
      - Compteur de decalage pour SHL/SHR/ROL/ROR (via CL)
      - Souvent utilise comme compteur de boucle

    EDX (Data) :
      - Extension de EAX pour MUL/IMUL (resultat 64 bits EDX:EAX)
      - Extension de EAX pour DIV/IDIV (dividende EDX:EAX)
      - Numero de port pour IN/OUT (via DX)
      - CDQ etend le signe de EAX dans EDX

    ESI (Source Index) :
      - Pointeur source implicite pour MOVS, CMPS, LODS (DS:[ESI])
      - Incremente/decremente automatiquement par les instructions
        de chaines selon le drapeau DF

    EDI (Destination Index) :
      - Pointeur destination implicite pour MOVS, CMPS, STOS,
        SCAS (ES:[EDI])
      - Incremente/decremente automatiquement par les instructions
        de chaines selon le drapeau DF

    EBP (Base Pointer) :
      - Pointeur de cadre de pile (stack frame) par convention
      - Utilise pour acceder aux parametres et variables locales
        dans les procedures : [EBP+8] = 1er parametre,
        [EBP-4] = 1ere variable locale
      - Adresse implicitement le segment SS

    ESP (Stack Pointer) :
      - Pointeur de sommet de pile, modifie par PUSH/POP/CALL/RET
      - Adresse implicitement le segment SS
      - Doit toujours pointer sur une adresse valide
      - Ne doit pas etre utilise comme registre general

  2.3 Registres de segment
  ------------------------

  Le 80386 dispose de 6 registres de segment de 16 bits. En mode
  reel, ils contiennent la base du segment (adresse >> 4). En mode
  protege, ils contiennent un selecteur qui reference un descripteur
  dans la GDT ou la LDT.

  Registre  Nom complet             Utilisation
  --------  ----------------------  -----------------------------------
  CS        Code Segment            Segment du code en cours d'execution
  DS        Data Segment            Segment de donnees par defaut
  SS        Stack Segment           Segment de la pile
  ES        Extra Segment           Segment supplementaire (destination
                                    pour les instructions de chaines)
  FS        (pas de nom officiel)   Segment supplementaire (nouveau 386)
  GS        (pas de nom officiel)   Segment supplementaire (nouveau 386)

  Notes :
    - CS ne peut etre modifie que par JMP FAR, CALL FAR, RET FAR,
      IRET ou une interruption
    - FS et GS sont de nouveaux registres introduits par le 80386
    - En mode protege, chaque registre de segment possede une partie
      cachee (base, limite, droits d'acces) chargee depuis le
      descripteur lors de l'affectation du selecteur
    - En mode FLAT (Windows 32 bits), CS/DS/ES/SS pointent
      generalement tous sur le meme espace d'adressage de 4 Go

  2.4 Pointeur d'instruction (EIP)
  ---------------------------------

  Le registre EIP (Extended Instruction Pointer) contient l'adresse
  (offset) de la prochaine instruction a executer dans le segment
  de code courant (CS).

    Taille : 32 bits (16 bits en mode reel, via IP)
    Acces  : Non accessible directement par les instructions MOV
    Modification : Uniquement via JMP, CALL, RET, IRET, INT,
                   interruptions et exceptions

  En mode reel, seuls les 16 bits bas (IP) sont significatifs,
  limitant l'offset dans le segment de code a 64 Ko.
  En mode protege 32 bits, EIP permet d'adresser jusqu'a 4 Go
  dans le segment de code.

  2.5 Registre de drapeaux (EFLAGS)
  ----------------------------------

  Le registre EFLAGS est un registre de 32 bits contenant les
  drapeaux d'etat, de controle et systeme du processeur. Les bits
  16 a 31 sont nouveaux par rapport au registre FLAGS du 8086/80286.

  Bit  Sigle  Nom                     Description
  ---  -----  ----------------------  --------------------------------
   0   CF     Carry Flag              Retenue/emprunt non signe
   1   (1)    (reserve, toujours 1)
   2   PF     Parity Flag             Parite de l'octet bas du resultat
   3   (0)    (reserve)
   4   AF     Auxiliary Carry Flag     Retenue BCD (bit 3 vers bit 4)
   5   (0)    (reserve)
   6   ZF     Zero Flag               Resultat egal a zero
   7   SF     Sign Flag               Bit de signe du resultat
   8   TF     Trap Flag               Mode pas-a-pas (debug)
   9   IF     Interrupt Enable Flag   Autorise les interruptions
                                      materielles (masquables)
  10   DF     Direction Flag           Sens des operations de chaines
                                      (0 = croissant, 1 = decroissant)
  11   OF     Overflow Flag           Debordement signe
  12-13 IOPL  I/O Privilege Level     Niveau de privilege pour E/S
                                      (0 a 3, mode protege uniquement)
  14   NT     Nested Task              Tache imbriquee (mode protege)
  15   (0)    (reserve)
  16   RF     Resume Flag              Reprendre apres un point d'arret
                                      debug (nouveau 80386)
  17   VM     Virtual 8086 Mode        Active le mode V86 (nouveau 386)
  18-31 (0)   (reserves sur le 80386, utilises sur les processeurs
              ulterieurs : AC, VIF, VIP, ID)

  Drapeaux d'etat (modifies par les instructions arithmetiques
  et logiques) :
    CF : Retenue pour les operations non signees
    PF : 1 si le nombre de bits a 1 dans l'octet bas est pair
    AF : Retenue auxiliaire pour l'arithmetique BCD
    ZF : 1 si le resultat est zero
    SF : Copie du bit de poids fort du resultat
    OF : 1 si debordement signe (le resultat ne tient pas dans
         la taille de la destination en complement a deux)

  Drapeaux de controle :
    DF : Controle le sens de ESI/EDI pour MOVS, CMPS, STOS, LODS,
         SCAS. Modifie par CLD (DF=0) et STD (DF=1)
    IF : Active/desactive les interruptions masquables. Modifie
         par CLI (IF=0) et STI (IF=1)
    TF : Provoque une exception INT 1 apres chaque instruction
         (mode de trace pour les debogueurs)

  Drapeaux systeme (mode protege) :
    IOPL : Determine le niveau de privilege minimum pour executer
           les instructions d'E/S (IN, OUT, INS, OUTS, CLI, STI)
    NT   : Indique que la tache courante est imbriquee (chainee
           via le TSS)
    RF   : Empeche les exceptions de debogage (#DB) lors de la
           prochaine instruction (evite les boucles infinies
           sur les points d'arret)
    VM   : Active le mode Virtual 8086 (le processeur execute le
           code comme un 8086 mais sous supervision du noyau
           protege)

  2.6 Registres de controle
  -------------------------

  Le 80386 possede 3 registres de controle de 32 bits accessibles
  uniquement en mode protege (anneau 0) via MOV CRn, reg / MOV reg, CRn.

  CR0 (Control Register 0) :
    Bit 0  : PE (Protection Enable) - Active le mode protege
    Bit 1  : MP (Monitor Coprocessor) - Surveillance du coprocesseur
    Bit 2  : EM (Emulation) - Emulation du coprocesseur x87
    Bit 3  : TS (Task Switched) - Indique un changement de tache
    Bit 4  : ET (Extension Type) - Type de coprocesseur (387 vs 287)
    Bit 16 : WP (Write Protect) - Protection en ecriture des pages
             utilisateur meme en anneau 0 (absent sur 80386 initial,
             ajoute sur 80486)
    Bit 31 : PG (Paging) - Active la pagination

    Passage en mode protege : activer PE (bit 0 de CR0)
    Activation de la pagination : activer PG (bit 31 de CR0)
    Les deux peuvent etre actives simultanement

  CR1 (Control Register 1) :
    Reserve par Intel. Non utilise sur le 80386.

  CR2 (Control Register 2) :
    Contient l'adresse lineaire de 32 bits qui a provoque le
    dernier defaut de page (#PF, exception 14). Le gestionnaire
    de defaut de page lit CR2 pour identifier la page fautive.

  CR3 (Control Register 3) - PDBR (Page Directory Base Register) :
    Contient l'adresse physique de base du repertoire de pages
    (Page Directory). Les 12 bits de poids faible sont ignores
    (le repertoire doit etre aligne sur une limite de 4 Ko).
    La modification de CR3 invalide le TLB (Translation Lookaside
    Buffer).

  2.7 Registres de debogage
  --------------------------

  Le 80386 possede 8 registres de debogage de 32 bits (DR0 a DR7)
  permettant de definir des points d'arret materiels sans modifier
  le code.

  DR0, DR1, DR2, DR3 :
    Contiennent les adresses lineaires des 4 points d'arret
    materiels. Chaque point d'arret est independant et peut
    surveiller une adresse differente.

  DR4, DR5 :
    Reserves (aliases de DR6/DR7 sur le 80386).

  DR6 (Debug Status Register) :
    Indique la cause de la derniere exception de debogage (#DB) :
    Bit 0 (B0) : Point d'arret 0 declenche
    Bit 1 (B1) : Point d'arret 1 declenche
    Bit 2 (B2) : Point d'arret 2 declenche
    Bit 3 (B3) : Point d'arret 3 declenche
    Bit 13 (BD) : Acces a un registre de debogage detecte
    Bit 14 (BS) : Mode pas-a-pas (TF) declenche
    Bit 15 (BT) : Changement de tache declenche
    Le processeur ne remet pas DR6 a zero automatiquement;
    le gestionnaire d'exception doit le faire manuellement.

  DR7 (Debug Control Register) :
    Configure le comportement des 4 points d'arret :
    Bits 0-7 : Activation locale (L0-L3) et globale (G0-G3)
      pour chaque point d'arret (L = tache courante, G = toutes)
    Bits 16-31 : Conditions et taille pour chaque point d'arret
      (4 bits par point d'arret : R/Wn sur 2 bits + LENn sur 2 bits)

    Conditions (R/Wn) :
      00 = Execution d'instruction uniquement
      01 = Ecriture en memoire uniquement
      10 = Acces port E/S (non supporte sur 80386)
      11 = Lecture ou ecriture en memoire

    Taille (LENn) :
      00 = 1 octet
      01 = 2 octets (adresse alignee sur mot)
      10 = (non defini sur 80386)
      11 = 4 octets (adresse alignee sur double mot)

  2.8 Registres de test
  ---------------------

  Le 80386 possede 2 registres de test de 32 bits pour le
  diagnostic du TLB (Translation Lookaside Buffer) :

  TR6 (Test Command Register) :
    Permet d'ecrire une entree dans le TLB ou de rechercher
    une correspondance. Contient l'adresse lineaire, les
    attributs et la commande (lecture/ecriture).

  TR7 (Test Data Register) :
    Contient l'adresse physique et les donnees associees a
    l'operation de test du TLB. Le bit HT indique si la
    recherche a trouve une correspondance.

  Ces registres sont principalement utilises par le BIOS et les
  systemes d'exploitation pour tester le bon fonctionnement du
  TLB. Ils sont rarement utilises en programmation applicative.

  2.9 Resume des registres
  ------------------------

  Categorie                  Registres            Taille  Nombre
  -------------------------  -------------------  ------  ------
  Registres generaux         EAX EBX ECX EDX      32 bits     4
                             ESI EDI EBP ESP      32 bits     4
  Registres de segment       CS DS ES FS GS SS    16 bits     6
  Pointeur d'instruction     EIP                  32 bits     1
  Drapeaux                   EFLAGS               32 bits     1
  Registres de controle      CR0 CR1 CR2 CR3      32 bits     4
  Registres de debogage      DR0-DR3 DR6 DR7      32 bits     6
  Registres de test          TR6 TR7              32 bits     2
  -------------------------  -------------------  ------  ------
  Total                                                      28

3. MODES D'ADRESSAGE
==============================================================================

  3.1 Vue d'ensemble
  ------------------

  Le 80386 dispose d'un jeu de modes d'adressage riche et regulier.
  Chaque operande d'une instruction peut etre :

    - Une valeur immediate (constante codee dans l'instruction)
    - Un registre (general, segment ou de controle)
    - Un emplacement memoire designe par une adresse effective

  L'adresse effective (Effective Address, EA) est calculee a partir
  d'une combinaison de :

    Base          : Un registre general 32 bits (EAX..ESP, EBP..EDI)
    Index         : Un registre general 32 bits sauf ESP
    Echelle       : Un facteur multiplicatif (1, 2, 4 ou 8)
    Deplacement   : Une constante signee de 0, 8 ou 32 bits

  La formule generale de l'adresse effective est :

    EA = Base + (Index * Echelle) + Deplacement

  Chaque composant est optionnel. Les differentes combinaisons
  donnent les modes d'adressage decrits ci-dessous.

  En mode reel, le calcul d'adresse physique ajoute la base du
  segment (segment << 4). En mode protege, la base du segment est
  obtenue via le descripteur reference par le selecteur de segment.

  3.2 Adressage immediat
  ----------------------

  L'operande est une constante codee directement dans l'instruction.
  Il ne peut etre utilise que comme operande source (jamais comme
  destination).

  Syntaxe :
    MOV  reg, imm          ; charger une constante dans un registre
    ADD  reg, imm          ; additionner une constante
    CMP  reg, imm          ; comparer avec une constante
    PUSH imm               ; empiler une constante (nouveau 80386)

  Tailles possibles :
    imm8   : Constante de 8 bits (-128 a +127 ou 0 a 255)
    imm16  : Constante de 16 bits (-32768 a +32767 ou 0 a 65535)
    imm32  : Constante de 32 bits (nouveau 80386)

  Exemples :
    MOV  EAX, 42                   ; EAX = 42 (imm32)
    MOV  AL, 0FFh                  ; AL = 255 (imm8)
    ADD  ECX, 1000                 ; ECX += 1000
    PUSH 12345678h                 ; empiler une constante 32 bits
    CMP  BYTE PTR [ESI], 0        ; comparer un octet avec zero
    AND  EAX, 0FF00FF00h          ; masquage de bits

  Remarques :
    - Les constantes 8 bits sont souvent etendues avec signe a
      32 bits, ce qui donne des instructions plus compactes
    - PUSH imm est une nouveaute du 80186, etendue a 32 bits
      sur le 80386
    - Les formes avec AL/AX/EAX ont un codage special plus court

  3.3 Adressage par registre
  --------------------------

  L'operande est un registre du processeur. C'est le mode le plus
  rapide car aucun acces memoire n'est necessaire.

  Syntaxe :
    MOV  reg1, reg2        ; copier un registre dans un autre
    ADD  reg1, reg2        ; additionner deux registres
    XCHG reg1, reg2        ; echanger deux registres

  Registres utilisables :
    32 bits : EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP
    16 bits : AX, BX, CX, DX, SI, DI, BP, SP
     8 bits : AL, AH, BL, BH, CL, CH, DL, DH
    Segment : CS, DS, ES, FS, GS, SS

  Exemples :
    MOV  EAX, EBX                  ; EAX = EBX
    ADD  ECX, EDX                  ; ECX += EDX
    XOR  EAX, EAX                 ; EAX = 0 (idiome classique)
    MOV  DS, AX                   ; charger un selecteur de segment
    MOVZX EAX, CL                 ; etendre CL (8 bits) vers EAX
    PUSH EBP                      ; sauvegarder le pointeur de base

  Remarques :
    - Les operations registre-registre s'executent en 1 cycle
      (pas d'acces memoire)
    - Les melanges de taille (ex: MOV EAX, BX) ne sont pas
      autorises directement; utiliser MOVZX ou MOVSX
    - ESP ne peut pas etre utilise comme registre d'index

  3.4 Adressage direct (deplacement seul)
  ----------------------------------------

  L'adresse effective est une constante de 32 bits codee dans
  l'instruction. Elle designe directement un emplacement memoire.

  Formule : EA = Deplacement32

  Syntaxe :
    MOV  reg, [depl32]     ; charger depuis une adresse fixe
    MOV  [depl32], reg     ; stocker a une adresse fixe

  Exemples :
    MOV  EAX, [00401000h]          ; lire le DWORD a l'adresse 401000h
    MOV  EAX, DWORD PTR [_var]     ; lire la variable globale _var
    MOV  BYTE PTR [_flag], 1       ; mettre _flag a 1
    INC  DWORD PTR [_compteur]     ; incrementer un compteur global
    FLD  QWORD PTR [_pi]           ; charger un flottant 64 bits

  Utilisation typique :
    - Acces aux variables globales
    - Acces aux constantes en memoire
    - Acces aux tables de donnees a adresse fixe

  Remarques :
    - Sur le 80386, le deplacement est toujours 32 bits en mode
      protege (contrairement au 16 bits du 8086)
    - L'assembleur traduit automatiquement les noms de variables
      en adresses (deplacements)

  3.5 Adressage indirect par registre
  ------------------------------------

  L'adresse effective est contenue dans un registre. Le registre
  sert de pointeur vers l'emplacement memoire.

  Formule : EA = [Registre]

  Syntaxe :
    MOV  reg, [base_reg]   ; charger depuis l'adresse dans base_reg
    MOV  [base_reg], reg   ; stocker a l'adresse dans base_reg

  Registres de base utilisables :
    EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP
    (tous les registres generaux 32 bits)

  Segment implicite :
    [EBP] et [ESP] utilisent SS par defaut
    Tous les autres utilisent DS par defaut
    Peut etre surcharge par un prefixe de segment (ex: ES:[EAX])

  Exemples :
    MOV  EAX, [EBX]                ; lire le DWORD pointe par EBX
    MOV  [ESI], ECX                ; stocker ECX a l'adresse dans ESI
    MOV  AL, [EDI]                 ; lire un octet pointe par EDI
    CALL [EAX]                     ; appel indirect via un pointeur
    JMP  [EBX]                     ; saut indirect via un pointeur

  Utilisation typique :
    - Dereferencement de pointeurs
    - Parcours de listes chainees
    - Appels de fonctions via des tables de pointeurs (vtable)

  Remarques :
    - Sur le 8086, seuls BX, SI, DI et BP pouvaient etre utilises
      comme registres de base. Le 80386 autorise tous les registres
      generaux, ce qui simplifie grandement la programmation
    - [ESP] utilise un octet SIB supplementaire dans l'encodage

  3.6 Adressage base + deplacement
  ---------------------------------

  L'adresse effective est la somme d'un registre de base et d'un
  deplacement (constante). C'est le mode le plus utilise pour
  acceder aux variables locales et aux parametres de fonctions.

  Formule : EA = Base + Deplacement

  Syntaxe :
    MOV  reg, [base_reg + depl]
    MOV  [base_reg + depl], reg

  Tailles de deplacement :
    disp8  : Deplacement signe de 8 bits (-128 a +127)
    disp32 : Deplacement signe de 32 bits

  L'assembleur choisit automatiquement la taille la plus compacte.

  Exemples :
    MOV  EAX, [EBP+8]             ; 1er parametre (convention cdecl)
    MOV  EAX, [EBP+12]            ; 2e parametre
    MOV  EAX, [EBP-4]             ; 1ere variable locale
    MOV  EAX, [EBP-8]             ; 2e variable locale
    MOV  EAX, [EBX+16]            ; champ d'une structure
    MOV  AL, [ESI+3]              ; 4e octet d'un buffer

  Utilisation typique :
    - Acces aux parametres de fonctions via EBP
      (convention : [EBP+8], [EBP+12], [EBP+16], ...)
    - Acces aux variables locales via EBP
      (convention : [EBP-4], [EBP-8], [EBP-12], ...)
    - Acces aux champs d'une structure via un pointeur
    - Acces aux elements d'un tableau via un pointeur de base

  Organisation typique du cadre de pile (stack frame) :

    [EBP+16]   3e parametre
    [EBP+12]   2e parametre
    [EBP+8]    1er parametre
    [EBP+4]    Adresse de retour (empilee par CALL)
    [EBP+0]    Ancien EBP (empile par PUSH EBP)
    [EBP-4]    1ere variable locale
    [EBP-8]    2e variable locale

  3.7 Adressage indexe (base + index)
  ------------------------------------

  L'adresse effective est la somme d'un registre de base et d'un
  registre d'index. Ce mode est utile pour le parcours de tableaux
  avec un pointeur de base fixe et un index variable.

  Formule : EA = Base + Index

  Syntaxe :
    MOV  reg, [base_reg + index_reg]
    MOV  [base_reg + index_reg], reg

  Le registre d'index peut etre n'importe quel registre general
  32 bits sauf ESP.

  Exemples :
    MOV  EAX, [EBX+ESI]           ; tableau pointe par EBX, index ESI
    MOV  [EDI+ECX], AL            ; ecrire un octet dans un buffer
    MOV  EAX, [EBP+ECX]           ; variable locale indexee
    LEA  EAX, [EBX+EDX]           ; calculer une adresse sans lire

  Utilisation typique :
    - Parcours de tableaux (base = debut du tableau, index = offset)
    - Copie de blocs memoire element par element

  3.8 Adressage indexe avec echelle (SIB)
  ----------------------------------------

  C'est le mode d'adressage le plus puissant du 80386, nouveau
  par rapport au 8086/80286. L'adresse effective combine un registre
  de base, un registre d'index multiplie par un facteur d'echelle,
  et un deplacement optionnel.

  Formule : EA = Base + (Index * Echelle) + Deplacement

  Facteurs d'echelle possibles : 1, 2, 4 ou 8

  L'octet SIB (Scale-Index-Base) est un octet supplementaire dans
  l'encodage de l'instruction qui specifie les trois composants :
    - Bits 7-6 : Echelle (00=1, 01=2, 10=4, 11=8)
    - Bits 5-3 : Registre d'index (sauf ESP = pas d'index)
    - Bits 2-0 : Registre de base

  Syntaxe :
    MOV  reg, [base + index * echelle]
    MOV  reg, [base + index * echelle + depl]
    MOV  reg, [index * echelle + depl]        ; sans base

  Exemples :

    Acces a un tableau de DWORD (4 octets par element) :
      MOV  EAX, [EBX+ECX*4]       ; tab[i], EBX=tab, ECX=i
      MOV  EAX, [_tab+ECX*4]      ; tab[i], adresse directe

    Acces a un tableau de WORD (2 octets par element) :
      MOV  AX, [ESI+EDI*2]        ; tab[i], ESI=tab, EDI=i

    Acces a un tableau de QWORD (8 octets par element) :
      FLD  QWORD PTR [EBX+ECX*8]  ; tab_double[i]

    Acces a un champ dans un tableau de structures :
      MOV  EAX, [EBX+ECX*8+4]     ; tab[i].champ2
                                   ; (structures de 8 octets,
                                   ;  champ2 a l'offset 4)

    Table de sauts (jump table) :
      JMP  [_jtable+EAX*4]        ; saut indirect indexe

    Calcul d'adresse avec LEA :
      LEA  EAX, [EAX+EAX*4]       ; EAX = EAX * 5
      LEA  EAX, [EBX+ECX*8+12]    ; calcul sans acces memoire

  Utilisation typique :
    - Acces aux tableaux de types scalaires (WORD, DWORD, QWORD)
      sans avoir a calculer l'offset manuellement
    - Acces aux champs dans des tableaux de structures
    - Tables de sauts pour les instructions switch/case
    - Calculs arithmetiques rapides via LEA (multiplication
      par 2, 3, 4, 5, 8, 9 sans utiliser MUL/IMUL)

  Correspondance echelle / type de donnees :
    Echelle 1 : BYTE (octets, caracteres)
    Echelle 2 : WORD (entiers 16 bits, Unicode)
    Echelle 4 : DWORD (entiers 32 bits, pointeurs, float)
    Echelle 8 : QWORD (entiers 64 bits, double)

  3.9 Adressage relatif (branchements)
  -------------------------------------

  Les instructions de branchement (JMP, Jcc, CALL, LOOP) utilisent
  un deplacement relatif par rapport a l'adresse de l'instruction
  suivante. Ce mode n'est pas utilise pour les operandes memoire
  mais uniquement pour le calcul de la cible des sauts.

  Formule : Cible = EIP + Deplacement

  Tailles de deplacement :
    rel8  : Deplacement signe de 8 bits (-128 a +127 octets)
            Utilise pour les sauts courts (short jump)
    rel32 : Deplacement signe de 32 bits (nouveau 80386)
            Utilise pour les sauts proches (near jump)

  Exemples :
    JMP  SHORT _boucle             ; saut court (rel8)
    JMP  NEAR PTR _fonction        ; saut proche (rel32)
    JE   _si_egal                  ; saut conditionnel (rel8 ou rel32)
    CALL _ma_procedure             ; appel relatif (rel32)
    LOOP _boucle                   ; boucle (rel8 uniquement)
    JECXZ _vide                    ; saut si ECX=0 (rel8)

  Remarques :
    - Les sauts conditionnels (Jcc) n'acceptaient que rel8 sur le
      8086. Le 80386 ajoute les sauts conditionnels rel32
      (prefixe 0Fh), ce qui elimine le besoin de sauts
      intermediaires pour les branches distantes
    - L'adressage relatif rend le code relogeable (position-
      independent) car les adresses ne sont pas absolues
    - LOOP et JECXZ n'existent qu'en version rel8

  3.10 Prefixes de surcharge de segment
  ---------------------------------------

  Par defaut, chaque mode d'adressage utilise un registre de
  segment implicite :

    DS : Pour la plupart des acces memoire (MOV, ADD, etc.)
    SS : Pour les acces via EBP ou ESP ([EBP+x], [ESP+x])
    CS : Pour le fetch des instructions (implicite)
    ES : Pour la destination des instructions de chaines
         (MOVS, STOS, SCAS, CMPS via ES:[EDI])

  Un prefixe de segment peut surcharger le segment par defaut :

    MOV  EAX, ES:[EBX]            ; utiliser ES au lieu de DS
    MOV  EAX, FS:[0]              ; utiliser FS (ex: TEB Windows)
    MOV  EAX, GS:[EDI+8]          ; utiliser GS
    MOV  EAX, CS:[_table]         ; lire dans le segment de code
    MOV  EAX, SS:[ECX+4]          ; forcer le segment de pile

  Utilisation typique :
    - FS : Acces au Thread Environment Block (TEB) sous Windows
      (FS:[0] = chaine SEH, FS:[18h] = pointeur TEB)
    - GS : Utilise par certains systemes d'exploitation
    - ES : Destination implicite pour les instructions de chaines
    - CS : Acces a des donnees dans le segment de code (tables
      de constantes dans du code relogeable)

  En mode FLAT 32 bits (Windows, Linux), les segments CS, DS, ES
  et SS pointent generalement sur le meme espace lineaire. Les
  prefixes de segment sont alors rarement necessaires, sauf pour
  FS et GS qui sont souvent configures differemment.

  3.11 Resume et tableau comparatif
  ----------------------------------

  Mode                     Syntaxe                  Formule EA
  -----------------------  -----------------------  ---------------------
  Immediat                 MOV EAX, 42              (pas d'EA, source)
  Registre                 MOV EAX, EBX             (pas d'EA)
  Direct                   MOV EAX, [_var]          EA = depl32
  Indirect registre        MOV EAX, [EBX]           EA = base
  Base + deplacement       MOV EAX, [EBP+8]         EA = base + depl
  Base + index             MOV EAX, [EBX+ESI]       EA = base + index
  Index * echelle + depl   MOV EAX, [ECX*4+_tab]    EA = idx*s + depl
  Base + index * echelle   MOV EAX, [EBX+ECX*4]     EA = base + idx*s
  Complet (SIB)            MOV EAX, [EBX+ECX*4+8]   EA = base+idx*s+depl
  Relatif (sauts)          JMP _label               cible = EIP + rel

  Comparaison avec le 8086 :

  Caracteristique             8086              80386
  --------------------------  ----------------  ----------------------
  Registres de base           BX, BP            EAX..EDI (tous les 8)
  Registres d'index           SI, DI            EAX..EDI sauf ESP
  Facteur d'echelle           Non               1, 2, 4, 8
  Taille deplacement          0, 8, 16 bits     0, 8, 32 bits
  Taille adresse effective    16 bits            32 bits
  Sauts conditionnels         rel8 uniquement    rel8 et rel32
  Segments supplementaires    Non                FS, GS
  Octet SIB                   Non               Oui

==============================================================================
4. JEU D'INSTRUCTIONS (OPCODES)
==============================================================================

  4.1 Vue d'ensemble
  ------------------

  Le 80386 herite du jeu d'instructions du 8086/80286 et l'etend
  avec de nouvelles instructions 32 bits. Toutes les instructions
  existantes sont disponibles en version 32 bits grace aux prefixes
  de taille d'operande (66h) et de taille d'adresse (67h).

  Le jeu d'instructions se divise en categories :
    - Transfert de donnees
    - Arithmetique entiere
    - Logique et manipulation de bits
    - Comparaison et test
    - Branchements et sauts
    - Operations sur les chaines
    - Manipulation des drapeaux
    - Entrees/sorties
    - Instructions systeme et mode protege
    - Conversions de type
    - Instructions nouvelles du 80386

  Conventions de notation utilisees dans cette section :
    reg    : Registre general (8, 16 ou 32 bits)
    r/m    : Registre ou operande memoire
    imm    : Valeur immediate (constante)
    mem    : Operande memoire
    seg    : Registre de segment
    acc    : Accumulateur (AL, AX ou EAX)
    [F]    : Drapeaux affectes

  4.2 Transfert de donnees
  ------------------------

  Ces instructions copient des donnees entre registres, memoire et
  valeurs immediates sans modifier les drapeaux (sauf indication).

  4.2.1 Transfert general

  MOV  dest, src         Copier src dans dest
                         Formes : reg<-reg, reg<-mem, mem<-reg,
                                  reg<-imm, mem<-imm, reg<-seg,
                                  seg<-r/m
                         MOV ne peut pas : mem<-mem, seg<-imm

  XCHG reg, r/m          Echanger les contenus de deux operandes
                         XCHG EAX,reg a un codage sur 1 octet (90h+r)
                         XCHG avec memoire implique un verrou LOCK

  4.2.2 Pile

  PUSH src               Empiler : ESP -= taille, [ESP] <- src
                         Formes : reg, mem, imm, seg
                         PUSH imm est nouveau depuis le 80186

  POP  dest              Depiler : dest <- [ESP], ESP += taille
                         Formes : reg, mem, seg (sauf CS)

  PUSHA                  Empiler les 8 registres generaux
  PUSHAD                 (version 32 bits : EAX,ECX,EDX,EBX,
                          ESP_original,EBP,ESI,EDI)

  POPA                   Depiler les 8 registres generaux
  POPAD                  (version 32 bits, restaure dans l'ordre inverse)

  PUSHF                  Empiler le registre EFLAGS (16 bits : FLAGS)
  PUSHFD                 Empiler le registre EFLAGS (32 bits)

  POPF                   Depiler dans FLAGS [F: tous]
  POPFD                  Depiler dans EFLAGS [F: tous]

  4.2.3 Chargement d'adresse et de pointeur

  LEA  reg, mem          Charger l'adresse effective de mem dans reg
                         (pas d'acces memoire, calcul d'adresse seul)
                         Souvent utilise pour l'arithmetique :
                           LEA EAX, [EBX+ECX*4+8]

  LDS  reg, mem          Charger un pointeur FAR 48 bits :
                         reg <- [mem], DS <- [mem+4]

  LES  reg, mem          Idem avec ES
  LFS  reg, mem          Idem avec FS (nouveau 80386)
  LGS  reg, mem          Idem avec GS (nouveau 80386)
  LSS  reg, mem          Idem avec SS (nouveau 80386)

  4.2.4 Conversion et extension

  CBW                    Etendre AL (signe) dans AX
  CWDE                   Etendre AX (signe) dans EAX (nouveau 80386)
  CWD                    Etendre AX (signe) dans DX:AX
  CDQ                    Etendre EAX (signe) dans EDX:EAX (nouveau 80386)

  MOVZX reg, r/m         Charger avec extension zero (nouveau 80386)
                         reg16 <- r/m8, reg32 <- r/m8, reg32 <- r/m16

  MOVSX reg, r/m         Charger avec extension de signe (nouveau 80386)
                         reg16 <- r/m8, reg32 <- r/m8, reg32 <- r/m16

  4.2.5 Transfert conditionnel de drapeaux

  LAHF                   AH <- FLAGS (bits SF, ZF, AF, PF, CF)
  SAHF                   FLAGS <- AH [F: SF, ZF, AF, PF, CF]

  4.3 Arithmetique entiere
  -------------------------

  4.3.1 Addition

  ADD  dest, src         dest <- dest + src [F: OF,SF,ZF,AF,PF,CF]
  ADC  dest, src         dest <- dest + src + CF [F: OF,SF,ZF,AF,PF,CF]
                         (addition avec retenue, pour l'arithmetique
                          multi-precision)

  INC  r/m               r/m <- r/m + 1 [F: OF,SF,ZF,AF,PF]
                         (ne modifie pas CF, utile dans les boucles)

  4.3.2 Soustraction

  SUB  dest, src         dest <- dest - src [F: OF,SF,ZF,AF,PF,CF]
  SBB  dest, src         dest <- dest - src - CF [F: OF,SF,ZF,AF,PF,CF]
                         (soustraction avec emprunt)

  DEC  r/m               r/m <- r/m - 1 [F: OF,SF,ZF,AF,PF]
                         (ne modifie pas CF)

  NEG  r/m               r/m <- 0 - r/m [F: OF,SF,ZF,AF,PF,CF]
                         (complement a deux, negation)

  4.3.3 Multiplication

  MUL  r/m               Multiplication non signee
                         8 bits  : AX <- AL * r/m8
                         16 bits : DX:AX <- AX * r/m16
                         32 bits : EDX:EAX <- EAX * r/m32
                         [F: OF,CF (definis si partie haute non nulle)]

  IMUL r/m               Multiplication signee (meme format que MUL)

  IMUL reg, r/m          reg <- reg * r/m (forme a 2 operandes)
                         (nouveau 80386, resultat tronque a la taille
                          du registre destination)

  IMUL reg, r/m, imm     reg <- r/m * imm (forme a 3 operandes)
                         (nouveau 80186)

  4.3.4 Division

  DIV  r/m               Division non signee
                         8 bits  : AL <- AX / r/m8, AH <- AX MOD r/m8
                         16 bits : AX <- DX:AX / r/m16, DX <- reste
                         32 bits : EAX <- EDX:EAX / r/m32, EDX <- reste
                         Exception #DE si diviseur = 0 ou quotient
                         depasse la taille du registre destination

  IDIV r/m               Division signee (meme format que DIV)

  4.3.5 Ajustement BCD (decimal code binaire)

  DAA                    Ajustement decimal apres addition (AL)
  DAS                    Ajustement decimal apres soustraction (AL)
  AAA                    Ajustement ASCII apres addition (AX)
  AAS                    Ajustement ASCII apres soustraction (AX)
  AAM                    Ajustement ASCII apres multiplication (AX)
  AAD                    Ajustement ASCII avant division (AX)

  Note : Ces instructions operent uniquement sur AL/AX et sont
  rarement utilisees en programmation 32 bits moderne.

  4.4 Logique et manipulation de bits
  ------------------------------------

  4.4.1 Operations logiques

  AND  dest, src         dest <- dest AND src [F: OF=0,SF,ZF,PF,CF=0]
  OR   dest, src         dest <- dest OR src  [F: OF=0,SF,ZF,PF,CF=0]
  XOR  dest, src         dest <- dest XOR src [F: OF=0,SF,ZF,PF,CF=0]
  NOT  r/m               r/m <- NOT r/m (complement a un) [F: aucun]

  Idiomes courants :
    XOR EAX, EAX         Mettre EAX a zero (2 octets, plus rapide
                          que MOV EAX, 0 qui fait 5 octets)
    OR  EAX, EAX         Tester si EAX est zero (positionne ZF)
    AND EAX, 0FFh        Masquer les bits de poids fort

  4.4.2 Decalages

  SHL  r/m, count        Decalage logique a gauche [F: CF,OF,SF,ZF,PF]
  SHR  r/m, count        Decalage logique a droite [F: CF,OF,SF,ZF,PF]
  SAR  r/m, count        Decalage arithmetique a droite (preserve signe)
  SAL  r/m, count        Synonyme de SHL

  Le compteur (count) peut etre :
    1    : Decalage d'une position (codage specifique)
    imm8 : Constante immediate (masquee a 5 bits, max 31)
    CL   : Valeur du registre CL (masquee a 5 bits)

  Note : SHL de n positions equivaut a une multiplication par 2^n.
  SHR de n positions equivaut a une division non signee par 2^n.
  SAR de n positions equivaut a une division signee par 2^n.

  4.4.3 Rotations

  ROL  r/m, count        Rotation a gauche [F: CF,OF]
  ROR  r/m, count        Rotation a droite [F: CF,OF]
  RCL  r/m, count        Rotation a gauche a travers CF [F: CF,OF]
  RCR  r/m, count        Rotation a droite a travers CF [F: CF,OF]

  Les rotations sont circulaires : les bits qui sortent d'un cote
  rentrent de l'autre. RCL/RCR incluent CF dans la rotation,
  formant une chaine de (n+1) bits.

  4.4.4 Decalage double (nouveau 80386)

  SHLD dest, src, count  Decalage a gauche double precision
                         Les bits sortants de src remplissent dest
                         [F: CF,SF,ZF,PF]

  SHRD dest, src, count  Decalage a droite double precision
                         Les bits sortants de src remplissent dest
                         [F: CF,SF,ZF,PF]

  Ces instructions sont tres utiles pour les operations sur des
  entiers de plus de 32 bits et pour les operations sur les bitmaps.

  4.4.5 Test et recherche de bits (nouveau 80386)

  BT   r/m, reg/imm      Test du bit : CF <- bit selectionne
  BTS  r/m, reg/imm      Test et mise a 1 : CF <- bit, bit <- 1
  BTR  r/m, reg/imm      Test et mise a 0 : CF <- bit, bit <- 0
  BTC  r/m, reg/imm      Test et complement : CF <- bit, bit <- NOT bit

  BSF  reg, r/m          Recherche du premier bit a 1 (depuis bit 0)
                         reg <- index du bit, ZF=1 si aucun bit a 1

  BSR  reg, r/m          Recherche du dernier bit a 1 (depuis bit MSB)
                         reg <- index du bit, ZF=1 si aucun bit a 1

  4.5 Comparaison et test
  ------------------------

  CMP  dest, src         Calcule dest - src, positionne les drapeaux
                         mais ne stocke pas le resultat
                         [F: OF,SF,ZF,AF,PF,CF]

  TEST dest, src         Calcule dest AND src, positionne les drapeaux
                         mais ne stocke pas le resultat
                         [F: OF=0,SF,ZF,PF,CF=0]

  Utilisation typique :
    CMP EAX, 10          Comparer EAX avec 10
    JE  _egal            Sauter si egal (ZF=1)

    TEST EAX, EAX        Tester si EAX est zero
    JZ  _est_zero        Sauter si zero (ZF=1)

    TEST AL, 01h         Tester si le bit 0 est a 1
    JNZ _impair          Sauter si non-zero (bit 0 = 1)

  4.6 Branchements et sauts
  --------------------------

  4.6.1 Saut inconditionnel

  JMP  cible             Saut inconditionnel
                         Court (rel8), proche (rel32), ou lointain
                         (seg:offset pour inter-segment)
                         Peut etre direct (label) ou indirect ([reg])

  4.6.2 Sauts conditionnels

  Les sauts conditionnels testent les drapeaux positionnes par
  CMP, TEST, ou toute instruction arithmetique/logique precedente.

  Sur le 80386, tous les sauts conditionnels acceptent les
  deplacements rel8 (court) et rel32 (proche), alors que le 8086
  ne supportait que rel8.

  Condition             Mnemoniques         Drapeaux testes
  --------------------  ------------------  ----------------
  Egal / zero           JE   / JZ           ZF = 1
  Non egal / non zero   JNE  / JNZ          ZF = 0
  Retenue / inf. (ns)   JC   / JB  / JNAE   CF = 1
  Pas de retenue         JNC  / JNB / JAE    CF = 0
  Signe (negatif)       JS                   SF = 1
  Pas de signe          JNS                  SF = 0
  Debordement           JO                   OF = 1
  Pas de debordement    JNO                  OF = 0
  Parite paire          JP   / JPE           PF = 1
  Parite impaire        JNP  / JPO           PF = 0

  Comparaisons non signees (unsigned) :
  Inferieur             JB   / JNAE / JC     CF = 1
  Inf. ou egal          JBE  / JNA           CF = 1 ou ZF = 1
  Superieur             JA   / JNBE          CF = 0 et ZF = 0
  Sup. ou egal          JAE  / JNB / JNC     CF = 0

  Comparaisons signees (signed) :
  Inferieur             JL   / JNGE          SF <> OF
  Inf. ou egal          JLE  / JNG           SF <> OF ou ZF = 1
  Superieur             JG   / JNLE          SF = OF et ZF = 0
  Sup. ou egal          JGE  / JNL           SF = OF

  4.6.3 Boucles

  LOOP  cible            ECX <- ECX - 1, sauter si ECX <> 0
  LOOPE cible            ECX <- ECX - 1, sauter si ECX <> 0 et ZF = 1
  LOOPNE cible           ECX <- ECX - 1, sauter si ECX <> 0 et ZF = 0
  JECXZ cible            Sauter si ECX = 0 (sans modifier ECX)

  Note : LOOP et JECXZ n'acceptent que des deplacements rel8.
  En mode 16 bits, LOOP utilise CX au lieu de ECX.

  4.6.4 Appels de sous-programmes

  CALL cible             Empiler EIP, sauter a cible
                         Proche (rel32 ou r/m32) : PUSH EIP, JMP cible
                         Lointain (seg:offset) : PUSH CS, PUSH EIP,
                           puis saut inter-segment

  RET                    Depiler EIP et reprendre l'execution
  RET  imm16             Depiler EIP, puis ESP <- ESP + imm16
                         (pour nettoyer les parametres de la pile,
                          convention stdcall)

  ENTER imm16, level     Creer un cadre de pile :
                         PUSH EBP, EBP <- ESP, ESP <- ESP - imm16
                         (level > 0 pour les cadres imbriques)

  LEAVE                  Detruire un cadre de pile :
                         ESP <- EBP, POP EBP
                         (equivalent a MOV ESP,EBP / POP EBP)

  4.6.5 Interruptions logicielles

  INT  imm8              Appel d'interruption logicielle
                         PUSH EFLAGS, PUSH CS, PUSH EIP,
                         IF <- 0, TF <- 0, saut via IDT[imm8]

  INTO                   INT 4 si OF = 1 (debordement)

  IRET                   Retour d'interruption
  IRETD                  POP EIP, POP CS, POP EFLAGS

  4.6.6 Positionnement conditionnel d'octets (nouveau 80386)

  SETcc r/m8             Mettre r/m8 a 1 si condition vraie, 0 sinon

  Les suffixes cc sont les memes que pour les sauts Jcc :
    SETE/SETZ, SETNE/SETNZ, SETB/SETC, SETNB/SETNC,
    SETS, SETNS, SETO, SETNO, SETP, SETNP,
    SETL, SETLE, SETG, SETGE, SETA, SETBE, SETAE, SETB

  Utilisation typique :
    CMP  EAX, EBX
    SETL AL               ; AL = 1 si EAX < EBX (signe), sinon 0
    MOVZX EAX, AL          ; etendre a 32 bits

  4.7 Operations sur les chaines
  -------------------------------

  Les instructions de chaines operent sur des blocs de memoire.
  Elles utilisent :
    - ESI (source) avec DS par defaut
    - EDI (destination) avec ES obligatoire
    - ECX comme compteur (avec REP)
    - DF (Direction Flag) : 0 = incrementer, 1 = decrementer

  4.7.1 Instructions de base

  MOVSB / MOVSW / MOVSD  Copier [ESI] vers ES:[EDI], avancer les deux
                         (B=octet, W=mot, D=double mot)

  CMPSB / CMPSW / CMPSD  Comparer [ESI] avec ES:[EDI], avancer
                         [F: OF,SF,ZF,AF,PF,CF]

  SCASB / SCASW / SCASD  Comparer AL/AX/EAX avec ES:[EDI], avancer
                         [F: OF,SF,ZF,AF,PF,CF]

  LODSB / LODSW / LODSD  Charger [ESI] dans AL/AX/EAX, avancer ESI

  STOSB / STOSW / STOSD  Stocker AL/AX/EAX dans ES:[EDI], avancer EDI

  INSB / INSW / INSD     Lire port DX dans ES:[EDI], avancer EDI
  OUTSB / OUTSW / OUTSD  Ecrire [ESI] vers port DX, avancer ESI

  4.7.2 Prefixes de repetition

  REP                    Repeter ECX fois (pour MOVS, STOS, INS, OUTS)
  REPE / REPZ            Repeter tant que ECX <> 0 et ZF = 1
                         (pour CMPS, SCAS)
  REPNE / REPNZ          Repeter tant que ECX <> 0 et ZF = 0
                         (pour CMPS, SCAS)

  Exemples :
    ; Copier 100 octets de ESI vers EDI
    MOV  ECX, 100
    CLD                    ; direction = incrementer
    REP  MOVSB

    ; Remplir 256 octets a zero
    MOV  ECX, 64           ; 256 / 4 = 64 double mots
    XOR  EAX, EAX
    CLD
    REP  STOSD

    ; Chercher un octet nul dans une chaine (strlen)
    MOV  ECX, -1           ; compter le maximum
    XOR  AL, AL            ; chercher 0
    CLD
    REPNE SCASB
    NOT  ECX
    DEC  ECX               ; ECX = longueur de la chaine

  4.8 Manipulation des drapeaux
  ------------------------------

  CLC                    CF <- 0 (Clear Carry)
  STC                    CF <- 1 (Set Carry)
  CMC                    CF <- NOT CF (Complement Carry)

  CLD                    DF <- 0 (direction = incrementer)
  STD                    DF <- 1 (direction = decrementer)

  CLI                    IF <- 0 (desactiver les interruptions)
  STI                    IF <- 1 (activer les interruptions)

  Note : CLI/STI sont des instructions privilegiees en mode
  protege et ne peuvent etre executees qu'au niveau de privilege
  IOPL ou superieur.

  4.9 Entrees/sorties
  --------------------

  IN   acc, port         Lire un port d'E/S dans l'accumulateur
                         IN AL, imm8   (port 0-255, codage compact)
                         IN AL, DX     (port 0-65535, via DX)
                         IN AX, DX     (lecture 16 bits)
                         IN EAX, DX    (lecture 32 bits, nouveau 80386)

  OUT  port, acc         Ecrire l'accumulateur vers un port d'E/S
                         OUT imm8, AL  (port 0-255)
                         OUT DX, AL    (port 0-65535, via DX)
                         OUT DX, AX    (ecriture 16 bits)
                         OUT DX, EAX   (ecriture 32 bits, nouveau 80386)

  Note : Les instructions IN/OUT sont privilegiees en mode protege.
  L'acces est controle par le bitmap d'E/S dans le TSS et le
  niveau IOPL dans EFLAGS.

  4.10 Instructions systeme et mode protege
  -------------------------------------------

  4.10.1 Registres de controle et descripteurs

  LGDT mem               Charger le registre GDTR (limite + base)
  SGDT mem               Stocker GDTR en memoire
  LIDT mem               Charger le registre IDTR (limite + base)
  SIDT mem               Stocker IDTR en memoire
  LLDT r/m16             Charger le registre LDTR (selecteur)
  SLDT r/m16             Stocker LDTR
  LTR  r/m16             Charger le registre TR (Task Register)
  STR  r/m16             Stocker TR
  LMSW r/m16             Charger le mot d'etat machine (bits bas de CR0)
  SMSW r/m16             Stocker le mot d'etat machine

  4.10.2 Registres de controle (nouveau 80386)

  MOV  CRn, reg32        Charger un registre de controle (CR0-CR3)
  MOV  reg32, CRn        Lire un registre de controle
  MOV  DRn, reg32        Charger un registre de debug (DR0-DR7)
  MOV  reg32, DRn        Lire un registre de debug
  MOV  TRn, reg32        Charger un registre de test (TR6-TR7)
  MOV  reg32, TRn        Lire un registre de test

  4.10.3 Verification et protection

  ARPL r/m16, reg16      Ajuster le champ RPL d'un selecteur
  LAR  reg, r/m16        Charger les droits d'acces d'un descripteur
  LSL  reg, r/m16        Charger la limite d'un segment
  VERR r/m16             Verifier qu'un segment est lisible (ZF=1 si oui)
  VERW r/m16             Verifier qu'un segment est inscriptible

  4.10.4 Commutation de taches et divers

  CLTS                   Effacer le drapeau TS dans CR0
                         (utilise apres une commutation de tache
                          pour eviter les exceptions #NM)

  HLT                    Arreter le processeur jusqu'a une interruption
                         ou un signal RESET

  WAIT / FWAIT           Attendre que le coprocesseur ait termine
  NOP                    Pas d'operation (XCHG EAX, EAX, codage 90h)
  LOCK                   Prefixe de verrouillage du bus (pour les
                         operations atomiques en multiprocesseur)

  4.11 Instructions nouvelles du 80386
  --------------------------------------

  Le 80386 ajoute les instructions suivantes par rapport au 80286 :

  Extensions 32 bits de toutes les instructions existantes :
    - Tous les registres peuvent etre utilises en 32 bits

    - Les sauts conditionnels acceptent rel32

  Nouvelles instructions :

  Instruction            Description

  BT    r/m, reg/imm     Test de bit
  BTS   r/m, reg/imm     Test et mise a 1
  BTR   r/m, reg/imm     Test et mise a 0
  BTC   r/m, reg/imm     Test et complement
  BSF   reg, r/m         Recherche de bit vers l'avant
  BSR   reg, r/m         Recherche de bit vers l'arriere
  SETcc r/m8             Positionnement conditionnel d'octet
  JECXZ cible            Saut si ECX = 0
  LFS   reg, mem         Charger pointeur avec FS
  LGS   reg, mem         Charger pointeur avec GS
  LSS   reg, mem         Charger pointeur avec SS
  MOV   CRn, reg32       Acces aux registres de controle
  MOV   DRn, reg32       Acces aux registres de debug
  MOV   TRn, reg32       Acces aux registres de test
  MOVSD                  MOVS en 32 bits (double mot)

  SCASD                  SCAS en 32 bits
  LODSD                  LODS en 32 bits

==============================================================================
5. CODAGE DES INSTRUCTIONS (FORMAT BINAIRE)
==============================================================================

  5.1 Vue d'ensemble du format d'instruction
  -------------------------------------------

  Chaque instruction du 80386 est codee en binaire sous la forme d'une
  sequence de 1 a 15 octets. Le format general est :

  +----------+--------+----------+-------+--------------+----------+
  | Prefixes | Opcode | Mod R/M  |  SIB  | Deplacement  | Immediat |
  | 0-4 oct  | 1-2 o  | 0-1 oct  | 0-1 o | 0,1 ou 4 oct | 0,1,2,4o |
  +----------+--------+----------+-------+--------------+----------+

  Chaque champ est optionnel sauf l'opcode. La presence des champs
  Mod R/M, SIB, deplacement et immediat depend de l'instruction et
  de ses operandes.

  Taille maximale d'une instruction : 15 octets. Si une combinaison
  de prefixes et d'operandes depasse cette limite, le processeur
  genere une exception #GP (General Protection Fault).

  5.2 Prefixes d'instruction
  --------------------------

  Les prefixes sont des octets optionnels places avant l'opcode.
  Ils modifient le comportement de l'instruction. Il existe quatre
  groupes de prefixes ; au maximum un prefixe de chaque groupe peut
  etre utilise simultanement.

  5.2.1 Groupe 1 : Prefixes de verrouillage et de repetition

  F0h  LOCK       Verrouillage du bus (operations atomiques)
                   Valide uniquement avec : ADD, ADC, AND, BTC, BTR,
                   BTS, CMPXCHG, DEC, INC, NEG, NOT, OR, SBB, SUB,
                   XADD, XCHG, XOR (avec operande memoire)

  F2h  REPNE      Repetition tant que ZF=0 et ECX<>0
       REPNZ      (pour CMPS, SCAS)

  F3h  REP        Repetition tant que ECX<>0
       REPE       (pour MOVS, STOS, INS, OUTS)
       REPZ       Repetition tant que ZF=1 et ECX<>0
                   (pour CMPS, SCAS)

  5.2.2 Groupe 2 : Prefixes de remplacement de segment

  2Eh  CS:        Utiliser le segment CS
  36h  SS:        Utiliser le segment SS
  3Eh  DS:        Utiliser le segment DS
  26h  ES:        Utiliser le segment ES
  64h  FS:        Utiliser le segment FS (nouveau 80386)
  65h  GS:        Utiliser le segment GS (nouveau 80386)

  En l'absence de prefixe, le segment par defaut est :
    - SS pour les acces via EBP ou ESP (pile)
    - DS pour tous les autres acces memoire
    - CS pour les lectures d'instructions (fetch)
    - ES pour la destination des operations sur les chaines

  5.2.3 Groupe 3 : Prefixe de taille d'operande

  66h  Bascule la taille d'operande par defaut :
       - En mode 32 bits (mode protege avec D=1) :
         les operandes deviennent 16 bits
       - En mode 16 bits (mode reel ou D=0) :
         les operandes deviennent 32 bits

  Exemple :
    En mode 32 bits, MOV EAX, EBX est l'instruction par defaut.
    Avec le prefixe 66h, elle devient MOV AX, BX.

  5.2.4 Groupe 4 : Prefixe de taille d'adresse

  67h  Bascule la taille d'adresse par defaut :
       - En mode 32 bits : les adresses deviennent 16 bits
         (modes d'adressage du 8086)
       - En mode 16 bits : les adresses deviennent 32 bits
         (modes d'adressage etendus du 80386 avec SIB)

  5.3 Opcode (code operation)
  ----------------------------

  L'opcode identifie l'instruction a executer. Il fait 1 ou 2 octets.

  5.3.1 Opcodes sur 1 octet

  La plupart des instructions courantes utilisent un opcode sur 1 octet.
  Certains bits de l'opcode encodent des informations supplementaires :

  Bit 0 (w) : Taille de l'operande
    w=0 : operande 8 bits
    w=1 : operande 16 ou 32 bits (selon le mode et le prefixe 66h)

  Bit 1 (d) : Direction (pour les instructions a deux operandes)
    d=0 : r/m <- reg  (le registre est la source)
    d=1 : reg <- r/m  (le registre est la destination)

  Exemple pour MOV :
    88h = MOV r/m8, reg8        (d=0, w=0)
    89h = MOV r/m16/32, reg16/32 (d=0, w=1)
    8Ah = MOV reg8, r/m8        (d=1, w=0)
    8Bh = MOV reg16/32, r/m16/32 (d=1, w=1)

  Bit 0-2 (reg) : Certaines instructions a 1 octet encodent le
  registre directement dans l'opcode :
    PUSH reg  : 50h + reg   (50h=EAX, 51h=ECX, ..., 57h=EDI)
    POP reg   : 58h + reg
    INC reg   : 40h + reg
    DEC reg   : 48h + reg
    XCHG EAX, reg : 90h + reg  (90h=NOP car XCHG EAX,EAX)
    MOV reg, imm  : B8h + reg (32 bits) ou B0h + reg (8 bits)

  5.3.2 Opcodes sur 2 octets (prefixe 0Fh)

  Les instructions ajoutees a partir du 80386 utilisent souvent
  un opcode de 2 octets commencant par 0Fh :

  0Fh + opcode2    Instruction
  ---------------  ------------------------------------------
  0F 80..0F 8F     Jcc rel32 (sauts conditionnels etendus)
  0F 90..0F 9F     SETcc r/m8
  0F A3             BT r/m, reg
  0F A4             SHLD r/m, reg, imm8
  0F A5             SHLD r/m, reg, CL
  0F AB             BTS r/m, reg
  0F AC             SHRD r/m, reg, imm8
  0F AD             SHRD r/m, reg, CL
  0F AF             IMUL reg, r/m
  0F B3             BTR r/m, reg
  0F B6             MOVZX reg, r/m8
  0F B7             MOVZX reg, r/m16
  0F BB             BTC r/m, reg
  0F BC             BSF reg, r/m
  0F BD             BSR reg, r/m
  0F BE             MOVSX reg, r/m8
  0F BF             MOVSX reg, r/m16
  0F 20..0F 23     MOV CRn/DRn (registres de controle/debug)

  5.3.3 Extensions d'opcode dans le champ reg

  Pour certains opcodes, le champ reg de l'octet Mod R/M ne designe
  pas un registre mais sert d'extension d'opcode. On parle alors de
  "groupe d'instructions". Exemples :

  Opcode 80h-83h (operations arithmetiques/logiques immediates) :
    reg=0 : ADD r/m, imm
    reg=1 : OR  r/m, imm
    reg=2 : ADC r/m, imm
    reg=3 : SBB r/m, imm
    reg=4 : AND r/m, imm
    reg=5 : SUB r/m, imm
    reg=6 : XOR r/m, imm
    reg=7 : CMP r/m, imm

  Opcode FEh/FFh (operations unaires) :
    reg=0 : INC r/m
    reg=1 : DEC r/m
    reg=2 : CALL r/m (indirect, proche)
    reg=3 : CALL mem (indirect, lointain)
    reg=4 : JMP r/m (indirect, proche)
    reg=5 : JMP mem (indirect, lointain)
    reg=6 : PUSH r/m

  Opcode F6h/F7h (operations unaires arithmetiques) :
    reg=0 : TEST r/m, imm
    reg=2 : NOT r/m
    reg=3 : NEG r/m
    reg=4 : MUL r/m
    reg=5 : IMUL r/m
    reg=6 : DIV r/m
    reg=7 : IDIV r/m

  Opcode D0h-D3h (decalages et rotations) :
    reg=0 : ROL        reg=4 : SHL/SAL
    reg=1 : ROR        reg=5 : SHR
    reg=2 : RCL        reg=7 : SAR
    reg=3 : RCR

  5.4 Octet Mod R/M
  ------------------

  L'octet Mod R/M (ModRegR/M) specifie les operandes de l'instruction.
  Il est present des qu'une instruction utilise un operande registre
  ou memoire via le format general reg, r/m.

  Structure (8 bits) :

    7  6  5  4  3  2  1  0
  +-----+--------+--------+
  | Mod |  Reg   |  R/M   |
  +-----+--------+--------+
    2b     3b       3b

  Mod (bits 7-6) : Mode d'adressage
    00 : Acces memoire, pas de deplacement
         (sauf R/M=101 : deplacement 32 bits seul, [disp32])
    01 : Acces memoire + deplacement 8 bits (signe-etendu)
    10 : Acces memoire + deplacement 32 bits
    11 : Operande registre (pas d'acces memoire)

  Reg (bits 5-3) : Numero du registre (ou extension d'opcode)
    000=EAX/AL    001=ECX/CL    010=EDX/DL    011=EBX/BL
    100=ESP/AH    101=EBP/CH    110=ESI/DH    111=EDI/BH

    Pour les registres 16 bits : AX, CX, DX, BX, SP, BP, SI, DI
    Pour les registres de segment : ES, CS, SS, DS, FS, GS

  R/M (bits 2-0) : Registre ou mode memoire
    Si Mod=11 : meme codage que Reg (operande registre)
    Si Mod<>11 :
      000 = [EAX]      001 = [ECX]      010 = [EDX]
      011 = [EBX]      100 = [SIB]      101 = [disp32] (si Mod=00)
                                               [EBP+disp] (si Mod=01/10)
      110 = [ESI]      111 = [EDI]

  Note : R/M=100 (ESP) indique toujours la presence d'un octet SIB.
  R/M=101 avec Mod=00 signifie un deplacement 32 bits absolu sans
  registre de base.

  Tableau complet du Mod R/M en mode 32 bits :

  Mod  R/M   Mode effectif                Deplacement
  ---  ---   --------------------------   -----------
  00   000   [EAX]                        aucun
  00   001   [ECX]                        aucun
  00   010   [EDX]                        aucun
  00   011   [EBX]                        aucun
  00   100   [SIB]                        aucun
  00   101   [disp32]                     32 bits
  00   110   [ESI]                        aucun
  00   111   [EDI]                        aucun
  01   000   [EAX + disp8]               8 bits
  01   001   [ECX + disp8]               8 bits
  01   010   [EDX + disp8]               8 bits
  01   011   [EBX + disp8]               8 bits
  01   100   [SIB + disp8]               8 bits
  01   101   [EBP + disp8]               8 bits
  01   110   [ESI + disp8]               8 bits
  01   111   [EDI + disp8]               8 bits
  10   000   [EAX + disp32]              32 bits
  10   001   [ECX + disp32]              32 bits
  10   010   [EDX + disp32]              32 bits
  10   011   [EBX + disp32]              32 bits
  10   100   [SIB + disp32]              32 bits
  10   101   [EBP + disp32]              32 bits
  10   110   [ESI + disp32]              32 bits
  10   111   [EDI + disp32]              32 bits
  11   000   EAX / AL                    -
  11   001   ECX / CL                    -
  11   010   EDX / DL                    -
  11   011   EBX / BL                    -
  11   100   ESP / AH                    -
  11   101   EBP / CH                    -
  11   110   ESI / DH                    -
  11   111   EDI / BH                    -

  5.5 Octet SIB (Scale-Index-Base)
  ---------------------------------

  L'octet SIB est present lorsque le champ R/M du Mod R/M vaut 100
  (binaire). Il permet les modes d'adressage avec echelle
  (scaled index) qui sont nouveaux sur le 80386.

  Structure (8 bits) :

    7  6  5  4  3  2  1  0
  +-----+--------+--------+
  |Scale| Index  |  Base  |
  +-----+--------+--------+
    2b     3b       3b

  Scale (bits 7-6) : Facteur d'echelle pour le registre d'index
    00 = x1
    01 = x2
    10 = x4
    11 = x8

  Index (bits 5-3) : Registre d'index
    000=EAX    001=ECX    010=EDX    011=EBX
    100=(aucun index, echelle ignoree)
    101=EBP    110=ESI    111=EDI

    Note : Index=100 signifie "pas d'index". Cela permet d'encoder
    [base] seul quand R/M=100 serait normalement [SIB].
    ESP ne peut jamais etre utilise comme registre d'index.

  Base (bits 2-0) : Registre de base
    000=EAX    001=ECX    010=EDX    011=EBX
    100=ESP    101=EBP (ou disp32 si Mod=00)
    110=ESI    111=EDI

    Note : Base=101 avec Mod=00 signifie pas de registre de base,
    seulement [Index*Scale + disp32]. Avec Mod=01 ou 10, la base
    est EBP normalement.

  Formule d'adresse avec SIB :

    Si Mod=00 et Base=101 :
      EA = Index * Scale + disp32
    Si Mod=00 et Base<>101 :
      EA = Base + Index * Scale
    Si Mod=01 :
      EA = Base + Index * Scale + disp8 (signe-etendu)
    Si Mod=10 :
      EA = Base + Index * Scale + disp32

  Exemples de SIB :

    [EAX + ECX*4]     : Scale=10, Index=001, Base=000
                         SIB = 10_001_000 = 88h

    [EBP + ESI*2 + 8] : Scale=01, Index=110, Base=101, Mod=01
                         SIB = 01_110_101 = 75h, disp8 = 08h

    [EDX*8 + 1000h]   : Scale=11, Index=010, Base=101, Mod=00
                         SIB = 11_010_101 = D5h, disp32 = 00001000h

    [ESP]              : R/M=100, Scale=00, Index=100, Base=100
                         SIB = 00_100_100 = 24h

  5.6 Deplacement
  ----------------

  Le deplacement est une valeur entiere signee ajoutee a l'adresse
  de base calculee par Mod R/M et SIB. Sa taille depend du champ Mod :

  Mod=00 : Pas de deplacement (sauf cas speciaux R/M=101 ou Base=101)
  Mod=01 : Deplacement de 8 bits, etendu en signe sur 32 bits
  Mod=10 : Deplacement de 32 bits

  Cas special (Mod=00, R/M=101) :
    Deplacement absolu de 32 bits, sans registre de base.
    Utilise pour l'adressage direct : MOV EAX, [adresse_absolue]

  L'assembleur choisit automatiquement la taille optimale :
    - Si le deplacement tient dans [-128, +127], il utilise disp8
    - Sinon, il utilise disp32
    - Le programmeur peut forcer la taille avec BYTE PTR ou DWORD PTR

  5.7 Valeur immediate
  ---------------------

  La valeur immediate est une constante codee directement dans
  l'instruction. Sa taille depend de l'instruction et du bit w :

  w=0 : Immediat 8 bits (instructions sur octets)
  w=1 : Immediat 16 ou 32 bits (selon la taille d'operande)

  Cas special (opcode 83h) :
    Immediat de 8 bits etendu en signe sur 16 ou 32 bits.
    Cela permet de coder ADD EAX, 5 sur 3 octets au lieu de 6 :
      83 C0 05  au lieu de  05 00000005

  L'immediat est toujours encode en format little-endian :
    L'octet de poids faible est place en premier.
    Exemple : MOV EAX, 12345678h est code : B8 78 56 34 12

  5.8 Exemples de codage complets
  --------------------------------

  Voici des exemples detailles de codage d'instructions courantes.
  Chaque octet est donne en hexadecimal.

  5.8.1 NOP

  Instruction : NOP
  Codage      : 90
  Explication : XCHG EAX, EAX (opcode 90h = 90h + 0 pour EAX)

  5.8.2 MOV EAX, EBX (registre a registre)

  Instruction : MOV EAX, EBX
  Codage      : 89 D8
  Explication :
    89h = MOV r/m32, reg32 (opcode d=0, w=1)
    D8h = Mod R/M = 11_011_000
      Mod=11 (registre)
      Reg=011 (EBX = source)
      R/M=000 (EAX = destination)

  Variante   : 8B C3  (opcode 8Bh avec d=1 : reg=EAX dest, R/M=EBX src)

  5.8.3 MOV EAX, [EBX] (registre indirect)

  Instruction : MOV EAX, [EBX]
  Codage      : 8B 03
  Explication :
    8Bh = MOV reg32, r/m32 (d=1, w=1)
    03h = Mod R/M = 00_000_011
      Mod=00 (memoire, pas de deplacement)
      Reg=000 (EAX = destination)
      R/M=011 ([EBX])

  5.8.4 MOV EAX, [EBP+8] (base + deplacement)

  Instruction : MOV EAX, [EBP+8]
  Codage      : 8B 45 08
  Explication :
    8Bh = MOV reg32, r/m32 (d=1, w=1)
    45h = Mod R/M = 01_000_101
      Mod=01 (memoire + disp8)
      Reg=000 (EAX = destination)
      R/M=101 ([EBP + disp8])
    08h = deplacement = +8

  5.8.5 MOV EAX, [ESI+ECX*4] (base + index scale)

  Instruction : MOV EAX, [ESI+ECX*4]
  Codage      : 8B 04 8E
  Explication :
    8Bh = MOV reg32, r/m32
    04h = Mod R/M = 00_000_100
      Mod=00 (memoire, pas de deplacement)
      Reg=000 (EAX = destination)
      R/M=100 (SIB suit)
    8Eh = SIB = 10_001_110
      Scale=10 (x4)
      Index=001 (ECX)
      Base=110 (ESI)

  5.8.6 MOV EAX, [EBX+ESI*2+100h] (base + index scale + disp32)

  Instruction : MOV EAX, [EBX+ESI*2+100h]
  Codage      : 8B 84 73 00 01 00 00
  Explication :
    8Bh = MOV reg32, r/m32
    84h = Mod R/M = 10_000_100
      Mod=10 (memoire + disp32)
      Reg=000 (EAX = destination)
      R/M=100 (SIB suit)
    73h = SIB = 01_110_011
      Scale=01 (x2)
      Index=110 (ESI)
      Base=011 (EBX)
    00 01 00 00 = deplacement 00000100h = 256

  5.8.7 ADD EAX, 42 (immediat optimise)

  Instruction : ADD EAX, 42
  Codage      : 83 C0 2A
  Explication :
    83h = ADD r/m32, imm8 (extension signe)
    C0h = Mod R/M = 11_000_000
      Mod=11 (registre)
      Reg=000 (extension opcode = ADD)
      R/M=000 (EAX)
    2Ah = 42 en hexadecimal

  Codage alternatif (non optimise) :
    05 2A 00 00 00  (ADD EAX, imm32 : opcode special pour EAX)

  5.8.8 PUSH 0FFFFFFFFh (immediat 32 bits)

  Instruction : PUSH -1
  Codage      : 6A FF
  Explication :
    6Ah = PUSH imm8 (etendu en signe)
    FFh = -1 en complement a deux (8 bits -> 32 bits)

  5.8.9 JE label (saut conditionnel rel32)

  Instruction : JE label  (avec label a +256 octets)
  Codage      : 0F 84 00 01 00 00
  Explication :
    0Fh 84h = JE rel32 (opcode sur 2 octets)
    00 01 00 00 = deplacement relatif = +256

  5.8.10 MOV AX, [BX+SI] (mode 16 bits avec prefixe)

  Instruction : MOV AX, [BX+SI]  (en mode 32 bits)
  Codage      : 67 66 8B 00
  Explication :
    67h = prefixe taille d'adresse (-> adresses 16 bits)
    66h = prefixe taille d'operande (-> operandes 16 bits)
    8Bh = MOV reg, r/m
    00h = Mod R/M 16 bits = 00_000_000 = [BX+SI] sans deplacement

  5.8.11 LEA EAX, [EBX+ECX*8+10h]

  Instruction : LEA EAX, [EBX+ECX*8+10h]
  Codage      : 8D 44 CB 10
  Explication :
    8Dh = LEA reg32, mem
    44h = Mod R/M = 01_000_100
      Mod=01 (memoire + disp8)
      Reg=000 (EAX)
      R/M=100 (SIB suit)
    CBh = SIB = 11_001_011
      Scale=11 (x8)
      Index=001 (ECX)
      Base=011 (EBX)
    10h = deplacement = +16

  5.8.12 LOCK XADD [EDI], EAX

  Instruction : LOCK XADD [EDI], EAX
  Codage      : F0 0F C1 07
  Explication :
    F0h = prefixe LOCK (operation atomique)
    0Fh C1h = XADD r/m32, reg32
    07h = Mod R/M = 00_000_111
      Mod=00 (memoire)
      Reg=000 (EAX)
      R/M=111 ([EDI])

  5.9 Mod R/M en mode 16 bits (adressage du 8086)
  -------------------------------------------------

  En mode 16 bits (mode reel ou avec prefixe 67h en mode 32 bits),
  l'octet Mod R/M utilise un codage different, herite du 8086.
  Il n'y a pas d'octet SIB et les modes d'adressage sont limites.

  Mod  R/M   Mode effectif 16 bits
  ---  ---   -------------------------
  00   000   [BX + SI]
  00   001   [BX + DI]
  00   010   [BP + SI]
  00   011   [BP + DI]
  00   100   [SI]
  00   101   [DI]
  00   110   [disp16]  (adressage direct)
  00   111   [BX]
  01   000   [BX + SI + disp8]
  01   001   [BX + DI + disp8]
  01   010   [BP + SI + disp8]
  01   011   [BP + DI + disp8]
  01   100   [SI + disp8]
  01   101   [DI + disp8]
  01   110   [BP + disp8]
  01   111   [BX + disp8]
  10   000   [BX + SI + disp16]
  10   001   [BX + DI + disp16]
  10   010   [BP + SI + disp16]
  10   011   [BP + DI + disp16]
  10   100   [SI + disp16]
  10   101   [DI + disp16]
  10   110   [BP + disp16]
  10   111   [BX + disp16]
  11   nnn   Registre (meme codage que mode 32 bits)

  Differences avec le mode 32 bits :
    - Pas d'octet SIB
    - Pas de facteur d'echelle
    - Seulement BX et BP comme registres de base
    - Seulement SI et DI comme registres d'index
    - Deplacements de 0, 8 ou 16 bits (au lieu de 0, 8 ou 32)

  5.10 Resume du processus de decodage
  --------------------------------------

  Le processeur decode une instruction selon l'algorithme suivant :

  1. Lire les prefixes (F0h, F2h, F3h, 2Eh, 36h, 3Eh, 26h,
     64h, 65h, 66h, 67h). Arreter des qu'un octet n'est pas
     un prefixe reconnu.

  2. Lire l'opcode (1 octet). Si l'opcode est 0Fh, lire un
     second octet pour former un opcode de 2 octets.

  3. Si l'instruction requiert un Mod R/M, lire l'octet Mod R/M.
     Determiner si un SIB est necessaire (R/M=100 et Mod<>11).

  4. Si un SIB est necessaire, lire l'octet SIB.

  5. Lire le deplacement (0, 1 ou 4 octets selon Mod et les
     cas speciaux R/M=101/Base=101).

  6. Lire la valeur immediate si l'instruction en requiert une
     (0, 1, 2 ou 4 octets).

  7. Executer l'instruction.

  Longueur totale de l'instruction :
    = prefixes + opcode + [Mod R/M] + [SIB] + [depl.] + [imm.]
    Minimum : 1 octet (NOP = 90h)
    Maximum : 15 octets

  Tableau recapitulatif des tailles de chaque champ :

  Champ            Taille (octets)   Condition
  ---------------  ----------------  ---------------------------
  Prefixes         0 a 4             Optionnel
  Opcode           1 ou 2            Toujours present
  Mod R/M          0 ou 1            Si operande r/m requis
  SIB              0 ou 1            Si R/M=100 et Mod<>11
  Deplacement      0, 1 ou 4         Selon Mod (0, 8 ou 32 bits)
  Immediat         0, 1, 2 ou 4      Si operande immediat requis

==============================================================================
6. MODE PROTEGE ET DESCRIPTEURS
==============================================================================

  6.1 Vue d'ensemble du mode protege
  ------------------------------------

  Le 80386 peut fonctionner dans trois modes principaux :

  1. Mode reel (Real Mode)
     - Compatible 8086. Segments de 64 Ko max.
     - Adressage : segment << 4 + offset = adresse physique (20 bits).
     - Pas de protection memoire ni de privileges.
     - C'est le mode au demarrage (reset).

  2. Mode protege (Protected Mode)
     - Adressage 32 bits (4 Go d'espace lineaire).
     - Segments decrits par des descripteurs dans des tables.
     - Protection memoire par niveaux de privilege (rings 0 a 3).
     - Pagination optionnelle (adresses lineaires -> physiques).
     - Support du multitache materiel (TSS).

  3. Mode virtuel 8086 (Virtual 8086 Mode, V86)
     - Sous-mode du mode protege.
     - Permet d'executer des programmes 8086 dans un environnement
       protege, sous le controle d'un moniteur en ring 0.
     - Active par le bit VM (bit 17) du registre EFLAGS.

  Le mode protege est le mode de fonctionnement principal du 80386
  pour les systemes d'exploitation modernes. Il offre :
    - Un espace d'adressage de 4 Go par processus
    - L'isolation des processus par la segmentation et la pagination
    - Quatre niveaux de privilege pour le controle d'acces
    - Un mecanisme de commutation de taches (task switching)
    - La gestion des interruptions et exceptions via l'IDT

  6.2 Descripteurs de segments
  ----------------------------

  En mode protege, chaque segment de memoire est decrit par un
  descripteur de 8 octets (64 bits). Les descripteurs contiennent
  toutes les informations necessaires pour definir un segment :
  son adresse de base, sa limite, son type et ses droits d'acces.

  6.2.1 Format general d'un descripteur de segment

  Un descripteur occupe 8 octets consecutifs en memoire.
  La disposition est la suivante (les octets sont numerotes 0 a 7) :

  Octet 7     Octet 6     Octet 5     Octet 4
  +--------+  +--------+  +--------+  +--------+
  |Base    |  |G|D|0|AVL|  |P|DPL|S|  |Base    |
  |[31:24] |  |Lim[19:16]| |  Type |  |[23:16] |
  +--------+  +--------+  +--------+  +--------+

  Octet 3     Octet 2     Octet 1     Octet 0
  +--------+  +--------+  +--------+  +--------+
  |  Base  |  |  Base  |  | Limite |  | Limite |
  | [15:8] |  | [7:0]  |  | [15:8] |  | [7:0]  |
  +--------+  +--------+  +--------+  +--------+

  Champs du descripteur :

  Base (32 bits)   : Adresse lineaire de debut du segment.
                     Repartie dans les octets 2-3 (bits 0-15),
                     octet 4 (bits 16-23) et octet 7 (bits 24-31).

  Limite (20 bits) : Taille du segment - 1.
                     Repartie dans les octets 0-1 (bits 0-15) et
                     les bits 0-3 de l'octet 6 (bits 16-19).
                     Avec G=0 : en octets (max 1 Mo).
                     Avec G=1 : en pages de 4 Ko (max 4 Go).

  G (bit 7, octet 6) : Granularite
    0 = Limite en octets (resolution a l'octet)
    1 = Limite en pages de 4096 octets

  D/B (bit 6, octet 6) : Taille par defaut
    Pour un segment de code :
      0 = Operandes et adresses 16 bits par defaut
      1 = Operandes et adresses 32 bits par defaut
    Pour un segment de pile (SS) :
      0 = SP 16 bits
      1 = ESP 32 bits

  AVL (bit 4, octet 6) : Disponible pour le systeme d'exploitation.
    Le processeur ne l'utilise pas.

  P (bit 7, octet 5) : Present
    1 = Le segment est present en memoire
    0 = Le segment n'est pas present (genere #NP si on y accede)

  DPL (bits 6-5, octet 5) : Descriptor Privilege Level
    Niveau de privilege du segment (0 a 3).
    0 = plus privilegie (noyau), 3 = moins privilegie (utilisateur).

  S (bit 4, octet 5) : Type de descripteur
    1 = Descripteur de segment (code ou donnees)
    0 = Descripteur systeme (TSS, LDT, porte)

  Type (bits 3-0, octet 5) : Sous-type du segment
    Depend du bit S. Voir ci-dessous.

  6.2.2 Types de descripteurs de segments (S=1)

  Pour les segments de donnees (bit 3 = 0) :

  Type  E  W  A  Description
  ----  -  -  -  ------------------------------------
  0000  0  0  0  Donnees, lecture seule
  0001  0  0  1  Donnees, lecture seule, accede
  0010  0  1  0  Donnees, lecture/ecriture
  0011  0  1  1  Donnees, lecture/ecriture, accede
  0100  1  0  0  Donnees, expand-down, lecture seule
  0101  1  0  1  Donnees, expand-down, lecture seule, accede
  0110  1  1  0  Donnees, expand-down, lecture/ecriture
  0111  1  1  1  Donnees, expand-down, lecture/ecriture, accede

  E (Expand-Down) : Utilise pour les segments de pile.
    Le segment grandit vers les adresses basses.
    La limite definit l'adresse la plus basse accessible.

  W (Writable) : Indique si le segment est inscriptible.
    Un segment de donnees sans W ne peut etre que lu.

  A (Accessed) : Mis a 1 par le processeur a chaque acces.
    Utile pour le systeme d'exploitation (LRU, swapping).

  Pour les segments de code (bit 3 = 1) :

  Type  C  R  A  Description
  ----  -  -  -  ------------------------------------
  1000  0  0  0  Code, execute seul
  1001  0  0  1  Code, execute seul, accede
  1010  0  1  0  Code, execute/lecture
  1011  0  1  1  Code, execute/lecture, accede
  1100  1  0  0  Code, conforming, execute seul
  1101  1  0  1  Code, conforming, execute seul, accede
  1110  1  1  0  Code, conforming, execute/lecture
  1111  1  1  1  Code, conforming, execute/lecture, accede

  C (Conforming) : Le segment de code peut etre appele depuis un
    niveau de privilege inferieur sans changement de CPL.
    Utile pour les bibliotheques partagees du systeme.

  R (Readable) : Le segment de code peut etre lu (en plus d'etre
    execute). Necessaire pour lire des constantes dans le code.

  6.2.3 Types de descripteurs systeme (S=0)

  Type   Description
  ----   --------------------------------------------------
  0000   Reserve
  0001   TSS 16 bits (disponible)
  0010   LDT (Local Descriptor Table)
  0011   TSS 16 bits (occupe/busy)
  0100   Porte d'appel 16 bits (Call Gate)
  0101   Porte de tache (Task Gate)
  0110   Porte d'interruption 16 bits (Interrupt Gate)
  0111   Porte de trappe 16 bits (Trap Gate)
  1000   Reserve
  1001   TSS 32 bits (disponible)       (nouveau 80386)
  1010   Reserve
  1011   TSS 32 bits (occupe/busy)       (nouveau 80386)
  1100   Porte d'appel 32 bits           (nouveau 80386)
  1101   Reserve
  1110   Porte d'interruption 32 bits    (nouveau 80386)
  1111   Porte de trappe 32 bits         (nouveau 80386)

  6.3 Tables de descripteurs
  --------------------------

  Les descripteurs sont regroupes dans des tables en memoire.
  Le 80386 utilise trois types de tables :

  6.3.1 GDT (Global Descriptor Table)

  Table de descripteurs globale, partagee par toutes les taches.
  Elle contient les descripteurs accessibles a l'ensemble du systeme :
    - Segments de code et de donnees du noyau
    - Segments partages entre les taches
    - Descripteurs de TSS pour chaque tache
    - Descripteurs de LDT pour chaque tache

  La GDT est localisee par le registre GDTR :
    - Base  : adresse lineaire 32 bits de la table
    - Limite : taille de la table en octets - 1

  L'entree 0 de la GDT (descripteur nul) est reservee et ne peut
  pas etre utilisee. Un selecteur qui pointe vers l'entree 0
  est appele "selecteur nul" et genere #GP s'il est charge dans
  CS ou SS.

  Instructions associees :
    LGDT mem48  : Charge GDTR depuis la memoire (ring 0 seulement)
    SGDT mem48  : Sauvegarde GDTR en memoire

  6.3.2 LDT (Local Descriptor Table)

  Table de descripteurs locale, propre a chaque tache.
  Elle contient les descripteurs prives d'une tache :
    - Segments de code et de donnees de l'application
    - Segments de pile de l'application

  La LDT est elle-meme decrite par un descripteur dans la GDT.
  Le registre LDTR contient le selecteur de ce descripteur.

  Chaque tache peut avoir sa propre LDT, ce qui permet l'isolation
  des espaces d'adressage entre les taches.

  Instructions associees :
    LLDT reg16  : Charge LDTR depuis un selecteur (ring 0)
    SLDT reg16  : Sauvegarde le selecteur de LDTR

  6.3.3 IDT (Interrupt Descriptor Table)

  Table des descripteurs d'interruptions. Elle contient les
  descripteurs de portes (interrupt gates, trap gates, task gates)
  pour les 256 vecteurs d'interruption possibles.

  L'IDT est localisee par le registre IDTR :
    - Base  : adresse lineaire 32 bits
    - Limite : taille en octets - 1 (max 256 * 8 = 2048 octets)

  Instructions associees :
    LIDT mem48  : Charge IDTR depuis la memoire (ring 0)
    SIDT mem48  : Sauvegarde IDTR en memoire

  6.4 Selecteurs de segments
  --------------------------

  Un selecteur de segment est une valeur de 16 bits chargee dans un
  registre de segment (CS, DS, ES, FS, GS, SS). Il identifie un
  descripteur dans une table de descripteurs.

  Format d'un selecteur :

    15                  3  2  1  0
  +---------------------+--+-----+
  |       Index         |TI| RPL |
  +---------------------+--+-----+
         13 bits         1b   2b

  Index (bits 15-3) : Numero d'entree dans la table de descripteurs.
    Permet d'adresser jusqu'a 8192 descripteurs (2^13).

  TI (bit 2) : Table Indicator
    0 = Le descripteur est dans la GDT
    1 = Le descripteur est dans la LDT

  RPL (bits 1-0) : Requested Privilege Level
    Niveau de privilege demande (0 a 3).
    Utilise dans les verifications de privilege.

  L'adresse du descripteur dans la table est :
    Adresse = Base_de_la_table + Index * 8

  Chaque table peut contenir jusqu'a 8192 descripteurs.
  Avec la GDT et une LDT par tache, on peut definir :
    - 8192 segments globaux (GDT)
    - 8192 segments locaux par tache (LDT)
  soit 16384 segments accessibles par tache, totalisant un espace
  d'adressage logique de 64 To (16384 segments * 4 Go chacun).

  Exemples de selecteurs :

    08h = Index 1, GDT, RPL 0   (premier segment utile de la GDT)
    10h = Index 2, GDT, RPL 0
    23h = Index 4, GDT, RPL 3   (segment utilisateur)
    0Fh = Index 1, LDT, RPL 3

  6.5 Traduction d'adresses en mode protege
  -------------------------------------------

  En mode protege, une adresse logique (selecteur:offset) est
  traduite en adresse lineaire par l'unite de segmentation :

  1. Le selecteur est utilise pour trouver le descripteur dans
     la GDT ou la LDT.
  2. Le processeur verifie les droits d'acces (privilege, type,
     presence, limites).
  3. L'adresse de base du descripteur est ajoutee a l'offset
     pour former l'adresse lineaire.

  Schema :

    Adresse logique     Unite de segmentation     Adresse lineaire
    selecteur:offset -> descripteur -> base+offset -> 32 bits

  Si la pagination est activee (CR0.PG = 1), l'adresse lineaire
  est ensuite traduite en adresse physique par l'unite de
  pagination (voir section 7).

  Si la pagination est desactivee (CR0.PG = 0), l'adresse lineaire
  est egale a l'adresse physique.

  Verifications effectuees lors de l'acces a un segment :

  a) Verification de la limite :
     L'offset ne doit pas depasser la limite du segment.
     Pour G=0 : offset <= limite (en octets)
     Pour G=1 : offset <= limite * 4096 + 4095
     Violation : exception #GP (General Protection Fault)

  b) Verification du type :
     - On ne peut pas executer un segment de donnees
     - On ne peut pas ecrire dans un segment de code
     - On ne peut pas ecrire dans un segment de donnees sans W
     Violation : exception #GP

  c) Verification de la presence :
     Si P=0, le segment n'est pas en memoire.
     Violation : exception #NP (Segment Not Present)

  d) Verification des privileges :
     Voir section 6.6 ci-dessous.

  6.6 Niveaux de privilege
  -------------------------

  Le 80386 implemente un systeme de protection a 4 niveaux de
  privilege, souvent appeles "anneaux" (rings) :

    Ring 0 : Noyau du systeme d'exploitation (plus privilegie)
    Ring 1 : Services du systeme d'exploitation
    Ring 2 : Extensions du systeme
    Ring 3 : Applications utilisateur (moins privilegie)

  En pratique, la plupart des systemes d'exploitation n'utilisent
  que les rings 0 (noyau) et 3 (utilisateur).

  6.6.1 Types de niveaux de privilege

  CPL (Current Privilege Level) :
    Niveau de privilege courant du code en execution.
    Stocke dans les bits RPL du selecteur de CS.
    CPL = CS.RPL a tout moment.

  DPL (Descriptor Privilege Level) :
    Niveau de privilege du segment ou de la porte.
    Stocke dans le descripteur lui-meme.

  RPL (Requested Privilege Level) :
    Niveau de privilege demande, stocke dans le selecteur.
    Peut etre utilise pour restreindre les droits d'acces
    (le niveau effectif est max(CPL, RPL)).

  6.6.2 Regles de verification des privileges

  Pour acceder a un segment de donnees :
    max(CPL, RPL) <= DPL

  Un programme ne peut acceder qu'aux segments de donnees dont
  le DPL est numeriquement superieur ou egal a son niveau effectif.

  Pour transferer le controle a un segment de code :

  a) Code non-conforming (C=0) :
     CPL == DPL  (le niveau doit etre exactement le meme)
     Exception : les portes d'appel permettent les transitions.

  b) Code conforming (C=1) :
     CPL >= DPL  (le code appele peut etre plus privilegie)
     Le CPL ne change pas apres le transfert.

  Pour utiliser une porte d'appel (Call Gate) :
     max(CPL, RPL) <= DPL de la porte
     CPL >= DPL du segment de code cible

  6.6.3 Schema des anneaux de privilege

              +-------------------+
              |      Ring 0       |
              |      Noyau        |
              |  +-------------+  |
              |  |   Ring 1    |  |
              |  |  Services   |  |
              |  | +---------+ |  |
              |  | |  Ring 2 | |  |
              |  | | Ext. OS | |  |
              |  | | +-----+ | |  |
              |  | | |Ring 3| | |  |
              |  | | | Appli| | |  |
              |  | | +-----+ | |  |
              |  | +---------+ |  |
              |  +-------------+  |
              +-------------------+

  Les transitions entre anneaux se font par :
    - Portes d'appel (CALL via call gate) : ring 3 -> ring 0
    - Instructions INT/IRET : via l'IDT
    - Commutation de taches : via TSS
    - Instructions SYSENTER/SYSEXIT (a partir du Pentium II)

  6.7 Portes (Gates)
  -------------------

  Les portes sont des descripteurs systeme (S=0) qui permettent
  des transitions controlees entre niveaux de privilege.

  6.7.1 Porte d'appel (Call Gate)

  Format (8 octets) :

  Octet 7     Octet 6     Octet 5     Octet 4
  +--------+  +--------+  +--------+  +--------+
  | Offset |  | Offset |  |P|DPL|0|  |000|Param|
  | [31:24]|  | [23:16]|  |  1100 |  |  Count |
  +--------+  +--------+  +--------+  +--------+

  Octet 3     Octet 2     Octet 1     Octet 0
  +--------+  +--------+  +--------+  +--------+
  |Selecteur  |Selecteur  | Offset |  | Offset |
  |  [15:8]   |  [7:0]    | [15:8] |  | [7:0]  |
  +--------+  +--------+  +--------+  +--------+

  Selecteur : Selecteur du segment de code cible
  Offset    : Offset du point d'entree dans le segment cible
  DPL       : Niveau de privilege requis pour utiliser la porte
  Param Count : Nombre de parametres (DWords) a copier de la pile
              de l'appelant vers la pile du noyau (0 a 31)
  Type      : 1100 (Call Gate 32 bits)

  Fonctionnement d'un CALL via une porte d'appel :
  1. Le processeur verifie : max(CPL, RPL) <= DPL de la porte
  2. Il verifie : CPL >= DPL du segment de code cible
  3. Si changement de privilege :
     a. Il charge SS et ESP depuis le TSS pour le nouveau ring
     b. Il empile sur la nouvelle pile : SS ancien, ESP ancien,
        les parametres copies, CS ancien, EIP ancien
  4. Il charge CS:EIP depuis la porte (selecteur + offset)
  5. L'execution continue au point d'entree avec le nouveau CPL

  Le retour se fait par RET (ou RETF) qui restaure CS:EIP et
  eventuellement SS:ESP si le privilege change.

  6.7.2 Porte d'interruption (Interrupt Gate)

  Format identique a la porte d'appel, mais avec Type = 1110.

  Difference avec la porte d'appel :
    - Utilisee pour les interruptions (materielles et logicielles)
    - Le drapeau IF est mis a 0 automatiquement (interruptions
      desactivees pendant le handler)
    - Pas de copie de parametres

  Le processeur empile automatiquement sur la pile du handler :
    - EFLAGS, CS, EIP (toujours)
    - SS, ESP (si changement de privilege)
    - Code d'erreur (pour certaines exceptions)

  6.7.3 Porte de trappe (Trap Gate)

  Format identique a la porte d'interruption, mais avec Type = 1111.

  Difference avec la porte d'interruption :
    - Le drapeau IF n'est PAS modifie (les interruptions restent
      dans l'etat ou elles etaient)
    - Utilisee typiquement pour les exceptions (debug, breakpoints)

  6.7.4 Porte de tache (Task Gate)

  Format (8 octets) :

  Octet 5 contient P|DPL|0|0101 (Type = 0101)
  Octets 2-3 contiennent le selecteur du TSS de la tache cible.
  Les autres champs sont reserves.

  Lorsqu'une porte de tache est activee (par CALL, JMP, INT ou
  exception), le processeur effectue une commutation de tache
  complete vers le TSS reference par le selecteur.

  6.8 Segment d'etat de tache (TSS)
  -----------------------------------

  Le TSS (Task State Segment) est une structure de donnees en
  memoire qui stocke l'etat complet d'une tache. Le 80386 utilise
  le TSS pour la commutation de taches materielle.

  6.8.1 Format du TSS 32 bits

  Le TSS 32 bits occupe au minimum 104 octets (68h) :

  Offset  Taille  Contenu
  ------  ------  -------------------------------------------
  00h     WORD    Lien arriere (selecteur du TSS precedent)
  02h     WORD    Reserve
  04h     DWORD   ESP0 (pile pour Ring 0)
  08h     WORD    SS0  (segment de pile pour Ring 0)
  0Ah     WORD    Reserve
  0Ch     DWORD   ESP1 (pile pour Ring 1)
  10h     WORD    SS1
  12h     WORD    Reserve
  14h     DWORD   ESP2 (pile pour Ring 2)
  18h     WORD    SS2
  1Ah     WORD    Reserve
  1Ch     DWORD   CR3 (repertoire de pages)
  20h     DWORD   EIP
  24h     DWORD   EFLAGS
  28h     DWORD   EAX
  2Ch     DWORD   ECX
  30h     DWORD   EDX
  34h     DWORD   EBX
  38h     DWORD   ESP
  3Ch     DWORD   EBP
  40h     DWORD   ESI
  44h     DWORD   EDI
  48h     WORD    ES
  4Ah     WORD    Reserve
  4Ch     WORD    CS
  4Eh     WORD    Reserve
  50h     WORD    SS
  52h     WORD    Reserve
  54h     WORD    DS
  56h     WORD    Reserve
  58h     WORD    FS
  5Ah     WORD    Reserve
  5Ch     WORD    GS
  5Eh     WORD    Reserve
  60h     WORD    Selecteur LDT
  62h     WORD    Reserve
  64h     WORD    Bit T (debug trap) | Reserve
  66h     WORD    Offset de la carte d'E/S (I/O Map Base)

  Apres l'offset 68h, on peut placer la carte de permissions
  d'E/S (I/O Permission Bitmap), terminee par un octet FFh.

  6.8.2 Piles par niveau de privilege

  Le TSS contient trois paires SS:ESP (pour les rings 0, 1 et 2).
  Lorsqu'un appel passe a un niveau de privilege superieur,
  le processeur charge automatiquement la pile correspondante
  depuis le TSS de la tache courante.

  Cela garantit que chaque niveau de privilege dispose de sa
  propre pile, evitant qu'un programme utilisateur puisse
  corrompre la pile du noyau.

  6.8.3 Carte de permissions d'E/S (I/O Permission Bitmap)

  La carte d'E/S est un tableau de bits ou chaque bit correspond
  a un port d'E/S. Elle permet de controler l'acces aux ports
  pour les programmes en ring > IOPL.

    Bit = 0 : L'acces au port est autorise
    Bit = 1 : L'acces au port genere une exception #GP

  L'offset de la carte dans le TSS est donne par le champ
  "I/O Map Base" a l'offset 66h. La carte peut couvrir jusqu'a
  65536 ports (8192 octets).

  Si IOPL >= CPL, tous les acces E/S sont autorises sans
  consulter la carte.

  6.8.4 Commutation de taches

  Une commutation de taches peut etre declenchee par :
    - JMP ou CALL vers un descripteur de TSS dans la GDT
    - JMP ou CALL via une porte de tache (Task Gate)
    - Une interruption ou exception via une porte de tache dans l'IDT
    - IRET lorsque le flag NT (Nested Task) est a 1

  Lors d'une commutation de tache, le processeur :

  1. Sauvegarde l'etat complet de la tache courante dans son TSS
     (EAX-EDI, EIP, EFLAGS, segments, CR3)
  2. Marque le TSS courant comme "busy" (occupe)
  3. Charge le registre TR avec le selecteur du nouveau TSS
  4. Charge l'etat complet depuis le nouveau TSS
  5. Si CALL ou INT : met le flag NT a 1 et sauvegarde le
     selecteur de l'ancien TSS dans le champ "lien arriere"
  6. Reprend l'execution au EIP du nouveau TSS

  Le registre TR (Task Register) contient le selecteur du TSS
  de la tache courante :
    LTR reg16  : Charge TR avec un selecteur (ring 0)
    STR reg16  : Sauvegarde le selecteur de TR

  6.9 Passage en mode protege
  ----------------------------

  La sequence typique pour passer du mode reel au mode protege :

  1. Desactiver les interruptions (CLI)

  2. Activer la ligne d'adresse A20
     (necessaire pour adresser au-dela de 1 Mo)
     Typiquement via le controleur de clavier 8042 :
       MOV AL, 0D1h
       OUT 64h, AL        ; commande d'ecriture au port de sortie
       MOV AL, 0DFh
       OUT 60h, AL        ; activer A20

  3. Preparer la GDT en memoire avec au minimum :
     - Descripteur nul (entree 0, obligatoire)
     - Descripteur de code (entree 1, ex: base=0, limite=FFFFFh, G=1)
     - Descripteur de donnees (entree 2, ex: base=0, limite=FFFFFh, G=1)

  4. Charger GDTR :
       LGDT [gdtr_value]
     ou gdtr_value est une structure de 6 octets :
       dw taille_gdt - 1   ; limite (16 bits)
       dd adresse_gdt       ; base (32 bits)

  5. Activer le bit PE (Protection Enable) dans CR0 :
       MOV EAX, CR0
       OR  EAX, 1          ; bit 0 = PE
       MOV CR0, EAX

  6. Effectuer un JMP FAR vers un selecteur de code protege
     pour vider le pipeline et charger CS :
       JMP 08h:debut_protege
     (08h = selecteur du descripteur de code dans la GDT)

  7. Charger les registres de segment de donnees :
       MOV AX, 10h         ; selecteur du descripteur de donnees
       MOV DS, AX
       MOV ES, AX
       MOV FS, AX
       MOV GS, AX
       MOV SS, AX

  8. Charger la pile :
       MOV ESP, adresse_sommet_pile

  9. Charger l'IDT si necessaire :
       LIDT [idtr_value]

  10. Reactiver les interruptions (STI) si un PIC est configure

  Note : Le JMP FAR a l'etape 6 est crucial. Sans lui, CS contient
  toujours un selecteur de mode reel et le processeur se comporte
  de facon imprevisible.

  6.10 Instructions specifiques au mode protege
  -----------------------------------------------

  Le 80386 fournit des instructions dediees a la gestion du mode
  protege. La plupart sont reservees au ring 0 (privilege maximum).

  6.10.1 Instructions pour les tables de descripteurs

  LGDT mem48   Charge GDTR (base + limite)               Ring 0
  SGDT mem48   Sauvegarde GDTR                            Tous
  LIDT mem48   Charge IDTR (base + limite)               Ring 0
  SIDT mem48   Sauvegarde IDTR                            Tous
  LLDT reg16   Charge LDTR depuis un selecteur            Ring 0
  SLDT reg16   Sauvegarde le selecteur de LDTR            Tous
  LTR  reg16   Charge le Task Register                    Ring 0
  STR  reg16   Sauvegarde le Task Register                Tous

  6.10.2 Instructions de verification des privileges

  ARPL reg16, reg16   Ajuste le RPL d'un selecteur
                       Met RPL(dest) = max(RPL(dest), RPL(src))
                       Met ZF=1 si modifie

  LAR  reg, r/m16     Charge les droits d'acces d'un descripteur
                       Retourne les octets 4-7 du descripteur
                       Met ZF=1 si le descripteur est valide

  LSL  reg, r/m16     Charge la limite d'un segment
                       Retourne la limite en octets ou en pages
                       Met ZF=1 si le descripteur est valide

  VERR r/m16          Verifie si un segment est lisible
                       Met ZF=1 si le segment peut etre lu
                       au niveau de privilege courant

  VERW r/m16          Verifie si un segment est inscriptible
                       Met ZF=1 si le segment peut etre ecrit
                       au niveau de privilege courant

  6.10.3 Instructions pour les registres de controle

  MOV CRn, reg32      Ecriture dans un registre de controle    Ring 0
  MOV reg32, CRn      Lecture d'un registre de controle         Ring 0

  Registres de controle du 80386 :

  CR0 : Controle du processeur
    Bit 0  (PE) : Protection Enable (active le mode protege)
    Bit 1  (MP) : Monitor Coprocessor
    Bit 2  (EM) : Emulation (FPU absent, emuler par logiciel)
    Bit 3  (TS) : Task Switched (mis a 1 lors d'un task switch)
    Bit 4  (ET) : Extension Type (type de FPU : 80287 ou 80387)
    Bit 16 (WP) : Write Protect (protege les pages en ring 0)
    Bit 31 (PG) : Paging (active la pagination)

  CR2 : Adresse lineaire de la faute de page
    Contient l'adresse qui a cause la derniere exception #PF.

  CR3 : Adresse physique du repertoire de pages (PDBR)
    Bits 31-12 : Adresse physique du repertoire (alignee 4 Ko)
    Bit 4 (PCD) : Page Cache Disable
    Bit 3 (PWT) : Page Write-Through

  6.10.4 Instructions pour les registres de debug

  MOV DRn, reg32      Ecriture dans un registre de debug       Ring 0
  MOV reg32, DRn      Lecture d'un registre de debug            Ring 0

  DR0-DR3 : Adresses lineaires des points d'arret materiels
  DR6      : Etat du debug (indique quel breakpoint a ete touche)
  DR7      : Controle du debug (active/configure les breakpoints)
    Chaque breakpoint peut etre configure pour :
    - Arret sur execution
    - Arret sur ecriture memoire
    - Arret sur lecture/ecriture memoire
    - Arret sur acces E/S

  6.10.5 Exemple : configuration minimale du mode protege

  ; --- Descripteurs GDT ---
  gdt_start:
    dq 0                        ; Descripteur nul (obligatoire)

  gdt_code:                     ; Descripteur de code ring 0
    dw 0FFFFh                   ; Limite [15:0]
    dw 0                        ; Base [15:0]
    db 0                        ; Base [23:16]
    db 10011010b                ; P=1, DPL=0, S=1, Type=1010 (code R/X)
    db 11001111b                ; G=1, D=1, Limit[19:16]=Fh
    db 0                        ; Base [31:24]

  gdt_data:                     ; Descripteur de donnees ring 0
    dw 0FFFFh                   ; Limite [15:0]
    dw 0                        ; Base [15:0]
    db 0                        ; Base [23:16]
    db 10010010b                ; P=1, DPL=0, S=1, Type=0010 (data R/W)
    db 11001111b                ; G=1, D=1, Limit[19:16]=Fh
    db 0                        ; Base [31:24]
  gdt_end:

  ; --- GDTR ---
  gdtr_value:
    dw gdt_end - gdt_start - 1 ; Limite
    dd gdt_start                ; Base

  ; --- Passage en mode protege ---
    CLI
    LGDT [gdtr_value]
    MOV EAX, CR0
    OR  EAX, 1
    MOV CR0, EAX
    JMP 08h:protege_start       ; 08h = selecteur du code

  protege_start:
    MOV AX, 10h                 ; 10h = selecteur des donnees
    MOV DS, AX
    MOV ES, AX
    MOV SS, AX
    MOV ESP, 90000h             ; Sommet de pile

  Cet exemple configure un modele "flat" (base=0, limite=4Go)
  couramment utilise par les systemes d'exploitation modernes.
  Tous les segments couvrent l'integralite de l'espace d'adressage,
  et la protection est assuree principalement par la pagination
  (voir section 7).

==============================================================================
7. PAGINATION ET MEMOIRE VIRTUELLE
==============================================================================

  7.1 Vue d'ensemble de la pagination
  -------------------------------------

  La pagination est un mecanisme materiel optionnel du 80386 qui
  traduit les adresses lineaires (produites par la segmentation)
  en adresses physiques. Elle permet :

    - La memoire virtuelle : chaque processus dispose de son propre
      espace d'adressage de 4 Go, independamment de la quantite de
      RAM physique disponible
    - L'isolation des processus : deux processus peuvent utiliser les
      memes adresses lineaires sans conflit, car elles sont mappees
      vers des pages physiques differentes
    - Le swapping : les pages peu utilisees peuvent etre transferees
      sur le disque et rechargees a la demande
    - La protection fine : chaque page peut etre protegee
      individuellement (lecture seule, superviseur/utilisateur)

  La pagination divise la memoire en blocs de taille fixe appeles
  "pages" de 4096 octets (4 Ko). L'espace d'adressage lineaire
  de 4 Go est ainsi divise en 1 048 576 pages (2^20).

  La pagination est activee par le bit PG (bit 31) du registre CR0.
  Elle est independante de la segmentation : les deux mecanismes
  s'appliquent en serie.

  Chaine de traduction complete :

    Adresse logique  ->  Segmentation  ->  Adresse lineaire
    (selecteur:offset)                     (32 bits)

    Adresse lineaire ->  Pagination    ->  Adresse physique
    (32 bits)                              (32 bits)

  Si la pagination est desactivee (CR0.PG = 0), l'adresse lineaire
  est directement utilisee comme adresse physique.

  7.2 Repertoire de pages et tables de pages
  --------------------------------------------

  Le 80386 utilise un schema de pagination a deux niveaux pour
  eviter de stocker une table unique de 4 Mo (2^20 entrees * 4 octets)
  en memoire pour chaque processus.

  Structure a deux niveaux :

  Niveau 1 : Repertoire de pages (Page Directory)
    - Une seule table de 1024 entrees de 4 octets chacune (4 Ko)
    - Chaque entree pointe vers une table de pages (niveau 2)
    - Localise par le registre CR3

  Niveau 2 : Tables de pages (Page Tables)
    - Jusqu'a 1024 tables de 1024 entrees de 4 octets chacune
    - Chaque entree mappe une page lineaire vers une page physique
    - Seules les tables necessaires doivent etre presentes en memoire

  Schema hierarchique :

    CR3 ──> Repertoire de pages (4 Ko, 1024 entrees)
              |
              ├──> Table de pages 0    (4 Ko, 1024 entrees)
              |      ├──> Page physique 0
              |      ├──> Page physique 1
              |      └──> ...
              |
              ├──> Table de pages 1    (4 Ko, 1024 entrees)
              |      ├──> Page physique 1024
              |      └──> ...
              |
              └──> Table de pages 1023 (4 Ko, 1024 entrees)
                     └──> Page physique 1048575

  Chaque table de pages couvre 4 Mo d'espace lineaire
  (1024 pages * 4 Ko = 4 Mo).

  L'espace memoire necessaire pour la pagination est minimal :
    - 4 Ko pour le repertoire de pages (toujours present)
    - 4 Ko par table de pages utilisee
    - Un processus utilisant 16 Mo n'a besoin que de 4 tables
      de pages = 4 Ko + 4 * 4 Ko = 20 Ko de tables

  7.3 Entrees de repertoire de pages (PDE)
  ------------------------------------------

  Chaque entree du repertoire de pages (Page Directory Entry, PDE)
  occupe 4 octets (32 bits) :

    31                  12 11  9 8 7 6 5 4  3  2  1  0
  +----------------------+------+-+-+-+-+--+--+--+--+--+
  | Adresse table pages  | AVL  |0|0|D|A|PCD|PWT|U/S|R/W| P|
  |      [31:12]         |      | | | | |   |   |   |   |  |
  +----------------------+------+-+-+-+-+--+--+--+--+--+

  Champs :

  Adresse [31:12] (20 bits) : Adresse physique de la table de pages
    alignee sur 4 Ko (les 12 bits de poids faible sont implicitement 0).

  P (bit 0) : Present
    1 = La table de pages est presente en memoire
    0 = La table de pages n'est pas presente
        (les bits 31-1 sont libres pour le systeme d'exploitation)
        L'acces genere une exception #PF (Page Fault)

  R/W (bit 1) : Read/Write
    0 = Les pages de cette table sont en lecture seule
    1 = Les pages de cette table sont en lecture/ecriture
    (combine avec le bit R/W de la PTE : acces = PDE.R/W AND PTE.R/W)

  U/S (bit 2) : User/Supervisor
    0 = Acces superviseur uniquement (ring 0, 1, 2)
    1 = Acces utilisateur autorise (ring 3)
    (combine avec la PTE : acces = PDE.U/S AND PTE.U/S)

  PWT (bit 3) : Page Write-Through
    0 = Politique de cache write-back pour cette table
    1 = Politique de cache write-through

  PCD (bit 4) : Page Cache Disable
    0 = La table de pages peut etre mise en cache
    1 = Le cache est desactive pour cette table

  A (bit 5) : Accessed
    Mis a 1 par le processeur lorsqu'une page de cette table
    est accedee (lecture ou ecriture). Jamais remis a 0 par
    le processeur (le systeme d'exploitation doit le faire).

  D (bit 6) : Dirty (reserve dans le PDE, utilise dans le PTE)
    Reserve (mis a 0) dans le repertoire de pages.

  AVL (bits 11-9) : Available
    3 bits disponibles pour le systeme d'exploitation.
    Le processeur ne les modifie pas.

  7.4 Entrees de tables de pages (PTE)
  --------------------------------------

  Chaque entree de table de pages (Page Table Entry, PTE) occupe
  4 octets (32 bits) :

    31                  12 11  9 8 7 6 5 4  3  2  1  0
  +----------------------+------+-+-+-+-+--+--+--+--+--+
  | Adresse page phys.   | AVL  |0|0|D|A|PCD|PWT|U/S|R/W| P|
  |      [31:12]         |      | | | | |   |   |   |   |  |
  +----------------------+------+-+-+-+-+--+--+--+--+--+

  Champs :

  Adresse [31:12] (20 bits) : Adresse physique de la page de 4 Ko.
    Les 12 bits de poids faible sont implicitement 0 (alignement 4 Ko).

  P (bit 0) : Present
    1 = La page physique est presente en memoire
    0 = La page n'est pas presente (swappee sur disque ou non allouee)
        L'acces genere une exception #PF
        Les bits 31-1 sont libres pour le systeme (ex: offset disque)

  R/W (bit 1) : Read/Write
    0 = Page en lecture seule (pour ring 3)
    1 = Page en lecture/ecriture
    Note : en ring 0, toutes les pages sont inscriptibles
    sauf si CR0.WP = 1

  U/S (bit 2) : User/Supervisor
    0 = Page accessible uniquement au superviseur (CPL 0-2)
    1 = Page accessible a tous les niveaux (CPL 0-3)

  PWT (bit 3) : Page Write-Through
    Politique de cache pour cette page specifique.

  PCD (bit 4) : Page Cache Disable
    Desactive le cache pour cette page (utile pour les E/S
    mappees en memoire, MMIO).

  A (bit 5) : Accessed
    Mis a 1 par le processeur lors de tout acces a la page.
    Utilise par le systeme pour les algorithmes de remplacement
    (LRU, clock algorithm).

  D (bit 6) : Dirty
    Mis a 1 par le processeur lors d'une ecriture dans la page.
    Utilise par le systeme pour savoir si la page doit etre
    ecrite sur le disque avant d'etre reutilisee.

  AVL (bits 11-9) : Available
    3 bits libres pour le systeme d'exploitation.

  7.5 Traduction d'adresses lineaires en adresses physiques
  -----------------------------------------------------------

  L'adresse lineaire de 32 bits est decomposee en trois champs :

    31         22 21        12 11          0
  +-------------+------------+-------------+
  |  Directory  |   Table    |   Offset    |
  |  (10 bits)  | (10 bits)  |  (12 bits)  |
  +-------------+------------+-------------+

  Directory (bits 31-22) : Index dans le repertoire de pages (0-1023)
  Table    (bits 21-12)  : Index dans la table de pages (0-1023)
  Offset   (bits 11-0)   : Offset dans la page physique (0-4095)

  Etapes de la traduction :

  1. Le processeur lit CR3 pour obtenir l'adresse physique du
     repertoire de pages.

  2. Il utilise les bits 31-22 (Directory) comme index dans le
     repertoire de pages pour lire la PDE correspondante.
     Adresse de la PDE = CR3 + Directory * 4

  3. Il verifie le bit P de la PDE :
     - Si P=0 : exception #PF (table de pages absente)
     - Si P=1 : il extrait l'adresse de la table de pages
       (bits 31-12 de la PDE)

  4. Il utilise les bits 21-12 (Table) comme index dans la
     table de pages pour lire la PTE correspondante.
     Adresse de la PTE = Adresse_table + Table * 4

  5. Il verifie le bit P de la PTE :
     - Si P=0 : exception #PF (page absente)
     - Si P=1 : il extrait l'adresse physique de la page
       (bits 31-12 de la PTE)

  6. Il concatene l'adresse de la page (20 bits de poids fort)
     avec l'offset (12 bits de poids faible) pour former
     l'adresse physique de 32 bits.
     Adresse physique = PTE[31:12] | Offset[11:0]

  Schema de traduction :

    Adresse lineaire : [ Directory | Table | Offset ]
                           |           |        |
                           v           |        |
    CR3 ──> Page Directory |           |        |
            [Directory] ───+           |        |
                           |           v        |
                           +──> Page Table      |
                                [Table] ────────+
                                        |       |
                                        v       v
                               Page physique + Offset
                                        |
                                        v
                               Adresse physique

  Exemple concret :

  Adresse lineaire = 00401000h
    Directory = 00401000h >> 22 = 1         (bits 31-22)
    Table     = (00401000h >> 12) & 3FFh = 1 (bits 21-12)
    Offset    = 00401000h & FFFh = 000h     (bits 11-0)

  Le processeur :
    1. Lit la PDE a l'index 1 du repertoire
    2. Lit la PTE a l'index 1 de la table de pages
    3. Combine l'adresse de la page avec l'offset 000h

  7.6 Le registre CR3 (PDBR)
  ----------------------------

  Le registre CR3, aussi appele PDBR (Page Directory Base Register),
  contient l'adresse physique du repertoire de pages de la tache
  courante.

    31                  12 11  5  4  3  2        0
  +----------------------+------+---+---+---------+
  | Adresse repertoire   |  0   |PCD|PWT|    0    |
  |      [31:12]         |      |   |   |         |
  +----------------------+------+---+---+---------+

  Adresse [31:12] : Adresse physique du repertoire de pages
    alignee sur 4 Ko.

  PWT (bit 3) : Write-Through pour le repertoire de pages.
  PCD (bit 4) : Cache Disable pour le repertoire de pages.

  Le registre CR3 est modifie :
    - Par MOV CR3, reg32 (instruction privilegiee, ring 0)
    - Automatiquement lors d'une commutation de taches via le TSS
      (le champ CR3 du TSS est charge dans CR3)

  Important : l'ecriture dans CR3 invalide automatiquement toutes
  les entrees du TLB (Translation Lookaside Buffer), sauf celles
  marquees "globales" (bit G, a partir du Pentium).

  Chaque processus a son propre repertoire de pages. Lors du
  changement de contexte, le systeme d'exploitation charge CR3
  avec l'adresse du repertoire du nouveau processus, ce qui
  change instantanement tout le mapping memoire.

  7.7 Cache de traduction (TLB)
  ------------------------------

  La traduction d'une adresse lineaire en adresse physique
  necessite deux acces memoire supplementaires (PDE + PTE), ce
  qui triplerait le temps d'acces. Pour eviter cette penalite,
  le 80386 utilise un TLB (Translation Lookaside Buffer).

  Le TLB est un cache interne au processeur qui stocke les
  traductions recentes (page lineaire -> page physique).

  Caracteristiques du TLB du 80386 :
    - 32 entrees (pages)
    - Associatif a 4 voies (4-way set associative)
    - Couvre 32 * 4 Ko = 128 Ko d'espace d'adressage
    - Taux de succes (hit rate) typique : > 98%

  Fonctionnement :

  1. Le processeur cherche la traduction dans le TLB
     (comparaison des 20 bits de poids fort de l'adresse lineaire)

  2. Si TLB hit : l'adresse physique est retournee immediatement
     (aucun acces memoire supplementaire)

  3. Si TLB miss : le processeur effectue un "page table walk"
     (lecture PDE puis PTE en memoire), met a jour le TLB
     avec la nouvelle traduction, puis retente l'acces

  Invalidation du TLB :

  Le TLB doit etre invalide lorsque les tables de pages sont
  modifiees. Sur le 80386, cela se fait par :

    - MOV CR3, reg32 : invalide tout le TLB
    - Commutation de tache : invalide tout le TLB (via CR3)

  Note : le 80486 et les processeurs suivants ajoutent
  l'instruction INVLPG adresse qui invalide une seule entree
  du TLB, ce qui est plus efficace.

  Impact sur les performances :

  Avec un taux de hit de 98%, la penalite moyenne de la
  pagination est tres faible :
    Temps moyen = 0.98 * T_cache + 0.02 * (T_cache + 2 * T_mem)
  soit environ 1% de surcharge par rapport a un acces direct.

  7.8 Protection par la pagination
  ----------------------------------

  La pagination fournit un deuxieme niveau de protection, en
  complement de la protection par segmentation.

  7.8.1 Protection en lecture/ecriture (R/W)

  Le bit R/W des PDE et PTE controle l'acces en ecriture :

  PDE.R/W  PTE.R/W  Acces effectif (ring 3)
  -------  -------  --------------------------------
    0        0       Lecture seule
    0        1       Lecture seule (PDE prioritaire)
    1        0       Lecture seule
    1        1       Lecture/ecriture

  L'acces effectif est le AND logique des bits R/W de la PDE
  et de la PTE.

  En ring 0 (superviseur), toutes les pages sont inscriptibles
  par defaut, sauf si le bit WP (Write Protect, bit 16 de CR0)
  est a 1. Le bit WP a ete ajoute dans le 80486 mais est
  retro-applique dans les descriptions du 80386 tardives.
  Avec CR0.WP=1, le noyau ne peut pas ecrire dans les pages
  marquees en lecture seule, ce qui est essentiel pour
  implementer le copy-on-write (fork() sous Unix).

  7.8.2 Protection superviseur/utilisateur (U/S)

  Le bit U/S controle l'acces selon le niveau de privilege :

  PDE.U/S  PTE.U/S  Ring 0-2        Ring 3
  -------  -------  --------------- ---------------
    0        0       Acces complet   #PF (interdit)
    0        1       Acces complet   #PF (interdit)
    1        0       Acces complet   #PF (interdit)
    1        1       Acces complet   Acces autorise

  Les deux bits doivent etre a 1 pour autoriser l'acces en
  ring 3. Le superviseur peut toujours acceder a toutes les pages.

  7.8.3 Combinaison segmentation + pagination

  La protection effective est la plus restrictive des deux
  mecanismes :

  - La segmentation verifie les privileges des segments
    (CPL, DPL, RPL) et les types de segments (code, donnees)
  - La pagination verifie les droits par page (R/W, U/S)

  Un acces est autorise seulement si les deux mecanismes
  l'autorisent. Cela permet une protection tres fine :
  par exemple, un segment de donnees en lecture/ecriture peut
  contenir des pages en lecture seule pour proteger certaines
  zones (tables du systeme, code partage, etc.).

  7.9 Faute de page (exception #PF)
  ------------------------------------

  L'exception #PF (Page Fault, vecteur 14) est generee lorsque :

  a) Le bit P (Present) d'une PDE ou PTE est a 0
     (la page ou la table de pages n'est pas en memoire)

  b) Une violation de protection est detectee :
     - Ecriture dans une page en lecture seule (R/W=0) en ring 3
     - Acces a une page superviseur (U/S=0) depuis le ring 3
     - Ecriture dans une page en lecture seule depuis le ring 0
       si CR0.WP=1

  7.9.1 Code d'erreur de #PF

  Le processeur empile un code d'erreur de 32 bits sur la pile
  du handler :

    31                               4  3  2  1  0
  +----------------------------------+--+--+--+--+
  |              Reserve             |RS|U/S|W/R| P|
  +----------------------------------+--+--+--+--+

  P (bit 0) : Cause de la faute
    0 = Page non presente (P=0 dans la PDE ou PTE)
    1 = Violation de protection (page presente mais acces interdit)

  W/R (bit 1) : Type d'acces qui a cause la faute
    0 = L'acces fautif etait une lecture
    1 = L'acces fautif etait une ecriture

  U/S (bit 2) : Niveau de privilege au moment de la faute
    0 = Le processeur etait en mode superviseur (CPL 0-2)
    1 = Le processeur etait en mode utilisateur (CPL 3)

  RS (bit 3) : Reserved bit set (a partir du Pentium)
    1 = La faute a ete causee par un bit reserve mis a 1
        dans une entree de page

  7.9.2 Registre CR2

  Lors d'une exception #PF, le processeur charge le registre CR2
  avec l'adresse lineaire qui a cause la faute. Le handler
  peut lire CR2 pour savoir quelle page doit etre traitee.

    MOV EAX, CR2    ; Lire l'adresse fautive

  7.9.3 Traitement typique d'une faute de page

  Le handler de #PF du systeme d'exploitation effectue
  typiquement les operations suivantes :

  1. Lire CR2 pour obtenir l'adresse fautive
  2. Lire le code d'erreur sur la pile
  3. Determiner la cause :

     a) Page non presente (P=0) :
        - Verifier si l'adresse est valide pour le processus
        - Si oui : charger la page depuis le disque (swap in),
          mettre a jour la PTE (adresse physique + P=1),
          invalider le TLB si necessaire, puis IRET
        - Si non : envoyer un signal SIGSEGV (segfault) au
          processus

     b) Violation de protection (P=1) :
        - Verifier si c'est un copy-on-write :
          Si W/R=1 et la page est partagee, copier la page,
          mettre a jour la PTE avec la copie, IRET
        - Sinon : erreur de protection, SIGSEGV

     c) Ecriture dans une page en lecture seule :
        - Implementer le copy-on-write si applicable
        - Sinon : SIGSEGV

  7.10 Activation de la pagination
  ----------------------------------

  La sequence typique pour activer la pagination :

  Prerequis :
    - Le processeur doit etre en mode protege (CR0.PE = 1)
    - Un repertoire de pages et au moins une table de pages
      doivent etre prepares en memoire physique
    - Le code d'activation doit etre mappe de facon identique
      (identity mapping) pour eviter un saut dans le vide
      apres l'activation

  7.10.1 Preparation du repertoire et des tables de pages

  ; Allouer une page physique pour le repertoire (alignee 4 Ko)
  ; Initialiser toutes les entrees a 0 (non present)
  ; Creer une table de pages pour les 4 premiers Mo (identity map)

  page_directory  equ 100000h     ; Adresse du repertoire
  page_table_0    equ 101000h     ; Adresse de la premiere table

  ; Remplir la table de pages 0 : identity mapping des 4 premiers Mo
  ; Chaque entree mappe page N vers l'adresse physique N * 4096

    MOV EDI, page_table_0
    MOV EAX, 3                  ; P=1, R/W=1, U/S=0
    MOV ECX, 1024
  .fill_pt0:
    STOSD                       ; Ecrire l'entree
    ADD EAX, 1000h              ; Page suivante (+4096)
    LOOP .fill_pt0

  ; Remplir l'entree 0 du repertoire de pages

    MOV DWORD PTR [page_directory], page_table_0 OR 3
    ; Adresse de la table | P=1, R/W=1

  7.10.2 Activation de la pagination

    ; Charger CR3 avec l'adresse du repertoire
    MOV EAX, page_directory
    MOV CR3, EAX

    ; Activer la pagination (bit PG de CR0)
    MOV EAX, CR0
    OR  EAX, 80000000h         ; Bit 31 = PG
    MOV CR0, EAX

    ; JMP pour vider le pipeline de prefetch
    JMP .paging_enabled
  .paging_enabled:
    ; La pagination est maintenant active
    ; Toutes les adresses lineaires sont desormais traduites

  Important : Apres l'activation de la pagination, le code en
  cours d'execution doit etre accessible via le nouveau mapping.
  C'est pourquoi un identity mapping (adresse lineaire = adresse
  physique) est necessaire au minimum pour la zone de code.

  7.10.3 Configuration typique pour un systeme d'exploitation

  Un systeme d'exploitation moderne utilise typiquement :

  - Identity mapping des premiers Mo pour le code du noyau
  - Le noyau mappe dans les adresses hautes (ex: C0000000h-FFFFFFFFh)
    de chaque processus (higher-half kernel)
  - Chaque processus a son propre repertoire de pages
  - Les pages du noyau sont marquees superviseur (U/S=0)
  - Les pages utilisateur sont marquees U/S=1

  Avec un higher-half kernel a l'adresse C0000000h :
    - Les PDE 0-767   couvrent l'espace utilisateur (0-BFFFFFFFh)
    - Les PDE 768-1023 couvrent l'espace noyau (C0000000h-FFFFFFFFh)

  Lors du changement de processus, seules les PDE 0-767 changent.
  Les PDE du noyau restent identiques dans tous les repertoires.

  7.11 Memoire virtuelle et swapping
  -------------------------------------

  La pagination permet d'implementer la memoire virtuelle, ou
  l'espace d'adressage total des processus peut depasser la
  quantite de RAM physique disponible.

  7.11.1 Principe du swapping

  Quand la RAM physique est pleine et qu'une nouvelle page
  doit etre chargee :

  1. Le systeme choisit une page "victime" a evincer
     (algorithme de remplacement de pages)
  2. Si la page victime est "dirty" (D=1), il l'ecrit sur le
     disque (swap out)
  3. Il marque la PTE de la page victime comme non presente (P=0)
     et stocke l'emplacement sur le disque dans les bits libres
  4. Il charge la nouvelle page depuis le disque (swap in)
  5. Il met a jour la PTE avec la nouvelle adresse physique (P=1)
  6. Il invalide l'entree du TLB correspondante

  7.11.2 Bits A et D pour le remplacement de pages

  Les bits Accessed (A) et Dirty (D) sont essentiels pour les
  algorithmes de remplacement de pages :

  Bit A (Accessed) :
    - Mis a 1 automatiquement par le processeur a chaque acces
    - Le systeme le remet periodiquement a 0
    - Si A=0 apres un certain temps, la page est "peu utilisee"
      et peut etre evincee en priorite

  Bit D (Dirty) :
    - Mis a 1 automatiquement lors d'une ecriture
    - Indique que la page a ete modifiee et doit etre
      sauvegardee sur le disque avant eviction
    - Une page "clean" (D=0) peut etre evincee sans ecriture
      disque (gain de performance)

  7.11.3 Algorithmes de remplacement courants

  Les systemes d'exploitation utilisent differents algorithmes
  pour choisir la page a evincer :

  - Clock algorithm (horloge) :
    Parcourt les pages en cercle, remet A a 0.
    Evince la premiere page avec A=0.
    Bonne approximation de LRU, facile a implementer.

  - LRU approxime (Least Recently Used) :
    Utilise le bit A pour estimer l'ordre d'acces.
    Evince la page la moins recemment accedee.

  - NRU (Not Recently Used) :
    Classe les pages en 4 categories selon A et D :
      Classe 0 : A=0, D=0 (non accedee, non modifiee) - meilleure victime
      Classe 1 : A=0, D=1 (non accedee, modifiee)
      Classe 2 : A=1, D=0 (accedee, non modifiee)
      Classe 3 : A=1, D=1 (accedee, modifiee) - pire victime

  7.11.4 Informations dans une PTE non presente

  Quand une page est swappee (P=0), le systeme d'exploitation
  peut utiliser les 31 bits restants (bits 31-1) de la PTE
  pour stocker des informations :

    31                               1 0
  +-----------------------------------+-+
  | Information systeme (emplacement)  |0|  <- P=0
  +-----------------------------------+-+

  Typiquement, ces bits contiennent :
    - L'offset dans le fichier de swap (partition ou fichier disque)
    - Un indicateur du type de page (swap, fichier mappe, etc.)
    - Le processeur ne lit jamais ces bits si P=0

  7.11.5 Exemple : tailles de memoire virtuelle

  Avec 16 Mo de RAM physique et 256 Mo de swap :

  - Chaque processus peut avoir un espace de 4 Go
  - Le systeme peut maintenir environ 272 Mo de pages
    (16 Mo RAM + 256 Mo swap)
  - Les pages actives restent en RAM
  - Les pages inactives sont deplacees sur le swap
  - L'exception #PF gere transparentement les chargements

  7.11.6 Resume : comparaison segmentation vs pagination

  Critere              Segmentation          Pagination
  ──────────────────   ──────────────────    ──────────────────
  Unite de memoire     Segment (variable)    Page (4 Ko fixe)
  Fragmentation        Externe               Interne (faible)
  Tables               GDT/LDT              Page Directory/Tables
  Protection           DPL, type segment     U/S, R/W par page
  Memoire virtuelle    Limitee (NP flag)     Complete (swap)
  Taille typique       Variable (1 a 4 Go)  Fixe (4096 octets)
  Localisation         GDTR, LDTR            CR3 (PDBR)
  Transparence         Non (selecteurs)      Oui (adresses plates)

  En pratique, les systemes d'exploitation modernes utilisent
  un modele "flat" avec segmentation minimale (base=0, limite=4Go)
  et s'appuient presque exclusivement sur la pagination pour
  la gestion de la memoire et la protection.

==============================================================================
8. INTERRUPTIONS ET EXCEPTIONS
==============================================================================

  8.1 Vue d'ensemble
  -------------------

  Les interruptions et exceptions sont des mecanismes qui
  suspendent l'execution normale du processeur pour traiter
  un evenement. Le 80386 supporte 256 vecteurs d'interruption
  (numerotes de 0 a 255), chacun associe a un handler via la
  table des descripteurs d'interruptions (IDT).

  Terminologie :

  - Interruption : evenement asynchrone externe au processeur
    (materiel) ou declenche explicitement par logiciel (INT n).
    L'execution peut reprendre normalement apres le traitement.

  - Exception : evenement synchrone genere par le processeur
    lui-meme en reponse a une erreur ou condition speciale
    detectee pendant l'execution d'une instruction.

  Les interruptions et exceptions provoquent :
    1. Sauvegarde du contexte (EFLAGS, CS, EIP) sur la pile
    2. Transfert du controle au handler via l'IDT
    3. Retour par IRET apres le traitement

  8.2 Classification
  -------------------

  8.2.1 Interruptions materielles (externes)

  Generees par des peripheriques via les lignes d'interruption
  du processeur :

  - INTR (Interrupt Request) : interruptions masquables.
    Peuvent etre desactivees par CLI (Clear Interrupt Flag)
    et reactivees par STI (Set Interrupt Flag).
    Le drapeau IF de EFLAGS controle leur acceptation.

  - NMI (Non-Maskable Interrupt) : interruption non masquable.
    Toujours acceptee, vecteur 2. Utilisee pour les erreurs
    critiques (erreur de parite memoire, watchdog, etc.).

  8.2.2 Interruptions logicielles

  Declenchees explicitement par l'instruction INT n :

    INT 21h     ; Appel systeme DOS (mode reel)
    INT 80h     ; Appel systeme Linux (mode protege)
    INT 3       ; Point d'arret (breakpoint, opcode CC)
    INTO        ; INT 4 si OF=1 (Overflow)

  Les interruptions logicielles ne sont pas affectees par
  le drapeau IF (elles ne sont pas masquables).

  8.2.3 Exceptions

  Generees par le processeur lors de la detection d'erreurs.
  Trois categories :

  a) Faults (fautes) :
     L'exception est signalee AVANT l'execution de l'instruction
     fautive. CS:EIP pointe sur l'instruction qui a cause la
     faute. Apres correction, l'instruction peut etre relancee.
     Exemple : #PF (Page Fault) - la page est chargee puis
     l'instruction est reexecutee.

  b) Traps (pieges) :
     L'exception est signalee APRES l'execution de l'instruction.
     CS:EIP pointe sur l'instruction suivante. L'execution
     reprend normalement apres le traitement.
     Exemple : #DB (Debug) apres une instruction en mode trace,
     INT 3 (Breakpoint).

  c) Aborts (abandons) :
     L'exception signale une erreur grave qui ne permet pas
     de localiser precisement l'instruction fautive ni de
     reprendre l'execution.
     Exemple : #DF (Double Fault), erreur materielle.

  8.3 Table des descripteurs d'interruptions (IDT)
  --------------------------------------------------

  L'IDT est une table en memoire contenant jusqu'a 256
  descripteurs de portes (gates). Chaque descripteur occupe
  8 octets, soit une taille maximale de 2048 octets (256 * 8).

  Le registre IDTR (IDT Register) contient :
    - L'adresse lineaire de base de l'IDT (32 bits)
    - La limite de l'IDT en octets (16 bits)

  Format du registre IDTR :

    47                16 15        0
  +--------------------+-----------+
  | Base (32 bits)     | Limite    |
  +--------------------+-----------+

  Instructions associees :

    LIDT mem48    ; Charger IDTR (instruction privilegiee)
    SIDT mem48    ; Sauvegarder IDTR (accessible depuis tout CPL)

  Exemple de chargement de l'IDT :

    idt_descriptor:
      DW  256*8 - 1       ; Limite = 2047
      DD  idt_table        ; Adresse de base

    LIDT [idt_descriptor]

  L'IDT peut contenir trois types de descripteurs :
    - Porte d'interruption (Interrupt Gate)
    - Porte de piege (Trap Gate)
    - Porte de tache (Task Gate)

  En mode reel, l'equivalent est l'IVT (Interrupt Vector Table)
  situee a l'adresse 0000:0000, contenant 256 pointeurs far
  (segment:offset) de 4 octets chacun (1024 octets total).

  8.4 Format des portes d'interruption et trap
  -----------------------------------------------

  8.4.1 Porte d'interruption (Interrupt Gate, type 0Eh pour 32 bits)

    63           48 47 46 45 44    40 39    37 36  35  32
  +----------------+--+-----+--------+--------+--+------+
  | Offset [31:16] | P| DPL | 0D110  |  000   |0 | res  |
  +----------------+--+-----+--------+--------+--+------+
    31           16 15                               0
  +----------------+----------------------------------+
  | Selecteur seg. |        Offset [15:0]             |
  +----------------+----------------------------------+

  Champs :

  Offset [31:16] et [15:0] : Adresse du handler dans le segment
    de code designe par le selecteur. Les deux parties forment
    l'offset 32 bits complet.

  Selecteur de segment (16 bits) : Selecteur du segment de code
    contenant le handler. Doit pointer vers un segment de code
    dans la GDT ou la LDT.

  P (bit 47) : Present
    1 = Le descripteur est valide
    0 = Le descripteur n'est pas present (#NP si utilise)

  DPL (bits 46-45) : Descriptor Privilege Level
    Niveau de privilege requis pour invoquer cette porte
    via INT n. Le CPL doit etre <= DPL pour les interruptions
    logicielles. Les exceptions et interruptions materielles
    ignorent le DPL.

  Type (bits 44-40) : 0D110 pour une porte d'interruption 32 bits
    D=1 pour 32 bits (type = 0Eh), D=0 pour 16 bits (type = 06h)

  Comportement specifique : quand le processeur passe par une
  porte d'interruption, le drapeau IF est automatiquement
  efface (IF=0), desactivant les interruptions masquables.
  Cela empeche les interruptions imbriquees involontaires.

  8.4.2 Porte de piege (Trap Gate, type 0Fh pour 32 bits)

  Format identique a la porte d'interruption, sauf :

  Type (bits 44-40) : 0D111 pour une porte de piege 32 bits
    D=1 pour 32 bits (type = 0Fh), D=0 pour 16 bits (type = 07h)

  Difference cle : le drapeau IF n'est PAS modifie lors du
  passage par une porte de piege. Les interruptions masquables
  restent activees si elles l'etaient avant.

  Utilisation typique :
    - Portes d'interruption : pour les interruptions materielles
      et les handlers qui ne doivent pas etre interrompus
    - Portes de piege : pour les exceptions (#PF, #GP, etc.)
      et les appels systeme (INT 80h sous Linux)

  8.4.3 Porte de tache (Task Gate, type 05h)

  Contient un selecteur de TSS au lieu d'un offset de handler.
  Le processeur effectue un changement de tache complet pour
  traiter l'interruption. Rarement utilise en pratique sauf
  pour le Double Fault (#DF) ou certains debuggers.

    63           48 47 46 45 44    40 39              32
  +----------------+--+-----+--------+-----------------+
  |   Reserve      | P| DPL | 00101  |    Reserve      |
  +----------------+--+-----+--------+-----------------+
    31           16 15                               0
  +----------------+----------------------------------+
  | Selecteur TSS  |           Reserve                |
  +----------------+----------------------------------+

  8.5 Exceptions du 80386
  -------------------------

  Le 80386 definit 17 exceptions (vecteurs 0 a 16). Les
  vecteurs 17-31 sont reserves par Intel pour les futurs
  processeurs. Les vecteurs 32-255 sont disponibles pour
  les interruptions materielles et logicielles.

  Table complete des exceptions :

  Vec  Mnemo  Type    Err  Description
  ---  -----  ------  ---  ------------------------------------
   0   #DE    Fault   Non  Division Error (DIV/IDIV par zero
                           ou quotient trop grand)
   1   #DB    Fault/  Non  Debug (point d'arret materiel,
              Trap         mode trace T=1, registres DRx)
   2   NMI    Int     Non  Non-Maskable Interrupt (erreur
                           materielle critique)
   3   #BP    Trap    Non  Breakpoint (instruction INT 3,
                           opcode CC sur 1 octet)
   4   #OF    Trap    Non  Overflow (instruction INTO
                           quand OF=1 dans EFLAGS)
   5   #BR    Fault   Non  BOUND Range Exceeded (instruction
                           BOUND, index hors limites)
   6   #UD    Fault   Non  Invalid Opcode (opcode inconnu
                           ou instruction invalide)
   7   #NM    Fault   Non  Device Not Available (instruction
                           FPU avec CR0.EM=1 ou CR0.TS=1)
   8   #DF    Abort   Oui  Double Fault (exception pendant
                           le traitement d'une exception,
                           code d'erreur toujours 0)
   9   ---    Abort   Non  Coprocessor Segment Overrun
                           (obsolete depuis le 80486)
  10   #TS    Fault   Oui  Invalid TSS (selecteur TSS invalide
                           lors d'une commutation de tache)
  11   #NP    Fault   Oui  Segment Not Present (bit P=0 dans
                           un descripteur de segment)
  12   #SS    Fault   Oui  Stack-Segment Fault (depassement
                           de pile ou segment SS invalide)
  13   #GP    Fault   Oui  General Protection (violation de
                           protection generique, la plus
                           courante des exceptions)
  14   #PF    Fault   Oui  Page Fault (page non presente ou
                           violation de protection par page,
                           voir section 7.9)
  15   ---    ---     ---  Reserve par Intel
  16   #MF    Fault   Non  x87 FPU Floating-Point Error
                           (erreur FPU non masquee)

  Colonne "Err" : indique si le processeur empile un code
  d'erreur de 32 bits en plus de EFLAGS/CS/EIP.

  8.5.1 Detail des exceptions les plus courantes

  #DE - Division Error (vecteur 0)
    Declenchee par DIV ou IDIV quand le diviseur est 0 ou
    quand le quotient ne tient pas dans le registre destination.
    CS:EIP pointe sur l'instruction DIV/IDIV fautive.

  #DB - Debug (vecteur 1)
    Declenchee dans plusieurs situations :
    - Point d'arret materiel (registres DR0-DR3 + DR7)
    - Mode trace pas-a-pas (TF=1 dans EFLAGS)
    - Acces aux registres de debug (DR4-DR5 quand GD=1)
    Le registre DR6 contient les informations sur la cause.

  #BP - Breakpoint (vecteur 3)
    Declenchee par l'instruction INT 3 (opcode CC, 1 octet).
    Utilisee par les debuggers pour poser des points d'arret
    logiciels. L'opcode court (1 octet) permet de remplacer
    n'importe quelle instruction en memoire.

  #UD - Invalid Opcode (vecteur 6)
    Declenchee quand le processeur rencontre un opcode
    invalide ou une combinaison d'operandes illegale.
    Aussi declenchee si LOCK est utilise avec une instruction
    qui ne le supporte pas.

  #NM - Device Not Available (vecteur 7)
    Declenchee par une instruction FPU (WAIT/FWAIT ou ESC)
    quand CR0.EM=1 (emulation) ou CR0.TS=1 (task switched).
    Utilisee pour implementer le "lazy FPU context switching" :
    le contexte FPU n'est sauvegarde/restaure que quand un
    processus utilise effectivement le FPU.

  #DF - Double Fault (vecteur 8)
    Declenchee quand une exception se produit pendant le
    traitement d'une autre exception et que les deux ne
    peuvent pas etre traitees en serie (voir section 8.11).
    Le code d'erreur est toujours 0.

  #GP - General Protection (vecteur 13)
    L'exception la plus generique, couvrant de nombreuses
    violations :
    - Acces memoire depassant la limite d'un segment
    - Ecriture dans un segment en lecture seule
    - Chargement d'un selecteur invalide dans un registre
    - Execution d'une instruction privilegiee en ring 3
    - Acces a une porte avec un DPL insuffisant
    - Reference a un descripteur NULL (selecteur 0)
    Le code d'erreur contient le selecteur fautif ou 0.

  #PF - Page Fault (vecteur 14)
    Voir la section 7.9 pour le detail complet. Le registre
    CR2 contient l'adresse lineaire fautive. Le code d'erreur
    indique la cause (P/W-R/U-S).

  #MF - x87 FPU Error (vecteur 16)
    Declenchee lors de l'execution de WAIT/FWAIT ou d'une
    instruction FPU quand une exception FPU est en attente
    (non masquee dans le registre de controle FPU).

  8.6 Interruptions materielles et le PIC 8259A
  ------------------------------------------------

  8.6.1 Le controleur d'interruptions programmable (PIC)

  Le 80386 ne gere pas directement les interruptions des
  peripheriques. Un controleur externe, le PIC 8259A, sert
  d'intermediaire. La configuration standard du PC utilise
  deux PIC en cascade :

  PIC maitre (ports 20h/21h) :
    IRQ 0 : Timer systeme (8253/8254)       -> Vecteur 08h (20h)
    IRQ 1 : Clavier                         -> Vecteur 09h (21h)
    IRQ 2 : Cascade vers PIC esclave        -> (pas de vecteur)
    IRQ 3 : Port serie COM2/COM4            -> Vecteur 0Bh (23h)
    IRQ 4 : Port serie COM1/COM3            -> Vecteur 0Ch (24h)
    IRQ 5 : Port parallele LPT2 / Son      -> Vecteur 0Dh (25h)
    IRQ 6 : Controleur de disquettes        -> Vecteur 0Eh (26h)
    IRQ 7 : Port parallele LPT1            -> Vecteur 0Fh (27h)

  PIC esclave (ports A0h/A1h) :
    IRQ 8  : Horloge temps reel (RTC)       -> Vecteur 70h (28h)
    IRQ 9  : Redirection IRQ 2 / ACPI       -> Vecteur 71h (29h)
    IRQ 10 : Disponible                     -> Vecteur 72h (2Ah)
    IRQ 11 : Disponible                     -> Vecteur 73h (2Bh)
    IRQ 12 : Souris PS/2                    -> Vecteur 74h (2Ch)
    IRQ 13 : Coprocesseur FPU               -> Vecteur 75h (2Dh)
    IRQ 14 : Controleur IDE primaire        -> Vecteur 76h (2Eh)
    IRQ 15 : Controleur IDE secondaire      -> Vecteur 77h (2Fh)

  Les vecteurs entre parentheses indiquent le remappage
  typique en mode protege (base 20h pour le maitre, 28h
  pour l'esclave) pour eviter les conflits avec les vecteurs
  d'exceptions Intel (0-1Fh).

  8.6.2 Remappage des IRQ en mode protege

  En mode reel, les IRQ 0-7 utilisent les vecteurs 08h-0Fh,
  ce qui entre en conflit avec les vecteurs d'exceptions du
  80386 (vecteurs 8-15). Il est essentiel de remapper les
  PIC en mode protege.

  Sequence de remappage des PIC :

    ; Reinitialisation des PIC (ICW1)
    MOV AL, 11h         ; ICW1: initialisation + ICW4 needed
    OUT 20h, AL         ; PIC maitre
    OUT A0h, AL         ; PIC esclave

    ; Vecteurs de base (ICW2)
    MOV AL, 20h         ; IRQ 0-7 -> vecteurs 20h-27h
    OUT 21h, AL         ; PIC maitre
    MOV AL, 28h         ; IRQ 8-15 -> vecteurs 28h-2Fh
    OUT A1h, AL         ; PIC esclave

    ; Cascade (ICW3)
    MOV AL, 04h         ; IRQ2 = cascade
    OUT 21h, AL         ; PIC maitre
    MOV AL, 02h         ; Esclave connecte sur IRQ2
    OUT A1h, AL         ; PIC esclave

    ; Mode 8086 (ICW4)
    MOV AL, 01h         ; Mode 8086/88
    OUT 21h, AL         ; PIC maitre
    OUT A1h, AL         ; PIC esclave

    ; Masques d'interruption (OCW1)
    MOV AL, 00h         ; Toutes les IRQ demasquees
    OUT 21h, AL         ; PIC maitre (ou FFh pour tout masquer)
    OUT A1h, AL         ; PIC esclave

  8.6.3 Acquittement d'interruption (EOI)

  Apres le traitement d'une IRQ, le handler doit envoyer
  un signal End-Of-Interrupt (EOI) au PIC pour debloquer
  les interruptions de meme priorite ou inferieure.

    ; EOI pour le PIC maitre (IRQ 0-7)
    MOV AL, 20h         ; Commande EOI non specifique
    OUT 20h, AL

    ; EOI pour le PIC esclave (IRQ 8-15)
    ; Il faut envoyer l'EOI aux DEUX PIC
    MOV AL, 20h
    OUT A0h, AL          ; EOI au PIC esclave
    MOV AL, 20h
    OUT 20h, AL          ; EOI au PIC maitre (cascade)

  8.6.4 Masquage des interruptions

  Chaque PIC possede un registre de masque (IMR) accessible
  via le port de donnees (21h pour le maitre, A1h pour l'esclave).
  Un bit a 1 masque l'IRQ correspondante.

    ; Masquer IRQ 1 (clavier) uniquement
    IN  AL, 21h          ; Lire le masque actuel
    OR  AL, 02h          ; Mettre le bit 1 a 1
    OUT 21h, AL          ; Ecrire le nouveau masque

    ; Demasquer IRQ 1 (clavier)
    IN  AL, 21h
    AND AL, NOT 02h      ; FDh = 11111101b
    OUT 21h, AL

  8.7 Interruptions logicielles (INT n)
  ----------------------------------------

  L'instruction INT n declenche l'interruption de vecteur n.
  Elle est utilisee pour les appels systeme et les services
  du BIOS/DOS.

  Encodage :

    INT 3       ; Opcode CC (1 octet, cas special breakpoint)
    INT n       ; Opcode CD n (2 octets, n = 0-255)
    INTO        ; Opcode CE (INT 4 si OF=1)

  Comportement de INT n :

  1. Le processeur verifie le DPL de la porte dans l'IDT :
     CPL <= DPL pour les interruptions logicielles.
     Si CPL > DPL : exception #GP (protection).

  2. Empile EFLAGS, CS et EIP sur la pile (du handler)

  3. Si porte d'interruption : efface IF et TF
     Si porte de piege : efface seulement TF

  4. Charge CS:EIP depuis le descripteur de la porte

  5. Execute le handler

  Interruptions logicielles courantes (mode reel) :

    INT 10h     ; Services video BIOS
    INT 13h     ; Services disque BIOS
    INT 16h     ; Services clavier BIOS
    INT 21h     ; Services DOS
    INT 33h     ; Driver souris

  Interruptions logicielles courantes (mode protege) :

    INT 80h     ; Appels systeme Linux (convention i386)
    INT 2Eh     ; Appels systeme Windows NT (ancienne methode)
    INT 3       ; Breakpoint (debugger)

  8.8 Mecanisme de dispatch des interruptions
  ----------------------------------------------

  Quand une interruption ou exception est declenchee, le
  processeur execute la sequence suivante :

  8.8.1 Sans changement de privilege (intra-privilege)

  Si le handler s'execute au meme niveau de privilege que
  le code interrompu (meme CPL) :

  1. Empiler EFLAGS (32 bits)
  2. Empiler CS (16 bits, etendu a 32 bits)
  3. Empiler EIP (32 bits)
  4. Si l'exception a un code d'erreur : empiler le code (32 bits)
  5. Si porte d'interruption : IF = 0, TF = 0
     Si porte de piege : TF = 0
  6. Charger CS depuis le selecteur de la porte
  7. Charger EIP depuis l'offset de la porte

  Etat de la pile apres l'interruption :

    ESP+12 : EFLAGS (ancien)
    ESP+8  : CS     (ancien)
    ESP+4  : EIP    (ancien)
    ESP    : Code d'erreur (si applicable)

  8.8.2 Avec changement de privilege (inter-privilege)

  Si le handler s'execute a un niveau de privilege plus
  eleve (CPL du handler < CPL du code interrompu), le
  processeur effectue un changement de pile :

  1. Lire le SS:ESP de la nouvelle pile dans le TSS
     (champ SS0:ESP0 pour passage au ring 0)
  2. Empiler l'ancien SS (16 bits, etendu a 32 bits)
  3. Empiler l'ancien ESP (32 bits)
  4. Empiler EFLAGS (32 bits)
  5. Empiler CS (16 bits, etendu a 32 bits)
  6. Empiler EIP (32 bits)
  7. Si code d'erreur : empiler le code (32 bits)
  8. Charger SS:ESP depuis le TSS
  9. Si porte d'interruption : IF = 0, TF = 0
  10. Charger CS:EIP depuis la porte

  Etat de la pile (ring 0) apres l'interruption :

    ESP+20 : SS     (ancien, ring 3)
    ESP+16 : ESP    (ancien, ring 3)
    ESP+12 : EFLAGS (ancien)
    ESP+8  : CS     (ancien)
    ESP+4  : EIP    (ancien)
    ESP    : Code d'erreur (si applicable)

  Ce mecanisme est essentiel pour la securite : le code
  utilisateur (ring 3) ne peut pas corrompre la pile du
  noyau (ring 0) car le processeur change automatiquement
  de pile avant d'empiler quoi que ce soit.

  8.8.3 Schema du mecanisme de dispatch

    Interruption/Exception
           |
           v
    Lire le descripteur dans l'IDT[vecteur]
           |
           v
    Verifier P=1, DPL >= CPL (pour INT n logiciel)
           |
           +--- Changement de privilege necessaire ?
           |            |                    |
           |           Oui                  Non
           |            |                    |
           |    Lire SS:ESP du TSS   Utiliser la pile
           |    Changer de pile       actuelle
           |            |                    |
           +------------+--------------------+
           |
           v
    Empiler [SS, ESP], EFLAGS, CS, EIP [, Error Code]
           |
           v
    IF = 0 (si Interrupt Gate), TF = 0
           |
           v
    CS:EIP = Handler (depuis la porte IDT)
           |
           v
    Execution du handler
           |
           v
    IRET -> Restauration de CS:EIP, EFLAGS [, SS:ESP]

  8.9 Instructions liees aux interruptions
  -------------------------------------------

  8.9.1 CLI - Clear Interrupt Flag

    CLI                 ; IF = 0 (desactive les interruptions masquables)

  Opcode : FA
  Privilege : IOPL (en mode protege, CPL <= IOPL requis)
  Si CPL > IOPL : exception #GP(0)
  Utilisation : sections critiques, handlers d'interruptions

  8.9.2 STI - Set Interrupt Flag

    STI                 ; IF = 1 (active les interruptions masquables)

  Opcode : FB
  Privilege : IOPL (en mode protege, CPL <= IOPL requis)
  Note importante : les interruptions ne sont activees qu'apres
  l'instruction SUIVANT STI, ce qui permet la sequence :
    STI
    HLT                 ; L'interruption reveille le processeur

  8.9.3 INT n - Software Interrupt

    INT imm8            ; Declenche l'interruption de vecteur n

  Opcode : CD n (2 octets), sauf INT 3 = CC (1 octet)
  Empile EFLAGS, CS, EIP. Efface IF et TF (si Interrupt Gate).
  Verifie DPL >= CPL pour les interruptions logicielles.

  8.9.4 INTO - Interrupt on Overflow

    INTO                ; Si OF=1 : INT 4, sinon NOP

  Opcode : CE
  Utilisee apres des operations arithmetiques signees pour
  detecter les debordements :
    ADD EAX, EBX
    INTO                ; Exception si debordement

  8.9.5 IRET / IRETD - Interrupt Return

    IRET                ; Retour d'interruption 16 bits
    IRETD               ; Retour d'interruption 32 bits

  Opcode : CF
  Depile EIP, CS, EFLAGS (et SS, ESP si changement de privilege).
  Restaure le contexte precedent l'interruption.

  En mode protege 32 bits, IRETD est utilise :
  - Depile EIP (32 bits)
  - Depile CS  (32 bits, seuls les 16 bits bas sont utilises)
  - Depile EFLAGS (32 bits)
  - Si retour a un niveau de privilege inferieur :
    depile aussi ESP et SS (retour ring 0 -> ring 3)

  Restrictions :
  - Le drapeau IOPL ne peut etre modifie que si CPL = 0
  - Le drapeau IF ne peut etre modifie que si CPL <= IOPL
  - Le drapeau VM (Virtual 8086) ne peut etre active que depuis ring 0

  8.9.6 BOUND - Check Array Bounds

    BOUND reg16, mem32  ; Verifie reg16 dans [mem:mem+2]
    BOUND reg32, mem64  ; Verifie reg32 dans [mem:mem+4]

  Opcode : 62 /r
  Si l'index est hors des limites, declenche l'exception
  #BR (vecteur 5). Utile pour la verification des bornes
  de tableaux dans les langages de haut niveau.

    bounds  DD 0, 99          ; Limites : 0 a 99
    BOUND EAX, [bounds]       ; #BR si EAX < 0 ou EAX > 99

  8.9.7 HLT - Halt

    HLT                 ; Arrete le processeur jusqu'a la
                        ; prochaine interruption, NMI ou RESET

  Opcode : F4
  Privilege : instruction privilegiee (CPL = 0 requis)
  Utilisation typique dans les boucles d'attente du noyau :
    .idle_loop:
      STI               ; Activer les interruptions
      HLT               ; Attendre une interruption
      JMP .idle_loop    ; Reprendre apres l'interruption

  8.10 Gestion des exceptions avec code d'erreur
  -------------------------------------------------

  Certaines exceptions empilent un code d'erreur de 32 bits
  apres EFLAGS/CS/EIP. Le handler doit depiler ce code
  avant d'executer IRET.

  8.10.1 Format general du code d'erreur

    31                              3  2  1  0
  +----------------------------------+--+--+--+
  |        Index de selecteur        |TI|IDT|EXT|
  +----------------------------------+--+--+--+

  EXT (bit 0) : External event
    1 = L'exception a ete causee par un evenement externe
        (interruption materielle)
    0 = L'exception a ete causee par le code en cours

  IDT (bit 1) : IDT reference
    1 = L'index fait reference a un descripteur dans l'IDT
    0 = L'index fait reference a un descripteur dans la
        GDT ou LDT (selon TI)

  TI (bit 2) : Table Indicator (quand IDT=0)
    0 = L'index fait reference a la GDT
    1 = L'index fait reference a la LDT

  Index (bits 15-3) : Index du descripteur dans la table
    (divise par 8 pour obtenir le selecteur)

  8.10.2 Code d'erreur de #PF (Page Fault)

  Le code d'erreur de #PF a un format different
  (voir section 7.9.1) :

    Bit 0 (P)   : 0 = page non presente, 1 = violation protection
    Bit 1 (W/R) : 0 = lecture, 1 = ecriture
    Bit 2 (U/S) : 0 = superviseur, 1 = utilisateur

  8.10.3 Depilage du code d'erreur dans le handler

  Le handler DOIT depiler le code d'erreur avant IRET,
  car IRET ne le fait pas automatiquement :

    handler_gp:           ; Handler de #GP
      POP EAX             ; Depiler le code d'erreur dans EAX
      ; ... traitement de l'exception ...
      ; Analyser le code d'erreur dans EAX
      IRETD               ; Retour (depile EIP, CS, EFLAGS)

  Alternative sans utiliser de registre :

    handler_gp:
      ADD ESP, 4          ; Ignorer le code d'erreur
      ; ... traitement ...
      IRETD

  8.10.4 Exceptions avec et sans code d'erreur

  Avec code d'erreur    : #DF(8), #TS(10), #NP(11), #SS(12),
                          #GP(13), #PF(14)

  Sans code d'erreur    : #DE(0), #DB(1), NMI(2), #BP(3),
                          #OF(4), #BR(5), #UD(6), #NM(7),
                          Copro(9), #MF(16)

  Le handler doit connaitre le type d'exception pour savoir
  s'il doit depiler un code d'erreur ou non.

  8.11 Double faute et triple faute
  ------------------------------------

  8.11.1 Double Fault (#DF, vecteur 8)

  Une double faute se produit quand une exception est
  declenchee pendant le traitement d'une autre exception,
  et que la combinaison des deux exceptions ne peut pas
  etre geree en serie.

  Classification des exceptions pour le double fault :

  Classe 1 (Benigne) : #DE, #TS, #NP, #SS, #GP
  Classe 2 (Contributive) : #PF
  Classe 3 (Page Fault) : #PF

  Regles de combinaison :

  Premiere exception  Deuxieme exception  Resultat
  ------------------  ------------------  --------
  Benigne             Benigne             Traitement en serie
  Benigne             #PF                 Traitement en serie
  #PF                 Benigne             Traitement en serie
  #PF                 #PF                 Double Fault (#DF)
  Contributive        Contributive        Double Fault (#DF)

  En pratique, la double faute se produit le plus souvent
  quand une faute de page se produit pendant le traitement
  d'une autre faute de page (par exemple, si le handler de
  #PF lui-meme provoque un #PF).

  Le handler de #DF devrait utiliser une porte de tache
  (Task Gate) dans l'IDT, car cela garantit un changement
  de pile propre via le TSS, meme si la pile courante est
  corrompue.

  8.11.2 Triple Fault

  Si une exception se produit pendant le traitement d'une
  double faute, le processeur ne peut plus recuperer.
  Il effectue un RESET (triple fault) :
    - Le processeur revient a l'etat initial (mode reel)
    - Equivalent a un redemarrage materiel
    - Aucune interruption n'est generee

  La triple faute est souvent causee par :
    - Un handler de #DF invalide ou absent
    - Une IDT corrompue ou inaccessible
    - Un TSS invalide pour le handler de #DF

  Sequence de degradation :

    Exception initiale --> Exception pendant le traitement
                           --> Double Fault (#DF)
                               --> Exception pendant #DF
                                   --> TRIPLE FAULT (RESET)

  8.12 Exemple : mise en place d'un handler d'interruption
  -----------------------------------------------------------

  Voici un exemple complet d'installation d'un handler
  d'interruption pour le timer (IRQ 0) en mode protege.

  8.12.1 Definition du handler

    timer_count  DD 0              ; Compteur de ticks

    timer_handler:
      PUSH EAX                    ; Sauvegarder les registres utilises
      PUSH DS

      MOV AX, 10h                 ; Selecteur du segment de donnees
      MOV DS, AX                  ; Charger DS pour acceder aux donnees

      INC DWORD PTR [timer_count] ; Incrementer le compteur

      ; Envoyer EOI au PIC maitre
      MOV AL, 20h
      OUT 20h, AL

      POP DS                      ; Restaurer les registres
      POP EAX
      IRETD                       ; Retour d'interruption

  8.12.2 Installation dans l'IDT

  Chaque entree de l'IDT fait 8 octets. Pour le vecteur 20h
  (IRQ 0 remappee) :

    ; Construire le descripteur de porte d'interruption
    ; Selecteur de code = 08h, DPL = 0, type = 0Eh (32-bit Int Gate)

    MOV EAX, timer_handler           ; Adresse du handler
    MOV WORD PTR [idt_table + 20h*8 + 0], AX     ; Offset [15:0]
    MOV WORD PTR [idt_table + 20h*8 + 2], 08h    ; Selecteur code
    MOV BYTE PTR [idt_table + 20h*8 + 4], 00h    ; Reserve
    MOV BYTE PTR [idt_table + 20h*8 + 5], 8Eh    ; P=1,DPL=0,type=0Eh
    SHR EAX, 16
    MOV WORD PTR [idt_table + 20h*8 + 6], AX     ; Offset [31:16]

  L'octet d'attribut 8Eh se decompose :
    1 0 0 0 1 1 1 0
    P DPL 0 D 1 1 0
    P=1, DPL=00, type=0Eh (32-bit Interrupt Gate)

  8.12.3 Installation d'un handler d'exception (#GP)

    gp_handler:
      POP EAX                     ; Depiler le code d'erreur
      ; EAX contient le code d'erreur (selecteur fautif)

      ; ... afficher un message d'erreur ...
      ; ... terminer le processus ou faire un HALT ...

      ; Si on veut reprendre (rarement possible pour #GP) :
      ; IRETD

      ; En pratique, on arrete :
      HLT

  Installation dans l'IDT pour le vecteur 13 (#GP) :

    MOV EAX, gp_handler
    MOV WORD PTR [idt_table + 0Dh*8 + 0], AX
    MOV WORD PTR [idt_table + 0Dh*8 + 2], 08h
    MOV BYTE PTR [idt_table + 0Dh*8 + 4], 00h
    MOV BYTE PTR [idt_table + 0Dh*8 + 5], 8Fh    ; Trap Gate, P=1
    SHR EAX, 16
    MOV WORD PTR [idt_table + 0Dh*8 + 6], AX

  Note : 8Fh = porte de piege (Trap Gate, type 0Fh) car on
  veut que les interruptions materielles restent actives
  pendant le traitement de #GP.

  8.13 Resume des vecteurs d'interruption
  ------------------------------------------

  Vecteurs  Usage
  --------  ---------------------------------------------------
   0 - 16   Exceptions definies par Intel (reserves au processeur)
  17 - 31   Reserves par Intel (futurs processeurs)
  32 - 47   IRQ 0-15 (apres remappage typique 20h-2Fh)
  48 - 255  Disponibles pour les interruptions logicielles
            (appels systeme, services utilisateur, etc.)

  Registres de controle lies aux interruptions :

  EFLAGS.IF  : Autorise/bloque les interruptions masquables
  EFLAGS.TF  : Active le mode trace (exception #DB apres
               chaque instruction)
  EFLAGS.IOPL: Niveau de privilege requis pour CLI/STI/IN/OUT
  EFLAGS.NT  : Nested Task (pour IRET avec commutation de tache)
  EFLAGS.VM  : Virtual 8086 Mode

  IDTR       : Adresse et limite de l'IDT
  CR0.NE     : Numeric Error (FPU, exception #MF vs IRQ 13)

  En resume, le systeme d'interruptions du 80386 fournit un
  mecanisme complet et securise pour gerer les evenements
  asynchrones (interruptions materielles), les appels systeme
  (interruptions logicielles) et les conditions d'erreur
  (exceptions), avec des niveaux de protection, un changement
  automatique de pile et une table de dispatch configurable (IDT).

==============================================================================
9. OPTIMISATION POSSIBLE DU 80386
==============================================================================

  Cette section decrit les differents niveaux d'optimisation applicables
  lors de la generation de code assembleur ciblant le processeur 80386.
  Les niveaux sont classes par ordre croissant de complexite et d'impact
  sur les performances.

  9.1 Niveau 0 - Aucune optimisation (code naif)
  ------------------------------------------------

  Le code est genere de maniere directe, instruction par instruction,
  sans aucune tentative d'amelioration. Chaque operation du langage
  source est traduite independamment.

  Caracteristiques :
    - Chaque variable est lue depuis la memoire avant utilisation
    - Chaque resultat est ecrit en memoire immediatement
    - Pas de reutilisation des registres entre instructions
    - Les expressions sont evaluees via la pile (PUSH/POP)
    - Les branchements sont generes sans analyse de flot

  Exemple (a = b + c) :
      MOV  EAX, DWORD PTR [_b]      ; charger b
      PUSH EAX                       ; sauvegarder sur la pile
      MOV  EAX, DWORD PTR [_c]      ; charger c
      POP  EBX                       ; recuperer b
      ADD  EAX, EBX                  ; additionner
      MOV  DWORD PTR [_a], EAX      ; stocker resultat

  Avantage  : Simplicite de generation, debogage facile
  Inconvenient : Code volumineux et lent

  9.2 Niveau 1 - Optimisations de base (peephole)
  -------------------------------------------------

  Des motifs d'instructions redondants sont detectes et remplaces
  par des sequences plus courtes apres generation. C'est une
  optimisation locale, limitee a une fenetre de quelques instructions.

  Techniques appliquees :
    - Suppression des PUSH/POP consecutifs sur le meme registre
        PUSH EAX / POP EAX  ->  (supprime)
    - Remplacement de MOV reg, 0 par XOR reg, reg
        MOV EAX, 0  ->  XOR EAX, EAX
    - Suppression des MOV reg, reg identiques
        MOV EAX, EAX  ->  (supprime)
    - Remplacement de multiplications par des puissances de 2
        IMUL EAX, 4  ->  SHL EAX, 2
    - Simplification des branchements sur le prochain label
        JMP label_suivant / label_suivant:  ->  label_suivant:
    - Remplacement de IMUL EAX, 1 par rien
    - Remplacement de ADD EAX, 0 par rien
    - Utilisation de LEA pour des calculs d'adresses combines
        MOV EAX, [_b] / ADD EAX, [_c]  ->  LEA EAX, [EBX+ECX]

  Exemple ameliore (a = b + c) :
      MOV  EAX, DWORD PTR [_b]
      ADD  EAX, DWORD PTR [_c]
      MOV  DWORD PTR [_a], EAX

  Avantage  : Gains significatifs pour un cout d'implementation faible
  Inconvenient : Optimisation locale uniquement

  9.3 Niveau 2 - Allocation de registres
  ----------------------------------------

  Les variables les plus utilisees sont conservees dans des registres
  du processeur au lieu d'etre rechargees depuis la memoire a chaque
  acces. Cela reduit considerablement les acces memoire.

  Techniques appliquees :
    - Analyse de la duree de vie des variables (liveness analysis)
    - Attribution de registres aux variables locales actives
    - Registres disponibles pour l'allocation : EAX, EBX, ECX, EDX,
      ESI, EDI (EBP et ESP reserves pour la pile)
    - Spilling : si plus de variables actives que de registres, les
      moins utilisees sont releguees en memoire (pile)
    - Propagation des constantes dans les registres
    - Elimination des rechargements inutiles (un registre contenant
      deja la bonne valeur n'est pas recharge)

  Exemple avec allocation (boucle i = 0 a 99, somme += i) :
      XOR  ECX, ECX                  ; i = 0, dans ECX
      XOR  EAX, EAX                  ; somme = 0, dans EAX
    _boucle:
      ADD  EAX, ECX                  ; somme += i
      INC  ECX                       ; i++
      CMP  ECX, 100                  ; i < 100 ?
      JL   _boucle
      MOV  DWORD PTR [_somme], EAX   ; stocker resultat final

  Avantage  : Reduction massive des acces memoire
  Inconvenient : Complexite d'implementation (analyse de graphe)

  9.4 Niveau 3 - Optimisations de boucles
  -----------------------------------------

  Les boucles sont les zones les plus critiques pour les performances.
  Ce niveau applique des transformations specifiques aux boucles.

  Techniques appliquees :
    - Deplacement du code invariant hors de la boucle
      (loop-invariant code motion) : les calculs dont le resultat
      ne change pas a chaque iteration sont places avant la boucle
    - Reduction de force (strength reduction) : remplacement des
      multiplications par des additions incrementales
        i * 4  ->  compteur incremente de 4 a chaque iteration
    - Deroulement de boucles (loop unrolling) : le corps de la
      boucle est duplique pour reduire le cout des branchements
    - Inversion de boucle (loop inversion) : transformation d'une
      boucle WHILE en DO-WHILE precedee d'un test, pour eliminer
      un branchement par iteration
    - Utilisation de LOOP/REP pour les boucles simples
    - Fusion de boucles adjacentes ayant les memes bornes

  Exemple de deroulement (boucle de copie, 4 elements par iteration) :
      MOV  ECX, _count
      SHR  ECX, 2                    ; count / 4
    _boucle:
      MOV  EAX, [ESI]
      MOV  [EDI], EAX
      MOV  EAX, [ESI+4]
      MOV  [EDI+4], EAX
      MOV  EAX, [ESI+8]
      MOV  [EDI+8], EAX
      MOV  EAX, [ESI+12]
      MOV  [EDI+12], EAX
      ADD  ESI, 16
      ADD  EDI, 16
      DEC  ECX
      JNZ  _boucle

  Avantage  : Amelioration majeure des boucles intensives
  Inconvenient : Augmentation de la taille du code

  9.5 Niveau 4 - Optimisations globales et specifiques au 80386
  ---------------------------------------------------------------

  Ce niveau regroupe les optimisations avancees qui tirent parti
  des specificites du pipeline et du cache du 80386, ainsi que des
  analyses inter-procedurales.

  Techniques appliquees :

    Ordonnancement des instructions (instruction scheduling) :
      - Eviter les blocages de pipeline en intercalant des
        instructions independantes entre une lecture memoire et
        son utilisation
      - Le 80386 a un pipeline a 6 etapes (Bus Interface, Code
        Prefetch, Instruction Decode, Execution, Segment, Page)
      - Eviter les dependances de donnees consecutives

    Alignement des donnees et du code :
      - Aligner les donnees 32 bits sur des adresses multiples de 4
        (ALIGN 4) pour eviter les penalites d'acces non alignes
      - Aligner les cibles de branchement sur des adresses paires
        ou multiples de 4 pour optimiser le prefetch
      - Aligner le debut des procedures sur des limites de 16 octets

    Utilisation optimale de LEA :
      - LEA permet de calculer des expressions complexes en une
        instruction sans affecter les drapeaux
      - LEA EAX, [EBX+ECX*4+8] combine addition et multiplication

    Optimisations specifiques au 80386 :
      - Preferer MOVZX/MOVSX aux sequences XOR+MOV pour les
        extensions de taille (8 vers 32 bits, 16 vers 32 bits)
      - Utiliser les modes d'adressage bases+index*echelle
        (SIB byte) pour les acces aux tableaux
      - Utiliser CDQ au lieu de XOR EDX,EDX / OR EAX,EAX / ...
        avant IDIV pour l'extension de signe
      - Preferer TEST reg, reg a CMP reg, 0 pour tester si nul
      - Preferer INC/DEC a ADD/SUB 1 (plus compact, 1 octet)
      - Eviter les prefixes de taille d'operande (melange 16/32
        bits) qui causent des penalites de decodage

    Optimisations inter-procedurales :
      - Inlining de petites fonctions (expansion en ligne)
      - Propagation des constantes a travers les appels
      - Elimination du code mort (dead code elimination)
      - Optimisation des appels terminaux (tail call optimization)

  Avantage  : Performance maximale pour le 80386
  Inconvenient : Complexite elevee, temps de compilation accru

==============================================================================
10. DIRECTIVES ASSEMBLEUR (MASM/TASM)
==============================================================================

  Les directives assembleur ne sont pas des instructions executees par
  le processeur. Ce sont des commandes interpretees par l'assembleur
  (MASM, TASM, WASM, etc.) pour controler l'assemblage : organisation
  de la memoire, definition de donnees, macros, assemblage conditionnel,
  etc. Cette section couvre les directives les plus courantes pour le
  mode 32 bits (80386+).

  10.1 Directives de processeur
  -------------------------------

  Ces directives indiquent a l'assembleur quel jeu d'instructions
  est autorise. Elles doivent apparaitre en debut de fichier.

    .8086           ; Jeu d'instructions 8086 uniquement (defaut)
    .186            ; Ajout des instructions du 80186
    .286            ; Ajout des instructions du 80286
    .286P           ; 80286, instructions privilegiees autorisees
    .386            ; Ajout des instructions du 80386
    .386P           ; 80386, instructions privilegiees autorisees
    .486            ; Ajout des instructions du 80486
    .486P           ; 80486, instructions privilegiees autorisees
    .586            ; Ajout des instructions du Pentium
    .586P           ; Pentium, instructions privilegiees autorisees
    .686            ; Ajout des instructions du Pentium Pro
    .686P           ; Pentium Pro, instructions privilegiees autorisees

  Directives de coprocesseur :

    .8087           ; Instructions du coprocesseur 8087
    .287            ; Instructions du coprocesseur 80287
    .387            ; Instructions du coprocesseur 80387
    .NO87           ; Desactive les instructions de coprocesseur

  Note : .386 et superieur active automatiquement les registres 32 bits
  (EAX, EBX, etc.) et les modes d'adressage etendus du 80386.

  Sans suffixe P, seules les instructions non privilegiees sont
  autorisees. Avec le suffixe P, les instructions privilegiees
  comme LGDT, LIDT, MOV CR0, etc. sont egalement acceptees.

  10.2 Directives de modele memoire (.MODEL)
  --------------------------------------------

  La directive .MODEL definit le modele memoire utilise par le
  programme. Elle controle la taille par defaut des pointeurs
  (NEAR ou FAR) et la convention d'appel.

  Syntaxe :

    .MODEL modele [, convention] [, type_pile]

  Modeles memoire :

  Modele      Segments de  Segments de   Pointeurs
              code         donnees       par defaut
  ----------  -----------  -----------   --------------------
  TINY        1            1 (=code)     NEAR (COM, < 64Ko)
  SMALL       1            1             NEAR code, NEAR data
  MEDIUM      Plusieurs    1             FAR code, NEAR data
  COMPACT     1            Plusieurs     NEAR code, FAR data
  LARGE       Plusieurs    Plusieurs     FAR code, FAR data
  HUGE        Plusieurs    Plusieurs     FAR, tableaux > 64Ko
  FLAT        1 (32 bits)  1 (32 bits)   NEAR 32 bits (0-4Go)

  Le modele FLAT est le plus utilise pour la programmation 32 bits
  (Win32, Linux 32 bits). Il utilise un espace d'adressage plat
  de 4 Go avec des pointeurs NEAR 32 bits.

  Conventions d'appel :

    C           Convention C : parametres empiles de droite a gauche,
                l'appelant nettoie la pile. Noms prefixes par _.
    STDCALL     Convention Windows : parametres de droite a gauche,
                l'appele nettoie la pile (RET n). Noms decores @.
    PASCAL      Convention Pascal : parametres de gauche a droite,
                l'appele nettoie la pile.
    BASIC       Similaire a Pascal.
    FORTRAN     Similaire a Pascal.
    SYSCALL     Convention des appels systeme OS/2.

  Exemples :

    .MODEL SMALL, C              ; Mode 16 bits, convention C
    .MODEL FLAT, STDCALL         ; Mode 32 bits Win32
    .MODEL FLAT, C               ; Mode 32 bits, convention C

  10.3 Directives de segment simplifiees
  -----------------------------------------

  Les directives simplifiees creent automatiquement les segments
  standards. Elles necessitent une directive .MODEL prealable.

    .CODE [nom]     ; Ouvre le segment de code (ou un segment
                    ; de code nomme en modele MEDIUM/LARGE)
    .DATA           ; Ouvre le segment de donnees initialisees
    .DATA?          ; Ouvre le segment de donnees non initialisees
    .CONST          ; Ouvre le segment de constantes (lecture seule)
    .STACK [taille] ; Definit le segment de pile
                    ; (defaut : 1024 octets)

  Exemple typique (programme 32 bits Win32) :

    .386
    .MODEL FLAT, STDCALL
    OPTION CASEMAP:NONE

    .STACK 4096

    .DATA
    message  DB 'Hello, World!', 0
    nombre   DD 42

    .DATA?
    buffer   DB 256 DUP(?)

    .CODE
    main PROC
      ; ... code du programme ...
      RET
    main ENDP
    END main

  10.4 Directives de segment completes
  ---------------------------------------

  Les directives completes offrent un controle total sur la
  definition des segments.

  Syntaxe :

    nom SEGMENT [align] [combine] [use] ['class']
      ; ... contenu du segment ...
    nom ENDS

  Parametres :

  align (alignement) :
    BYTE      ; Aucun alignement
    WORD      ; Aligne sur 2 octets
    DWORD     ; Aligne sur 4 octets
    PARA      ; Aligne sur 16 octets (paragraphe, defaut)
    PAGE      ; Aligne sur 256 octets

  combine (combinaison) :
    PRIVATE   ; Le segment n'est pas combine (defaut)
    PUBLIC    ; Les segments de meme nom sont concatenes
    COMMON    ; Les segments de meme nom se superposent
    STACK     ; Concatenation pour le segment de pile
    AT adresse; Le segment est place a une adresse fixe

  use (mode) :
    USE16     ; Segment 16 bits (defaut si .286 ou inferieur)
    USE32     ; Segment 32 bits (defaut si .386 ou superieur)

  'class' : Chaine de classification pour le linker.
    'CODE'    ; Segment de code
    'DATA'    ; Segment de donnees
    'STACK'   ; Segment de pile
    'BSS'     ; Segment de donnees non initialisees

  Exemple (mode 16 bits) :

    _TEXT SEGMENT PARA PUBLIC USE16 'CODE'
      ; ... code ...
    _TEXT ENDS

    _DATA SEGMENT PARA PUBLIC USE16 'DATA'
      ; ... donnees ...
    _DATA ENDS

    STACK SEGMENT PARA STACK USE16 'STACK'
      DB 256 DUP(?)
    STACK ENDS

  Exemple (mode 32 bits) :

    _TEXT SEGMENT DWORD PUBLIC USE32 'CODE'
      ; ... code 32 bits ...
    _TEXT ENDS

    _DATA SEGMENT DWORD PUBLIC USE32 'DATA'
      ; ... donnees 32 bits ...
    _DATA ENDS

  10.5 Directives ASSUME et GROUP
  ---------------------------------

  ASSUME indique a l'assembleur quel registre de segment pointe
  vers quel segment. Essentiel en mode 16 bits segmente, moins
  utile en mode FLAT 32 bits.

    ASSUME CS:_TEXT, DS:_DATA, SS:STACK
    ASSUME ES:NOTHING           ; Pas d'association pour ES
    ASSUME NOTHING              ; Retire toutes les associations

  GROUP regroupe plusieurs segments sous un meme nom de groupe
  pour qu'ils partagent un registre de segment :

    DGROUP GROUP _DATA, _BSS, STACK

  En mode FLAT (32 bits), ASSUME est rarement necessaire car
  tous les segments partagent la meme base.

  10.6 Directives de donnees
  ----------------------------

  10.6.1 Definition de variables

  Les directives de donnees reservent et initialisent de la memoire.

    DB   (Define Byte)       1 octet   (8 bits)
    DW   (Define Word)       2 octets  (16 bits)
    DD   (Define Doubleword) 4 octets  (32 bits)
    DF   (Define Fword)      6 octets  (48 bits, pointeur far 32)
    DQ   (Define Quadword)   8 octets  (64 bits)
    DT   (Define Tenbyte)   10 octets  (80 bits, extended float)

  Syntaxe :

    [nom]  directive  valeur [, valeur, ...]

  Exemples :

    octet1    DB  42              ; Un octet = 42
    mot1      DW  1234h           ; Un mot = 1234h
    dword1    DD  12345678h       ; Un double mot
    qword1    DQ  0               ; Un quadruple mot = 0
    float1    DD  3.14            ; Flottant simple precision
    double1   DQ  2.718281828     ; Flottant double precision
    extend1   DT  1.0             ; Flottant etendu 80 bits

  Chaines de caracteres :

    msg1      DB  'Hello', 0      ; Chaine terminee par un zero
    msg2      DB  'A', 'B', 'C'   ; Equivalent a DB 41h, 42h, 43h
    msg3      DB  "Guillemets"    ; Guillemets doubles aussi acceptes
    crlf      DB  0Dh, 0Ah       ; Retour chariot + saut de ligne

  Tableaux et repetitions (DUP) :

    tab1      DB  10 DUP(0)       ; 10 octets initialises a 0
    tab2      DW  5  DUP(?)       ; 5 mots non initialises
    tab3      DD  3  DUP(1)       ; 3 double mots = 1, 1, 1
    matrice   DB  4  DUP(8 DUP(0)); Matrice 4x8 d'octets
    buf256    DB  256 DUP(?)      ; Tampon de 256 octets

  Valeur speciale ? : donnee non initialisee. L'assembleur
  reserve l'espace sans ecrire de valeur. Utilisee dans .DATA?
  pour les variables BSS.

  10.6.2 Constantes symboliques

  EQU - Constante textuelle ou numerique (non modifiable) :

    MAX_SIZE    EQU  256          ; Constante numerique
    CR          EQU  0Dh         ; Retour chariot
    LF          EQU  0Ah         ; Saut de ligne
    GREETING    EQU  <'Hello'>   ; Constante textuelle
    PSIZE       EQU  <DWORD PTR> ; Alias de type

  = - Constante numerique (modifiable, MASM seulement) :

    compteur    =    0            ; Valeur initiale
    compteur    =    compteur + 1 ; Reassignation autorisee

  Difference entre EQU et = :
    - EQU ne peut pas etre redefini. = peut l'etre.
    - EQU accepte du texte et des nombres. = uniquement des nombres.
    - EQU est supporte par MASM et TASM. = est MASM seulement.

  10.6.3 Operateurs de type et de taille

    BYTE PTR [addr]   ; Force l'acces en 8 bits
    WORD PTR [addr]   ; Force l'acces en 16 bits
    DWORD PTR [addr]  ; Force l'acces en 32 bits
    QWORD PTR [addr]  ; Force l'acces en 64 bits
    FWORD PTR [addr]  ; Force l'acces en 48 bits
    TBYTE PTR [addr]  ; Force l'acces en 80 bits

    TYPE variable     ; Retourne la taille en octets de la variable
    LENGTHOF tableau  ; Nombre d'elements du tableau (MASM 6+)
    SIZEOF tableau    ; Taille totale en octets (MASM 6+)
    LENGTH tableau    ; Nombre d'elements (ancienne syntaxe)
    SIZE tableau      ; Taille totale (ancienne syntaxe)

    OFFSET variable   ; Adresse (offset) de la variable
    SEG variable      ; Segment de la variable (mode 16 bits)
    ADDR variable     ; Adresse pour INVOKE (MASM)
    THIS type         ; Cree un alias a la position courante

  Exemples :

    MOV AL, BYTE PTR [ESI]       ; Lire un octet
    MOV EAX, DWORD PTR [ESI]    ; Lire un double mot
    MOV ECX, LENGTHOF tableau   ; Nombre d'elements
    LEA EAX, [variable]         ; Charger l'adresse effective
    PUSH OFFSET message         ; Empiler l'adresse du message

  10.7 Directives de procedures
  --------------------------------

  10.7.1 PROC et ENDP

  PROC definit une procedure avec gestion automatique du prologue,
  de l'epilogue et des parametres (MASM 6+).

  Syntaxe simple :

    nom PROC [distance]
      ; ... corps de la procedure ...
      RET
    nom ENDP

  Distance :
    NEAR    ; Appel intra-segment (defaut en FLAT)
    FAR     ; Appel inter-segment

  Syntaxe etendue (MASM 6+) avec parametres :

    nom PROC [distance] [convention] [USES regs] [, params]
      ; ... corps ...
      RET
    nom ENDP

  USES : liste des registres a sauvegarder automatiquement
  (PUSH au debut, POP a la fin).

  Parametres : nom:type declares apres le nom de la procedure.

  Exemple :

    Addition PROC STDCALL USES EBX ECX, a:DWORD, b:DWORD
      MOV EAX, a          ; Acces au parametre a = [EBP+8]
      ADD EAX, b          ; Acces au parametre b = [EBP+12]
      RET                 ; Retour avec nettoyage de la pile
    Addition ENDP

  L'assembleur genere automatiquement :
    - PUSH EBP / MOV EBP, ESP (prologue)
    - PUSH EBX / PUSH ECX (sauvegarde USES)
    - POP ECX / POP EBX (restauration USES)
    - MOV ESP, EBP / POP EBP (epilogue)
    - RET 8 (nettoyage de 2 parametres DWORD en STDCALL)

  10.7.2 LOCAL

  LOCAL declare des variables locales sur la pile a l'interieur
  d'une procedure PROC.

    nom PROC ...
      LOCAL var1:DWORD           ; 1 variable locale de 4 octets
      LOCAL buf[256]:BYTE        ; Tableau local de 256 octets
      LOCAL x:DWORD, y:DWORD    ; Plusieurs variables locales
      ; ...
    nom ENDP

  L'assembleur genere SUB ESP, n dans le prologue pour reserver
  l'espace des variables locales et les rend accessibles par
  nom (traduites en [EBP-offset]).

  10.7.3 INVOKE (MASM seulement)

  INVOKE appelle une procedure avec verification automatique
  du nombre et du type des parametres, et empile les arguments
  dans l'ordre correct selon la convention d'appel.

  Syntaxe :

    INVOKE nom_procedure, arg1, arg2, ...

  Necessite une declaration PROTO :

    nom_procedure PROTO [convention] :type1, :type2, ...

  Exemple :

    ; Declaration du prototype
    MessageBoxA PROTO STDCALL :DWORD, :DWORD, :DWORD, :DWORD

    ; Appel
    INVOKE MessageBoxA, 0, ADDR message, ADDR titre, 0

  INVOKE genere automatiquement :
    - PUSH des arguments dans l'ordre inverse (convention C/STDCALL)
    - CALL nom_procedure
    - ADD ESP, n si convention C (nettoyage par l'appelant)

  10.7.4 PROTO

  PROTO declare le prototype d'une procedure (MASM 6+).

    nom PROTO [convention] [:type1 [, :type2, ...]]

  Exemples :

    Addition PROTO STDCALL :DWORD, :DWORD
    ExitProcess PROTO STDCALL :DWORD
    printf PROTO C :DWORD, :VARARG     ; Nombre variable d'arguments

  PROTO permet a INVOKE de verifier les types des arguments
  et a l'assembleur de generer le bon code d'appel.

  10.8 Directives de declarations externes et publiques
  -------------------------------------------------------

  PUBLIC - Rend un symbole visible pour le linker :

    PUBLIC nom_symbole
    PUBLIC _main, _data_table

  EXTRN / EXTERN - Declare un symbole defini dans un autre module :

    EXTRN symbole:type           ; Syntaxe ancienne (MASM 5)
    EXTERN symbole:type          ; Syntaxe MASM 6+ (alias)
    EXTERNDEF symbole:type       ; Definition+declaration (MASM 6+)

  Types pour EXTRN/EXTERN :
    BYTE, WORD, DWORD, QWORD     ; Variables
    NEAR, FAR                     ; Procedures
    PROC                          ; Procedure (MASM 6+)
    ABS                           ; Constante absolue

  Exemples :

    EXTRN _printf:NEAR            ; Fonction C externe
    EXTRN ExitProcess@4:NEAR      ; Fonction Win32
    EXTERN MessageBoxA@16:NEAR    ; Fonction Win32 decoree
    PUBLIC _main                  ; Exporter le point d'entree

  EXTERNDEF (MASM 6+) : combine PUBLIC et EXTERN. Si le symbole
  est defini dans le module courant, il est exporte. Sinon, il
  est importe. Utile dans les fichiers d'en-tete .INC.

    EXTERNDEF compteur:DWORD     ; Export si defini, import sinon

  10.9 Directives d'inclusion et de liaison
  --------------------------------------------

  INCLUDE - Inclut un fichier source dans le fichier courant :

    INCLUDE fichier.inc           ; Insere le contenu du fichier
    INCLUDE windows.inc           ; En-tetes Win32 pour MASM
    INCLUDE macros.inc            ; Fichier de macros personnalisees

  INCLUDELIB - Indique une bibliotheque a lier :

    INCLUDELIB kernel32.lib       ; Liaison avec kernel32.dll
    INCLUDELIB user32.lib         ; Liaison avec user32.dll
    INCLUDELIB msvcrt.lib         ; Liaison avec la CRT Microsoft

  INCLUDELIB ne lie pas directement la bibliotheque, mais insere
  une directive dans le fichier objet pour que le linker la trouve.

  COMMENT - Commentaire multi-lignes :

    COMMENT * 
    Ceci est un commentaire
    sur plusieurs lignes.
    *

  Le delimiteur est le premier caractere non-blanc apres COMMENT.
  Le commentaire se termine a la prochaine occurrence de ce caractere.

  10.10 Directives d'assemblage conditionnel
  ---------------------------------------------

  L'assemblage conditionnel permet d'inclure ou d'exclure du code
  selon des conditions evaluees au moment de l'assemblage.

  10.10.1 Conditions sur des expressions

    IF expression               ; Si expression <> 0
      ; ... code assemble ...
    ELSEIF expression2          ; Sinon si expression2 <> 0
      ; ... code alternatif ...
    ELSE                        ; Sinon
      ; ... code par defaut ...
    ENDIF                       ; Fin de la condition

  Exemples :

    DEBUG = 1

    IF DEBUG
      CALL AfficherRegistres    ; Code de debug
    ENDIF

    IF ($ - debut) GT 100h
      ; Le code depasse 256 octets
    ENDIF

  10.10.2 Conditions sur des symboles

    IFDEF symbole               ; Si le symbole est defini
    IFNDEF symbole              ; Si le symbole n'est pas defini
    IFB <argument>              ; Si l'argument est vide (blank)
    IFNB <argument>             ; Si l'argument n'est pas vide
    IFIDN <s1>, <s2>           ; Si les chaines sont identiques
    IFIDNI <s1>, <s2>          ; Idem, insensible a la casse
    IFDIF <s1>, <s2>           ; Si les chaines sont differentes
    IFDIFI <s1>, <s2>          ; Idem, insensible a la casse

  Exemples :

    IFDEF _WIN32
      .MODEL FLAT, STDCALL
    ELSE
      .MODEL SMALL, C
    ENDIF

    IFNDEF MAX_BUF
      MAX_BUF EQU 256
    ENDIF

  10.10.3 Conditions sur les types (MASM seulement)

    IF1                         ; Passe 1 de l'assemblage
    IF2                         ; Passe 2 de l'assemblage
    IFE expression              ; Si expression = 0

  10.11 Directives de macro
  ----------------------------

  10.11.1 Definition et appel

  MACRO/ENDM definit une macro-instruction. A chaque appel,
  le corps de la macro est expanse avec substitution des
  parametres.

  Syntaxe :

    nom MACRO [param1 [, param2, ...]]
      ; ... corps de la macro ...
    ENDM

  Exemple :

    ; Definition de la macro
    PushAll MACRO
      PUSH EAX
      PUSH EBX
      PUSH ECX
      PUSH EDX
    ENDM

    ; Utilisation
    PushAll                      ; Expanse les 4 PUSH

  Macro avec parametres :

    PrintStr MACRO addr
      PUSH OFFSET addr
      CALL _puts
      ADD ESP, 4
    ENDM

    ; Utilisation
    PrintStr message             ; Affiche la chaine 'message'

  Macro avec parametres par defaut :

    AllocBuf MACRO taille:=<256>
      SUB ESP, taille
    ENDM

    AllocBuf                     ; SUB ESP, 256
    AllocBuf 512                 ; SUB ESP, 512

  10.11.2 Operateurs de macro

    &           ; Operateur de substitution (colle le texte)
    <>          ; Litteraux de texte
    !           ; Caractere litteral (echappe les caracteres speciaux)
    %          ; Expression evaluee comme texte
    ;;          ; Commentaire de macro (non expanse)

  Exemple avec & :

    DeclareVar MACRO nom, taille
      _&nom  DB taille DUP(0)   ;; Genere _buffer DB 64 DUP(0)
    ENDM

    DeclareVar buffer, 64

  10.11.3 Directives dans les macros

    LOCAL label1, label2         ; Etiquettes locales a la macro
                                 ; (renommees automatiquement a
                                 ; chaque expansion pour eviter
                                 ; les conflits)

    EXITM [valeur]               ; Sortie prematuree de la macro
                                 ; (retourne une valeur optionnelle)

    .ERR [message]               ; Genere une erreur d'assemblage
    .ERRNZ expression            ; Erreur si expression <> 0
    .ERRE expression             ; Erreur si expression = 0
    .ERRB <texte>                ; Erreur si texte est vide
    .ERRNB <texte>               ; Erreur si texte n'est pas vide
    .ERRIDN <s1>, <s2>          ; Erreur si chaines identiques
    .ERRDIF <s1>, <s2>          ; Erreur si chaines differentes

  Exemple avec LOCAL :

    CompareAndJump MACRO val, dest
      LOCAL skip
      CMP EAX, val
      JNE skip
      JMP dest
    skip:
    ENDM

  10.11.4 Macros de repetition

    REPT n                       ; Repete le bloc n fois
      ; ... corps ...
    ENDM

    IRP param, <val1, val2, ...> ; Repete pour chaque valeur
      ; ... corps avec param ...
    ENDM

    IRPC param, chaine           ; Repete pour chaque caractere
      ; ... corps avec param ...
    ENDM

  Exemples :

    ; Generer 10 NOP
    REPT 10
      NOP
    ENDM

    ; Empiler plusieurs registres
    IRP reg, <EAX, EBX, ECX, EDX>
      PUSH reg
    ENDM

    ; Generer des DB pour chaque caractere
    IRPC c, HELLO
      DB '&c'
    ENDM

  10.11.5 Macros fonctions (MASM seulement)

  Les macros fonctions retournent une valeur via EXITM :

    IsEven MACRO n
      IF (n) MOD 2 EQ 0
        EXITM <1>                ;; Retourne 1 (pair)
      ELSE
        EXITM <0>                ;; Retourne 0 (impair)
      ENDIF
    ENDM

    IF IsEven(42)
      DB "Pair"
    ENDIF

  10.12 Directives d'alignement et d'organisation
  ---------------------------------------------------

  ALIGN n - Aligne la position courante sur une frontiere de
  n octets (n doit etre une puissance de 2) :

    ALIGN 4                      ; Aligne sur 4 octets (DWORD)
    ALIGN 16                     ; Aligne sur 16 octets (paragraphe)
    ALIGN 256                    ; Aligne sur 256 octets (page)

  L'assembleur insere des octets de remplissage (NOP dans le code,
  00h dans les donnees) pour atteindre l'alignement demande.

  EVEN - Aligne sur une frontiere de mot (2 octets) :

    EVEN                         ; Equivalent a ALIGN 2

  ORG expression - Fixe le compteur de position a l'adresse
  specifiee :

    ORG 0100h                    ; Debut a l'offset 100h (fichiers COM)
    ORG $ + 10                   ; Saute 10 octets

  $ - Symbole special representant la position courante du
  compteur de localisation :

    msg      DB  'Hello', 0
    msg_len  EQU $ - msg         ; Longueur de la chaine = 6

    tableau  DD  1, 2, 3, 4, 5
    nb_elem  EQU ($ - tableau) / 4  ; Nombre d'elements = 5

  $$ - Adresse de debut du segment courant.

  10.13 Directives OPTION (MASM seulement)
  -------------------------------------------

  OPTION configure divers comportements de l'assembleur.

    OPTION CASEMAP:NONE          ; Sensible a la casse (majuscules
                                 ; et minuscules distinguees)
    OPTION CASEMAP:ALL           ; Insensible a la casse
    OPTION CASEMAP:NOTPUBLIC     ; Sensible sauf pour PUBLIC/EXTRN
    OPTION DOTNAME               ; Autorise les points dans les noms
    OPTION PROC:PRIVATE          ; Les procedures sont privees
                                 ; par defaut
    OPTION PROLOGUE:nom          ; Utilise un prologue personnalise
    OPTION EPILOGUE:nom          ; Utilise un epilogue personnalise
    OPTION LANGUAGE:convention   ; Convention d'appel par defaut
    OPTION SCOPED                ; Etiquettes locales aux procedures
    OPTION NOSCOPED              ; Etiquettes globales (defaut)
    OPTION LJMP                  ; Active les sauts longs automatiques
    OPTION NOLJMP                ; Desactive les sauts longs auto

  La combinaison la plus courante en Win32 :

    .386
    .MODEL FLAT, STDCALL
    OPTION CASEMAP:NONE

  10.14 Directives de base de numeration
  -----------------------------------------

  .RADIX base - Change la base par defaut des nombres :

    .RADIX 16                    ; Base hexadecimale par defaut
    .RADIX 10                    ; Base decimale (defaut)
    .RADIX 8                     ; Base octale
    .RADIX 2                     ; Base binaire

  Sans .RADIX, la base par defaut est decimale (10).

  Suffixes explicites (toujours valides quelle que soit la base) :

    42              ; Decimal (ou depend de .RADIX)
    42d             ; Decimal explicite
    2Ah             ; Hexadecimal (suffixe h)
    0FFh            ; Hexadecimal (doit commencer par un chiffre)
    52o             ; Octal (suffixe o ou q)
    52q             ; Octal (suffixe q)
    101010b         ; Binaire (suffixe b)

  Attention : avec .RADIX 16, les nombres comme 0FFh doivent
  etre ecrits 0FFt ou avec le suffixe t pour eviter l'ambiguite
  avec le suffixe h qui devient un chiffre hexadecimal.

  10.15 Directive END
  ---------------------

  END marque la fin du fichier source et specifie optionnellement
  le point d'entree du programme.

    END [point_entree]

  Exemples :

    END                          ; Fin du module (pas de point d'entree)
    END main                     ; Fin du programme, entree a 'main'
    END _start                   ; Point d'entree _start

  Le point d'entree n'est specifie que dans le module principal.
  Les modules secondaires utilisent END sans argument.

  10.16 Directives de structures et unions
  -------------------------------------------

  STRUCT/STRUC et UNION permettent de definir des types composes.

  10.16.1 Structures

    nom STRUCT
      champ1  type  valeur_initiale
      champ2  type  valeur_initiale
      ; ...
    nom ENDS

  Exemple :

    POINT STRUCT
      x  DD  0
      y  DD  0
    POINT ENDS

    RECT STRUCT
      gauche  DD  0
      haut    DD  0
      droite  DD  0
      bas     DD  0
    RECT ENDS

  Utilisation :

    ; Declaration d'une variable de type POINT
    MonPoint  POINT  <10, 20>         ; x=10, y=20
    MonPoint2 POINT  <>               ; Valeurs par defaut (0, 0)
    MonPoint3 POINT  <, 30>           ; x=defaut(0), y=30

    ; Acces aux champs
    MOV EAX, MonPoint.x               ; Lire x
    MOV MonPoint.y, 50                ; Ecrire y

    ; Acces via un registre pointeur
    LEA ESI, MonPoint
    MOV EAX, [ESI].POINT.x           ; Lire x via pointeur
    MOV [ESI].POINT.y, 100           ; Ecrire y via pointeur

    ; Taille de la structure
    MOV ECX, SIZEOF POINT             ; ECX = 8 (2 * 4 octets)

  10.16.2 Unions

    nom UNION
      champ1  type  valeur_initiale
      champ2  type  valeur_initiale
    nom ENDS

  Tous les champs d'une union occupent la meme zone memoire.
  La taille de l'union est la taille du plus grand champ.

  Exemple :

    VARIANT UNION
      b  DB  ?
      w  DW  ?
      d  DD  ?
    VARIANT ENDS
    ; SIZEOF VARIANT = 4 (taille du plus grand champ DD)

  10.16.3 Structures imbriquees

    FENETRE STRUCT
      pos     POINT   <>
      taille  POINT   <>
      titre   DD      0
    FENETRE ENDS

    fen1  FENETRE  <<10,20>, <640,480>, OFFSET str_titre>

  10.17 Directives de listing et de debug
  -------------------------------------------

  Ces directives controlent le fichier listing (.LST) genere
  par l'assembleur.

    .LIST            ; Active la generation du listing
    .NOLIST          ; Desactive la generation du listing
    .XLIST           ; Desactive completement le listing
    .LISTALL         ; Liste tout, y compris les macros
    .LISTMACRO       ; Liste le corps des macros expandees
    .LISTMACROALL    ; Liste toutes les expansions de macros

    TITLE texte      ; Titre en haut de chaque page du listing
    SUBTITLE texte   ; Sous-titre
    PAGE [lignes, colonnes] ; Dimensions de la page du listing
    PAGE +           ; Nouvelle page dans le listing

    .CREF            ; Active la table des references croisees
    .NOCREF          ; Desactive les references croisees

  10.18 Differences entre MASM et TASM
  ----------------------------------------

  MASM (Microsoft Macro Assembler) et TASM (Turbo Assembler de
  Borland) partagent la majorite de la syntaxe, mais presentent
  quelques differences :

  10.18.1 Mode IDEAL (TASM seulement)

  TASM supporte un mode alternatif appele IDEAL qui clarifie
  certaines ambiguites de la syntaxe MASM :

    IDEAL                         ; Active le mode IDEAL
    MASM                          ; Retour au mode MASM (defaut)

  Differences en mode IDEAL :
    - Les crochets [] sont obligatoires pour l'acces memoire
    - Les crochets signifient toujours "contenu a l'adresse"
    - Les types sont ecrits avant l'operande : [DWORD PTR var]
      devient [DWORD var]
    - OFFSET est obligatoire pour obtenir l'adresse d'une variable
    - Les structures utilisent la syntaxe . pour les champs

  Exemple (MASM vs IDEAL) :

    ; MASM :
    MOV EAX, variable             ; Acces memoire
    MOV EAX, OFFSET variable      ; Adresse
    MOV EAX, DWORD PTR [ESI]     ; Acces via pointeur

    ; IDEAL :
    MOV EAX, [variable]           ; Acces memoire
    MOV EAX, OFFSET variable      ; Adresse
    MOV EAX, [DWORD ESI]         ; Acces via pointeur

  10.18.2 Directives specifiques a TASM

    IDEAL           ; Mode de syntaxe IDEAL
    MASM            ; Mode de syntaxe MASM (defaut)
    QUIRKS          ; Active les compatibilites MASM (bugs inclus)
    SMART           ; Optimisation automatique des sauts
    NOSMART         ; Desactive l'optimisation des sauts
    JUMPS           ; Genere des sauts longs si necessaire
    NOJUMPS         ; Pas de sauts longs automatiques
    LOCALS prefix   ; Active les etiquettes locales (@@)
    NOLOCALS        ; Desactive les etiquettes locales
    WARN            ; Active tous les avertissements
    NOWARN          ; Desactive les avertissements
    EMUL            ; Active l'emulation du coprocesseur
    NOEMUL          ; Pas d'emulation

  10.18.3 Directives specifiques a MASM 6+

    OPTION          ; Configuration de l'assembleur (voir 10.13)
    INVOKE          ; Appel de procedure avec verification de types
    PROTO           ; Prototype de procedure
    EXTERNDEF       ; Declaration/definition externe
    .WHILE          ; Boucle while de haut niveau
    .ENDW           ; Fin de la boucle while
    .REPEAT         ; Boucle repeat
    .UNTIL condition; Fin de la boucle repeat (condition de sortie)
    .BREAK          ; Sortie de boucle
    .CONTINUE       ; Iteration suivante
    .IF condition   ; Condition de haut niveau
    .ELSEIF condition
    .ELSE
    .ENDIF

  Exemple de constructions de haut niveau (MASM 6+) :

    .IF EAX == 0
      CALL GererErreur
    .ELSEIF EAX > 100
      MOV EAX, 100
    .ELSE
      INC EAX
    .ENDIF

    .WHILE ECX > 0
      LODSB
      CALL TraiterOctet
      DEC ECX
    .ENDW

    .REPEAT
      CALL LireOctet
    .UNTIL AL == 0

  L'assembleur traduit ces constructions en CMP/Jcc et JMP
  standards. Elles n'ajoutent aucun surcout par rapport au
  code ecrit manuellement.

11. EXEMPLES DE CODE
==============================================================================

  Cette section presente des exemples complets de programmes assembleur
  80386, illustrant les concepts decrits dans les sections precedentes.
  Tous les exemples utilisent la syntaxe MASM en mode 32 bits FLAT.

  11.1 Hello World (Win32 Console)
  ----------------------------------

  Programme minimal affichant "Hello, World!" sur la console Windows
  en utilisant les API WriteConsoleA et ExitProcess.

    .386
    .MODEL FLAT, STDCALL
    OPTION CASEMAP:NONE

    ; Declarations externes des fonctions Win32
    EXTRN GetStdHandle@4:NEAR
    EXTRN WriteConsoleA@20:NEAR
    EXTRN ExitProcess@4:NEAR

    .STACK 4096

    .DATA
    message   DB  'Hello, World!', 0Dh, 0Ah, 0
    msg_len   EQU $ - message - 1     ; Longueur sans le zero final

    .DATA?
    hStdOut   DD  ?                    ; Handle de la sortie standard
    written   DD  ?                    ; Nombre de caracteres ecrits

    .CODE
    main PROC
      ; Obtenir le handle de la sortie standard
      PUSH -11                         ; STD_OUTPUT_HANDLE
      CALL GetStdHandle@4
      MOV [hStdOut], EAX

      ; Ecrire le message
      PUSH 0                           ; Parametre reserve (NULL)
      PUSH OFFSET written              ; Adresse du compteur
      PUSH msg_len                     ; Nombre de caracteres
      PUSH OFFSET message              ; Adresse du message
      PUSH [hStdOut]                   ; Handle de sortie
      CALL WriteConsoleA@20

      ; Terminer le programme
      PUSH 0                           ; Code de sortie 0
      CALL ExitProcess@4
    main ENDP
    END main

  11.2 Calcul factoriel (iteratif et recursif)
  -----------------------------------------------

  11.2.1 Version iterative

    ; Calcule n! de maniere iterative
    ; Entree : ECX = n
    ; Sortie : EAX = n!
    Factoriel_Iter PROC
      MOV EAX, 1                 ; Resultat = 1
      CMP ECX, 1                 ; Si n <= 1, retourner 1
      JLE @fin
    @boucle:
      MUL ECX                    ; EAX = EAX * ECX
      DEC ECX                    ; ECX = ECX - 1
      CMP ECX, 1                 ; Continuer si ECX > 1
      JG  @boucle
    @fin:
      RET
    Factoriel_Iter ENDP

  11.2.2 Version recursive

    ; Calcule n! de maniere recursive
    ; Parametre sur la pile : n (DWORD)
    ; Sortie : EAX = n!
    Factoriel_Rec PROC
      PUSH EBP
      MOV EBP, ESP
      MOV EAX, [EBP+8]          ; EAX = n

      CMP EAX, 1                ; Si n <= 1
      JLE @cas_base              ; Retourner 1

      DEC EAX                    ; n - 1
      PUSH EAX                   ; Empiler (n-1)
      CALL Factoriel_Rec         ; EAX = (n-1)!
      ADD ESP, 4                 ; Nettoyer la pile

      MOV ECX, [EBP+8]          ; ECX = n
      MUL ECX                    ; EAX = (n-1)! * n
      JMP @fin

    @cas_base:
      MOV EAX, 1                 ; Retourner 1

    @fin:
      MOV ESP, EBP
      POP EBP
      RET
    Factoriel_Rec ENDP

  11.3 Manipulation de chaines
  -------------------------------

  11.3.1 Longueur d'une chaine (strlen)

    ; Calcule la longueur d'une chaine terminee par zero
    ; Entree : ESI = adresse de la chaine
    ; Sortie : ECX = longueur
    StrLen PROC
      PUSH ESI                   ; Sauvegarder ESI
      XOR ECX, ECX               ; Compteur = 0
    @boucle:
      LODSB                      ; AL = [ESI], ESI++
      TEST AL, AL                ; AL == 0 ?
      JZ @fin                    ; Oui : fin de chaine
      INC ECX                    ; Compteur++
      JMP @boucle
    @fin:
      POP ESI                    ; Restaurer ESI
      RET
    StrLen ENDP

  11.3.2 Copie de chaine (strcpy)

    ; Copie une chaine source vers une destination
    ; Entree : EDI = destination, ESI = source
    StrCpy PROC
      PUSH ESI
      PUSH EDI
    @boucle:
      LODSB                      ; AL = [ESI], ESI++
      STOSB                      ; [EDI] = AL, EDI++
      TEST AL, AL                ; Caractere nul ?
      JNZ @boucle                ; Non : continuer
      POP EDI
      POP ESI
      RET
    StrCpy ENDP

  11.3.3 Comparaison de chaines (strcmp)

    ; Compare deux chaines
    ; Entree : ESI = chaine1, EDI = chaine2
    ; Sortie : Flags ZF=1 si egales, EAX = difference
    StrCmp PROC
      PUSH ESI
      PUSH EDI
    @boucle:
      LODSB                      ; AL = [ESI], ESI++
      MOV AH, [EDI]              ; AH = [EDI]
      INC EDI
      CMP AL, AH                 ; Comparer les caracteres
      JNE @different             ; Differents : sortir
      TEST AL, AL                ; Fin de chaine ?
      JNZ @boucle                ; Non : continuer
      XOR EAX, EAX               ; Egal : retourner 0
      JMP @fin
    @different:
      MOVZX EAX, AL              ; Etendre AL en EAX
      MOVZX ECX, AH              ; Etendre AH en ECX
      SUB EAX, ECX               ; Difference
    @fin:
      POP EDI
      POP ESI
      RET
    StrCmp ENDP

  11.3.4 Recherche de caractere (strchr)

    ; Recherche un caractere dans une chaine
    ; Entree : ESI = chaine, AL = caractere a chercher
    ; Sortie : EDI = adresse du caractere (0 si non trouve)
    StrChr PROC
      MOV AH, AL                 ; Sauvegarder le caractere
    @boucle:
      MOV AL, [ESI]
      CMP AL, AH                 ; Caractere trouve ?
      JE @trouve
      TEST AL, AL                ; Fin de chaine ?
      JZ @pas_trouve
      INC ESI
      JMP @boucle
    @trouve:
      MOV EDI, ESI               ; Retourner l'adresse
      RET
    @pas_trouve:
      XOR EDI, EDI               ; Retourner 0
      RET
    StrChr ENDP

  11.4 Conversion nombre-chaine
  --------------------------------

  11.4.1 Entier en chaine decimale (itoa)

    ; Convertit un entier 32 bits non signe en chaine decimale
    ; Entree : EAX = nombre, EDI = tampon de sortie (min 11 octets)
    IntToStr PROC
      PUSH EBX
      PUSH ECX
      PUSH EDX
      PUSH EDI

      MOV EBX, 10                ; Diviseur = 10
      XOR ECX, ECX               ; Compteur de chiffres = 0

      ; Extraire les chiffres (ordre inverse)
    @extraire:
      XOR EDX, EDX               ; Preparer la division
      DIV EBX                    ; EAX = quotient, EDX = reste
      ADD DL, '0'                ; Convertir en ASCII
      PUSH EDX                   ; Empiler le chiffre
      INC ECX                    ; Compteur++
      TEST EAX, EAX              ; Quotient = 0 ?
      JNZ @extraire              ; Non : continuer

      ; Depiler les chiffres (ordre correct)
      POP EDI                    ; Restaurer EDI
      PUSH EDI                   ; Re-empiler pour la fin
    @ecrire:
      POP EDX                    ; Depiler un chiffre
      MOV [EDI], DL              ; Ecrire dans le tampon
      INC EDI
      DEC ECX
      JNZ @ecrire

      MOV BYTE PTR [EDI], 0     ; Terminateur nul

      POP EDI
      POP EDX
      POP ECX
      POP EBX
      RET
    IntToStr ENDP

  11.4.2 Chaine decimale en entier (atoi)

    ; Convertit une chaine decimale en entier 32 bits non signe
    ; Entree : ESI = adresse de la chaine
    ; Sortie : EAX = valeur
    StrToInt PROC
      PUSH EBX
      PUSH ECX
      XOR EAX, EAX               ; Resultat = 0
      MOV EBX, 10                ; Multiplicateur

    @boucle:
      MOVZX ECX, BYTE PTR [ESI]  ; Lire un caractere
      TEST CL, CL                ; Fin de chaine ?
      JZ @fin
      SUB CL, '0'                ; Convertir en valeur
      CMP CL, 9                  ; Chiffre valide ?
      JA @fin                    ; Non : arreter
      MUL EBX                    ; EAX = EAX * 10
      ADD EAX, ECX               ; EAX = EAX + chiffre
      INC ESI
      JMP @boucle

    @fin:
      POP ECX
      POP EBX
      RET
    StrToInt ENDP

  11.5 Tri a bulles (Bubble Sort)
  ---------------------------------

    ; Tri a bulles d'un tableau de DWORD
    ; Entree : ESI = adresse du tableau, ECX = nombre d'elements
    BubbleSort PROC
      PUSH EAX
      PUSH EBX
      PUSH ECX
      PUSH EDX
      PUSH ESI

      DEC ECX                    ; n-1 comparaisons max
      JLE @fin                   ; Tableau vide ou 1 element

    @passe:
      XOR EDX, EDX               ; Flag echange = 0
      MOV EBX, ECX               ; Compteur de la passe
      PUSH ESI                   ; Sauvegarder le debut du tableau

    @comparer:
      MOV EAX, [ESI]             ; EAX = element courant
      CMP EAX, [ESI+4]           ; Comparer avec le suivant
      JLE @pas_echanger          ; Ordre correct : passer

      ; Echanger les deux elements
      XCHG EAX, [ESI+4]         ; Echanger EAX <-> [ESI+4]
      MOV [ESI], EAX             ; Stocker dans [ESI]
      INC EDX                    ; Flag echange = 1

    @pas_echanger:
      ADD ESI, 4                 ; Element suivant
      DEC EBX                    ; Decrementation compteur
      JNZ @comparer              ; Continuer la passe

      POP ESI                    ; Restaurer le debut du tableau
      TEST EDX, EDX              ; Des echanges ont ete faits ?
      JZ @fin                    ; Non : tableau trie
      DEC ECX                    ; Reduire la zone a trier
      JNZ @passe                 ; Passe suivante

    @fin:
      POP ESI
      POP EDX
      POP ECX
      POP EBX
      POP EAX
      RET
    BubbleSort ENDP

  11.6 Recherche binaire (Binary Search)
  -----------------------------------------

    ; Recherche binaire dans un tableau de DWORD trie
    ; Entree : ESI = tableau, ECX = nombre d'elements,
    ;          EAX = valeur recherchee
    ; Sortie : EDI = index (ou -1 si non trouve)
    BinarySearch PROC
      PUSH EBX
      PUSH EDX

      XOR EBX, EBX               ; gauche = 0
      DEC ECX                    ; droite = n - 1

    @boucle:
      CMP EBX, ECX               ; gauche > droite ?
      JG @non_trouve

      MOV EDI, EBX               ; milieu = (gauche + droite) / 2
      ADD EDI, ECX
      SHR EDI, 1

      MOV EDX, [ESI + EDI*4]    ; EDX = tableau[milieu]
      CMP EAX, EDX               ; Comparer avec la valeur cherchee
      JE @trouve                 ; Egal : trouve !
      JL @gauche                 ; Inferieur : chercher a gauche

      ; Chercher a droite
      LEA EBX, [EDI+1]          ; gauche = milieu + 1
      JMP @boucle

    @gauche:
      LEA ECX, [EDI-1]          ; droite = milieu - 1
      JMP @boucle

    @non_trouve:
      MOV EDI, -1                ; Retourner -1 (non trouve)

    @trouve:                     ; EDI contient l'index
      POP EDX
      POP EBX
      RET
    BinarySearch ENDP

  11.7 Operations sur les bits
  ------------------------------

  11.7.1 Compter les bits a 1 (popcount)

    ; Compte le nombre de bits a 1 dans un registre 32 bits
    ; Entree : EAX = valeur
    ; Sortie : ECX = nombre de bits a 1
    PopCount PROC
      XOR ECX, ECX               ; Compteur = 0
    @boucle:
      TEST EAX, EAX              ; EAX == 0 ?
      JZ @fin
      MOV EDX, EAX
      DEC EDX                    ; EDX = EAX - 1
      AND EAX, EDX               ; Effacer le bit le plus bas
      INC ECX                    ; Compteur++
      JMP @boucle
    @fin:
      RET
    PopCount ENDP

  11.7.2 Inversion des bits (bit reverse)

    ; Inverse l'ordre des bits dans EAX (bit 0 <-> bit 31, etc.)
    ; Entree/Sortie : EAX
    BitReverse PROC
      PUSH EBX
      PUSH ECX
      MOV ECX, 32                ; 32 bits a traiter
      XOR EBX, EBX               ; Resultat = 0
    @boucle:
      SHL EBX, 1                 ; Decaler le resultat a gauche
      MOV EDX, EAX
      AND EDX, 1                 ; Extraire le bit de poids faible
      OR EBX, EDX                ; Ajouter au resultat
      SHR EAX, 1                 ; Bit suivant de la source
      DEC ECX
      JNZ @boucle
      MOV EAX, EBX               ; Resultat dans EAX
      POP ECX
      POP EBX
      RET
    BitReverse ENDP

  11.7.3 Plus grande puissance de 2 inferieure ou egale

    ; Trouve la plus grande puissance de 2 <= EAX
    ; Entree : EAX = valeur (> 0)
    ; Sortie : EAX = puissance de 2
    FloorPow2 PROC
      BSR ECX, EAX               ; ECX = position du bit le plus haut
      MOV EAX, 1
      SHL EAX, CL                ; EAX = 2^ECX
      RET
    FloorPow2 ENDP

  11.8 Utilisation de la pile et cadres de pile
  ------------------------------------------------

    ; Exemple de fonction avec variables locales et cadre de pile
    ;
    ; int Exemple(int a, int b) {
    ;   int temp = a + b;
    ;   int result = temp * 2;
    ;   return result;
    ; }

    Exemple PROC
      PUSH EBP                   ; Sauvegarder l'ancien cadre
      MOV EBP, ESP               ; Etablir le nouveau cadre
      SUB ESP, 8                 ; Reserver 2 variables locales

      ; [EBP+8]  = parametre a
      ; [EBP+12] = parametre b
      ; [EBP-4]  = variable locale temp
      ; [EBP-8]  = variable locale result

      MOV EAX, [EBP+8]          ; EAX = a
      ADD EAX, [EBP+12]         ; EAX = a + b
      MOV [EBP-4], EAX          ; temp = a + b

      SHL EAX, 1                 ; EAX = temp * 2
      MOV [EBP-8], EAX          ; result = temp * 2

      MOV EAX, [EBP-8]          ; Retourner result dans EAX

      MOV ESP, EBP               ; Restaurer la pile
      POP EBP                    ; Restaurer l'ancien cadre
      RET                        ; Retourner a l'appelant
    Exemple ENDP

    ; Appel de la fonction :
    ;   PUSH 20                  ; Parametre b = 20
    ;   PUSH 10                  ; Parametre a = 10
    ;   CALL Exemple
    ;   ADD ESP, 8               ; Nettoyer la pile (convention C)
    ;   ; EAX = (10 + 20) * 2 = 60

    ; Etat de la pile pendant l'execution :
    ;
    ;   ESP -> [EBP-8] result
    ;          [EBP-4] temp
    ;   EBP -> [EBP]   ancien EBP
    ;          [EBP+4] adresse de retour
    ;          [EBP+8] parametre a = 10
    ;          [EBP+12] parametre b = 20

  11.9 Acces memoire avec modes d'adressage avances
  -----------------------------------------------------

    ; Exemples des modes d'adressage du 80386

    .DATA
    tableau   DD 10, 20, 30, 40, 50   ; Tableau de 5 DWORD
    matrice   DD 1, 2, 3              ; Ligne 0
              DD 4, 5, 6              ; Ligne 1
              DD 7, 8, 9              ; Ligne 2
    COLS      EQU 3                    ; 3 colonnes

    .CODE
    ExempleAdressage PROC
      ; Adressage direct
      MOV EAX, [tableau]              ; Premier element

      ; Adressage base + deplacement
      MOV ESI, OFFSET tableau
      MOV EAX, [ESI + 8]              ; Troisieme element (offset 8)

      ; Adressage indexe avec facteur d'echelle
      MOV ECX, 2                       ; Index = 2
      MOV EAX, [tableau + ECX*4]      ; tableau[2] = 30

      ; Adressage base + index * echelle + deplacement
      ; Acces matrice[ligne][colonne] :
      ; adresse = matrice + (ligne * COLS + colonne) * 4
      MOV EBX, 1                       ; ligne = 1
      IMUL EBX, COLS                   ; EBX = ligne * COLS
      ADD EBX, 2                       ; EBX = ligne * COLS + colonne
      MOV EAX, [matrice + EBX*4]      ; matrice[1][2] = 6

      ; Meme chose avec LEA
      MOV ESI, OFFSET matrice
      MOV EBX, 1                       ; ligne = 1
      IMUL EBX, COLS * 4              ; EBX = ligne * COLS * 4
      MOV ECX, 2                       ; colonne = 2
      LEA EAX, [ESI + EBX + ECX*4]   ; Adresse de matrice[1][2]
      MOV EAX, [EAX]                  ; Valeur = 6

      RET
    ExempleAdressage ENDP

  11.10 Utilisation des instructions de chaine (REP)
  -----------------------------------------------------

    ; Copie rapide de bloc memoire (memcpy)
    ; Entree : EDI = destination, ESI = source, ECX = taille en octets
    MemCopy PROC
      PUSH ECX
      PUSH ESI
      PUSH EDI
      CLD                        ; Direction : increment

      ; Copier par blocs de 4 octets (DWORD)
      MOV EBX, ECX               ; Sauvegarder la taille totale
      SHR ECX, 2                 ; Nombre de DWORD
      REP MOVSD                  ; Copier les DWORD

      ; Copier les octets restants
      MOV ECX, EBX
      AND ECX, 3                 ; Reste = taille MOD 4
      REP MOVSB                  ; Copier les octets restants

      POP EDI
      POP ESI
      POP ECX
      RET
    MemCopy ENDP

    ; Remplissage de bloc memoire (memset)
    ; Entree : EDI = destination, AL = valeur, ECX = taille en octets
    MemSet PROC
      PUSH ECX
      PUSH EDI
      CLD

      ; Etendre AL sur les 4 octets de EAX
      MOV AH, AL                 ; AX = AL:AL
      MOV BX, AX
      SHL EAX, 16
      MOV AX, BX                 ; EAX = AL:AL:AL:AL

      ; Remplir par blocs de 4 octets
      MOV EBX, ECX
      SHR ECX, 2
      REP STOSD

      ; Remplir les octets restants
      MOV ECX, EBX
      AND ECX, 3
      REP STOSB

      POP EDI
      POP ECX
      RET
    MemSet ENDP

    ; Recherche d'un octet dans un bloc (memchr)
    ; Entree : EDI = bloc, AL = octet, ECX = taille
    ; Sortie : EDI = adresse (0 si non trouve), ZF = 1 si trouve
    MemChr PROC
      CLD
      REPNE SCASB                ; Chercher AL dans [EDI]
      JNE @non_trouve
      DEC EDI                    ; Revenir sur l'octet trouve
      RET
    @non_trouve:
      XOR EDI, EDI               ; Retourner 0
      RET
    MemChr ENDP

  11.11 Exemple avec macro et assemblage conditionnel
  -------------------------------------------------------

    ; Macro de trace pour le debug
    DEBUG_MODE = 1

    PrintDebug MACRO msg
      IF DEBUG_MODE
        PUSH OFFSET msg
        CALL _puts
        ADD ESP, 4
      ENDIF
    ENDM

    ; Macro d'assertion
    ASSERT MACRO condition
      LOCAL ok
      IF DEBUG_MODE
        condition
        JMP ok
        INT 3                    ; Point d'arret en mode debug
      ok:
      ENDIF
    ENDM

    ; Macro pour declarer une chaine avec sa longueur
    DEFSTR MACRO nom, texte
      nom       DB texte, 0
      nom&_len  EQU $ - nom - 1
    ENDM

    .DATA
    DEFSTR salut, 'Bonjour le monde'
    ; Genere :
    ;   salut     DB 'Bonjour le monde', 0
    ;   salut_len EQU 16

    ; Macro de boucle FOR (haut niveau)
    FOR_LOOP MACRO reg, debut, fin
      MOV reg, debut
    ENDM

    FOR_NEXT MACRO reg, etiq_debut
      INC reg
      JMP etiq_debut
    ENDM

  11.12 Programme complet : Conversion hexadecimale
  ----------------------------------------------------

  Programme Win32 complet qui convertit un nombre 32 bits en
  sa representation hexadecimale et l'affiche sur la console.

    .386
    .MODEL FLAT, STDCALL
    OPTION CASEMAP:NONE

    EXTRN GetStdHandle@4:NEAR
    EXTRN WriteConsoleA@20:NEAR
    EXTRN ExitProcess@4:NEAR

    .STACK 4096

    .DATA
    hexchars  DB '0123456789ABCDEF'   ; Table de conversion
    buffer    DB '00000000', 0Dh, 0Ah, 0  ; 8 chiffres + CRLF + nul
    buf_len   EQU 10                   ; 8 chiffres + CR + LF
    nombre    DD 0DEADBEEFh           ; Valeur a convertir

    .DATA?
    hStdOut   DD ?
    written   DD ?

    .CODE
    ; Convertit EAX en hexadecimal dans le buffer
    DwordToHex PROC
      PUSH EBX
      PUSH ECX
      PUSH EDI

      MOV EDI, OFFSET buffer + 7 ; Pointer sur le dernier chiffre
      MOV ECX, 8                  ; 8 chiffres hexadecimaux

    @boucle:
      MOV EBX, EAX
      AND EBX, 0Fh               ; Extraire le quartet bas
      MOV BL, [hexchars + EBX]  ; Convertir en caractere
      MOV [EDI], BL              ; Ecrire dans le buffer
      SHR EAX, 4                 ; Quartet suivant
      DEC EDI                    ; Position precedente
      DEC ECX
      JNZ @boucle

      POP EDI
      POP ECX
      POP EBX
      RET
    DwordToHex ENDP

    main PROC
      ; Obtenir le handle de sortie
      PUSH -11
      CALL GetStdHandle@4
      MOV [hStdOut], EAX

      ; Convertir le nombre en hexadecimal
      MOV EAX, [nombre]
      CALL DwordToHex

      ; Afficher le resultat
      PUSH 0
      PUSH OFFSET written
      PUSH buf_len
      PUSH OFFSET buffer
      PUSH [hStdOut]
      CALL WriteConsoleA@20

      ; Quitter
      PUSH 0
      CALL ExitProcess@4
    main ENDP
    END main

==============================================================================
12. CONVERSION DE CODE 8086 EN 80386
==============================================================================

  Cette section presente les techniques et les regles pour convertir du
  code assembleur 8086 (mode reel 16 bits) en code 80386 (mode protege
  32 bits). La migration permet d'exploiter les registres etendus, l'espace
  d'adressage lineaire de 4 Go et les nouveaux modes d'adressage.

  12.1 Principes generaux
  ---------------------------------------------------------------------------

  Le 80386 est retro-compatible avec le 8086 : tout programme 8086 peut
  tourner en mode reel sur un 80386 sans modification. Toutefois, pour
  profiter des capacites 32 bits (mode protege, modele FLAT), il faut
  adapter le code source. Les grandes lignes de la conversion sont :

    1. Remplacer les registres 16 bits par leurs equivalents 32 bits
    2. Adapter les directives assembleur (.MODEL, .386, segments)
    3. Convertir les appels systeme (INT 21h -> API Win32 ou INT 80h)
    4. Ajuster les tailles de donnees et d'adresses
    5. Eliminer la segmentation et passer en modele FLAT
    6. Exploiter les nouveaux modes d'adressage et instructions

  12.2 Correspondance des registres
  ---------------------------------------------------------------------------

  La table suivante montre l'equivalence directe des registres :

    8086 (16 bits)    80386 (32 bits)    Notes
    ----------------  -----------------  --------------------------------
    AX                EAX                Accumulateur etendu a 32 bits
    BX                EBX                Base etendue
    CX                ECX                Compteur etendu
    DX                EDX                Donnees etendu
    SI                ESI                Index source etendu
    DI                EDI                Index destination etendu
    BP                EBP                Pointeur de base etendu
    SP                ESP                Pointeur de pile etendu
    IP                EIP                Pointeur d'instruction (implicite)
    FLAGS             EFLAGS             Registre d'etat etendu

  Les registres de segment (CS, DS, ES, SS) existent toujours mais en
  mode protege FLAT, ils pointent tous sur le meme descripteur couvrant
  4 Go. FS et GS sont deux nouveaux registres de segment du 80386.

  Regles de conversion des registres :

    - AX -> EAX, BX -> EBX, CX -> ECX, DX -> EDX
    - SI -> ESI, DI -> EDI, BP -> EBP, SP -> ESP
    - AH, AL, BH, BL, CH, CL, DH, DL restent identiques
    - Les registres 16 bits (AX, BX...) restent utilisables dans du
      code 32 bits mais provoquent des prefixes de taille (ralentissement)

  12.3 Adaptation des directives assembleur
  ---------------------------------------------------------------------------

  12.3.1 En-tete du programme

    8086 :                              80386 :
    --------------------------------    --------------------------------
    .MODEL SMALL                        .386
    .STACK 100h                         .MODEL FLAT, STDCALL
    .DATA                               .DATA
    ...                                 ...
    .CODE                               .CODE
    main:                               main PROC
      MOV AX, @DATA                       ; Pas besoin de charger DS
      MOV DS, AX                          ; DS est deja configure
      ...                                 ...
      MOV AX, 4C00h                       PUSH 0
      INT 21h                             CALL ExitProcess@4
    END main                            main ENDP
                                        END main

  12.3.2 Directive processeur

    8086 : pas de directive necessaire (defaut)
    80386 : ajouter .386 ou .386P (mode privilegie) en premiere ligne

  12.3.3 Modele memoire

    8086 : .MODEL TINY / SMALL / MEDIUM / COMPACT / LARGE / HUGE
    80386 : .MODEL FLAT, STDCALL (un seul segment de 4 Go)

  12.3.4 Initialisation des segments

    En 8086, il faut initialiser DS manuellement :

      MOV AX, @DATA
      MOV DS, AX

    En 80386 mode FLAT, DS, ES, SS pointent deja sur le meme espace
    d'adressage. Ces deux lignes doivent etre supprimees.

  12.4 Conversion des types de donnees
  ---------------------------------------------------------------------------

    8086                    80386                   Notes
    ----------------------  ----------------------  -----------------------
    DB  (1 octet)           DB  (1 octet)           Identique
    DW  (2 octets, WORD)    DW  (2 octets, WORD)    Identique
    DD  (4 octets, DWORD)   DD  (4 octets, DWORD)   Identique
    -                       DQ  (8 octets, QWORD)    Nouveau
    -                       DT  (10 octets, TBYTE)   Nouveau

  Regles de conversion des donnees :

    - Les pointeurs passent de DW (16 bits) a DD (32 bits)
    - Les offsets passent de 16 a 32 bits
    - OFFSET retourne 32 bits au lieu de 16
    - Les tableaux de pointeurs : DW -> DD
    - Les variables compteur utilisant CX (16 bits max 65535)
      peuvent maintenant utiliser ECX (32 bits max 4 milliards)

  Exemple :

    ; 8086                          ; 80386
    msg    DB 'Hello',0             msg    DB 'Hello',0
    msgptr DW OFFSET msg            msgptr DD OFFSET msg
    count  DW 0                     count  DD 0
    buffer DB 256 DUP(0)            buffer DB 256 DUP(0)

  12.5 Conversion des instructions courantes
  ---------------------------------------------------------------------------

  12.5.1 Instructions de deplacement de donnees

    8086                              80386
    --------------------------------  --------------------------------
    MOV AX, valeur                    MOV EAX, valeur
    MOV AX, [BX]                      MOV EAX, [EBX]
    MOV AX, [BX+SI]                   MOV EAX, [EBX+ESI]
    MOV AX, [BX+SI+4]                MOV EAX, [EBX+ESI+4]
    MOV AL, [BX]                      MOV AL, [EBX]
    LEA BX, [var]                     LEA EBX, [var]
    LDS SI, [ptr]                     (non utilise en FLAT)
    LES DI, [ptr]                     (non utilise en FLAT)
    XCHG AX, BX                      XCHG EAX, EBX
    PUSH AX                           PUSH EAX
    POP AX                            POP EAX

  Note : LDS et LES ne sont plus necessaires en modele FLAT car il
  n'y a qu'un seul segment. Remplacer par un simple MOV :

    ; 8086                          ; 80386
    LDS SI, [ptr]                   MOV ESI, [ptr]
    LES DI, [ptr]                   MOV EDI, [ptr]

  12.5.2 Instructions arithmetiques

    8086                              80386
    --------------------------------  --------------------------------
    ADD AX, BX                        ADD EAX, EBX
    SUB AX, 10                        SUB EAX, 10
    MUL BX                            MUL EBX         (ou IMUL EAX,EBX)
    DIV BX                            DIV EBX
    INC AX                            INC EAX
    DEC CX                            DEC ECX
    NEG AX                            NEG EAX
    CWD                               CDQ              (etend EAX -> EDX:EAX)
    CBW                               CWDE             (etend AX -> EAX)

  Nouvelles instructions utiles du 80386 :

    IMUL EAX, EBX, 10      ; Multiplication 3 operandes (nouveau)
    MOVZX EAX, AL           ; Extension zero 8->32 bits (nouveau)
    MOVSX EAX, AX           ; Extension signe 16->32 bits (nouveau)
    MOVZX EAX, BYTE PTR [ESI]  ; Chargement avec extension zero
    MOVSX ECX, WORD PTR [EDI]  ; Chargement avec extension signe

  12.5.3 Instructions de chaines

    8086                              80386
    --------------------------------  --------------------------------
    REP MOVSB                         REP MOVSB        (identique)
    REP MOVSW                         REP MOVSD        (deplacer DWORD)
    REP STOSB                         REP STOSD        (stocker DWORD)
    LODSB                             LODSD            (charger DWORD dans EAX)
    SCASB                             SCASD            (comparer DWORD)
    CLD                               CLD              (identique)

  En 8086, les operations de chaines utilisent DS:SI et ES:DI.
  En 80386 mode FLAT, elles utilisent ESI et EDI (meme segment).

  12.5.4 Instructions de branchement

    8086                              80386
    --------------------------------  --------------------------------
    JMP SHORT label                   JMP SHORT label   (identique)
    JMP label                         JMP label         (rel32 maintenant)
    CALL proc                         CALL proc         (identique)
    RET                               RET               (identique)
    Jcc label (rel8 seulement)        Jcc label         (rel8 ou rel32)
    LOOP label                        LOOP label        (utilise ECX)
    JCXZ label                        JECXZ label       (teste ECX)

  Note importante : en 8086, les sauts conditionnels (Jcc) ne peuvent
  atteindre que des cibles a +/-128 octets (rel8). En 80386, ils
  acceptent aussi des deplacements rel32 (portee quasi illimitee).

  12.5.5 Instructions de pile

    8086                              80386
    --------------------------------  --------------------------------
    PUSH AX                           PUSH EAX          (empile 32 bits)
    POP BX                            POP EBX           (depile 32 bits)
    PUSH 1234h                        PUSH 12345678h    (immediats 32 bits)
    PUSHF                             PUSHFD            (EFLAGS 32 bits)
    POPF                              POPFD             (EFLAGS 32 bits)
    PUSHA                             PUSHAD            (tous les reg 32 bits)
    POPA                              POPAD             (tous les reg 32 bits)

  En mode 32 bits, chaque PUSH/POP manipule 4 octets (au lieu de 2).

  12.6 Conversion des appels systeme
  ---------------------------------------------------------------------------

  C'est le changement le plus important. En 8086 sous DOS, les appels
  systeme se font par INT 21h. En 80386, on utilise les API du systeme
  d'exploitation (Win32, Linux, etc.).

  12.6.1 Afficher une chaine de caracteres

    ; 8086 (DOS, INT 21h)              ; 80386 (Win32)
    ; -------------------------------- ; --------------------------------
    .DATA                               .DATA
    msg DB 'Hello$'                     msg DB 'Hello', 0
                                        written DD 0
    .CODE                               .CODE
    MOV AH, 09h                         PUSH -11
    LEA DX, msg                         CALL GetStdHandle@4
    INT 21h                             PUSH 0
                                        PUSH OFFSET written
                                        PUSH 5
                                        PUSH OFFSET msg
                                        PUSH EAX
                                        CALL WriteConsoleA@20

  12.6.2 Lire un caractere au clavier

    ; 8086 (DOS)                       ; 80386 (Win32)
    ; -------------------------------- ; --------------------------------
    MOV AH, 01h                         PUSH -10
    INT 21h                             CALL GetStdHandle@4
    ; caractere dans AL                 MOV [hStdIn], EAX
                                        PUSH 0
                                        PUSH OFFSET bytesRead
                                        PUSH 1
                                        PUSH OFFSET charBuf
                                        PUSH [hStdIn]
                                        CALL ReadConsoleA@20
                                        ; caractere dans charBuf

  12.6.3 Quitter le programme

    ; 8086 (DOS)                       ; 80386 (Win32)
    ; -------------------------------- ; --------------------------------
    MOV AX, 4C00h                       PUSH 0
    INT 21h                             CALL ExitProcess@4

  12.6.4 Ouvrir un fichier

    ; 8086 (DOS)                       ; 80386 (Win32)
    ; -------------------------------- ; --------------------------------
    MOV AH, 3Dh                         PUSH 0            ; template
    MOV AL, 00h  ; lecture              PUSH 80h           ; attribut normal
    LEA DX, filename                    PUSH 3             ; OPEN_EXISTING
    INT 21h                             PUSH 0             ; securite
    ; handle dans AX                    PUSH 1             ; FILE_SHARE_READ
                                        PUSH 80000000h     ; GENERIC_READ
                                        PUSH OFFSET filename
                                        CALL CreateFileA@28
                                        ; handle dans EAX

  12.6.5 Table de correspondance DOS -> Win32

    Fonction DOS (INT 21h)       API Win32 equivalente
    --------------------------   ----------------------------
    AH=01h  Lire caractere       ReadConsoleA
    AH=02h  Ecrire caractere     WriteConsoleA
    AH=09h  Afficher chaine $    WriteConsoleA
    AH=0Ah  Lire chaine          ReadConsoleA
    AH=3Ch  Creer fichier        CreateFileA
    AH=3Dh  Ouvrir fichier       CreateFileA
    AH=3Eh  Fermer fichier       CloseHandle
    AH=3Fh  Lire fichier         ReadFile
    AH=40h  Ecrire fichier       WriteFile
    AH=41h  Supprimer fichier    DeleteFileA
    AH=43h  Attributs fichier    GetFileAttributesA
    AH=48h  Allouer memoire      VirtualAlloc / HeapAlloc
    AH=49h  Liberer memoire      VirtualFree / HeapFree
    AH=4Ch  Quitter programme    ExitProcess

  12.7 Conversion des modes d'adressage
  ---------------------------------------------------------------------------

  Le 80386 offre des modes d'adressage beaucoup plus flexibles.

  12.7.1 Adressage indirect

    8086 : seulement [BX], [SI], [DI], [BP]
    80386 : tout registre sauf ESP -> [EAX], [EBX], [ECX], [EDX],
            [ESI], [EDI], [EBP]

    ; 8086                          ; 80386 (equivalent + plus)
    MOV AX, [BX]                    MOV EAX, [EBX]
    MOV AX, [SI]                    MOV EAX, [ESI]
                                    MOV EAX, [ECX]       ; nouveau!
                                    MOV EAX, [EDX]       ; nouveau!

  12.7.2 Adressage base+index

    8086 : seulement [BX+SI], [BX+DI], [BP+SI], [BP+DI]
    80386 : toute combinaison [reg_base + reg_index]

    ; 8086                          ; 80386
    MOV AX, [BX+SI]                 MOV EAX, [EBX+ESI]
    MOV AX, [BP+DI]                 MOV EAX, [EBP+EDI]
                                    MOV EAX, [EAX+ECX]   ; nouveau!
                                    MOV EAX, [EDX+EDI]   ; nouveau!

  12.7.3 Adressage avec facteur d'echelle (nouveau 80386)

    Le 80386 introduit le facteur d'echelle (SIB) qui n'existait pas
    sur le 8086. Il permet de multiplier l'index par 1, 2, 4 ou 8 :

    MOV EAX, [EBX + ESI*4]          ; acces tableau de DWORD
    MOV EAX, [EBX + ECX*8]          ; acces tableau de QWORD
    MOV EAX, [array + ESI*4]        ; tableau global indexe
    LEA EAX, [EAX + EAX*4]          ; EAX = EAX * 5 (astuce)
    LEA EAX, [EAX + EAX*2]          ; EAX = EAX * 3

  Conversion typique d'acces tableau :

    ; 8086 : acces tableau de WORD       ; 80386 : acces tableau de DWORD
    MOV BX, index                        MOV ECX, index
    SHL BX, 1          ; *2              ; pas de shift necessaire
    MOV AX, [table+BX]                   MOV EAX, [table + ECX*4]

  12.8 Conversion des structures de controle
  ---------------------------------------------------------------------------

  12.8.1 Boucle avec compteur

    ; 8086                          ; 80386
    ; --------------------------------  --------------------------------
    MOV CX, 100                     MOV ECX, 100
    boucle:                         boucle:
      ; ... corps ...                 ; ... corps ...
      LOOP boucle                     LOOP boucle       ; ou DEC ECX/JNZ

  Note : sur 80386, LOOP utilise ECX en mode 32 bits. Si la boucle
  depasse 65535 iterations, il faut obligatoirement passer en 32 bits.

  12.8.2 Boucle avec index de tableau

    ; 8086                          ; 80386
    ; --------------------------------  --------------------------------
    XOR SI, SI                      XOR ESI, ESI
    MOV CX, 10                      MOV ECX, 10
    boucle:                         boucle:
      MOV AX, [table+SI]             MOV EAX, [table + ESI*4]
      ADD SI, 2                       INC ESI
      LOOP boucle                     DEC ECX
                                      JNZ boucle

  12.8.3 Comparaison de chaines

    ; 8086                          ; 80386
    ; --------------------------------  --------------------------------
    LEA SI, str1                    LEA ESI, str1
    LEA DI, str2                    LEA EDI, str2
    MOV CX, longueur                MOV ECX, longueur
    CLD                             CLD
    REPE CMPSB                      REPE CMPSB         ; identique
    JE egal                         JE egal

  12.9 Conversion des procedures et conventions d'appel
  ---------------------------------------------------------------------------

  12.9.1 Cadre de pile (stack frame)

    ; 8086                          ; 80386
    ; --------------------------------  --------------------------------
    maProc PROC NEAR                maProc PROC
      PUSH BP                         PUSH EBP
      MOV BP, SP                       MOV EBP, ESP
      SUB SP, 4    ; 2 vars locales    SUB ESP, 8    ; 2 vars locales (DD)
      ; [BP+4] = adr retour            ; [EBP+4] = adr retour
      ; [BP+6] = 1er parametre         ; [EBP+8] = 1er parametre
      ; [BP-2] = 1ere locale           ; [EBP-4] = 1ere locale
      MOV SP, BP                       MOV ESP, EBP
      POP BP                           POP EBP
      RET                              RET
    maProc ENDP                     maProc ENDP

  Differences principales :
    - BP -> EBP, SP -> ESP
    - L'adresse de retour occupe 4 octets (au lieu de 2)
    - Les parametres commencent a [EBP+8] (au lieu de [BP+4])
    - Chaque variable locale occupe 4 octets (DWORD)

  12.9.2 Passage de parametres

    ; 8086 : convention C             ; 80386 : convention C
    ; --------------------------------  --------------------------------
    PUSH word_param2                  PUSH dword_param2
    PUSH word_param1                  PUSH dword_param1
    CALL maProc                       CALL maProc
    ADD SP, 4      ; 2 * 2 octets    ADD ESP, 8     ; 2 * 4 octets

    ; 8086 : convention Pascal        ; 80386 : convention STDCALL
    ; --------------------------------  --------------------------------
    PUSH word_param1                  PUSH dword_param1
    PUSH word_param2                  PUSH dword_param2
    CALL maProc                       CALL maProc
    ; pas d'ajustement                ; pas d'ajustement (RET 8)

  12.9.3 Conventions d'appel Win32

    En mode 32 bits Win32, la convention STDCALL est la norme pour
    les API systeme. Les parametres sont empiles de droite a gauche
    et la procedure appelee nettoie la pile (RET n).

  12.10 Conversion des interruptions
  ---------------------------------------------------------------------------

  Les interruptions materielles et BIOS changent en mode protege :

    8086 (mode reel)                  80386 (mode protege)
    --------------------------------  --------------------------------
    INT 10h (video BIOS)              API graphiques du SE
    INT 13h (disque BIOS)             API fichiers du SE
    INT 16h (clavier BIOS)            API console du SE
    INT 21h (DOS)                     API Win32 / syscall Linux
    INT 33h (souris)                  API souris du SE
    CLI / STI                         Niveau privilege 0 requis

  En mode protege, les interruptions logicielles DOS (INT 21h) ne sont
  plus disponibles. Il faut les remplacer par les API du systeme cible.

  Sous Linux 32 bits, les appels systeme utilisent INT 80h :

    ; Equivalent de AH=09h INT 21h sous Linux
    MOV EAX, 4            ; sys_write
    MOV EBX, 1            ; stdout
    MOV ECX, OFFSET msg   ; adresse du message
    MOV EDX, msg_len      ; longueur
    INT 80h

  12.11 Exemple complet de conversion
  ---------------------------------------------------------------------------

  Voici un programme 8086 complet et sa version 80386 equivalente.

  Programme original 8086 (DOS) :

    ; Programme : affiche "Hello World" puis quitte
    .MODEL SMALL
    .STACK 100h
    .DATA
    msg DB 'Hello World!$'
    .CODE
    main:
      MOV AX, @DATA
      MOV DS, AX
      MOV AH, 09h
      LEA DX, msg
      INT 21h
      MOV AX, 4C00h
      INT 21h
    END main

  Version convertie 80386 (Win32 console) :

    ; Programme : affiche "Hello World" puis quitte
    .386
    .MODEL FLAT, STDCALL

    EXTRN GetStdHandle@4  : PROC
    EXTRN WriteConsoleA@20 : PROC
    EXTRN ExitProcess@4   : PROC

    .DATA
    msg     DB 'Hello World!', 0
    msg_len EQU $ - msg - 1
    written DD 0

    .CODE
    main PROC
      ; Obtenir le handle stdout
      PUSH -11                       ; STD_OUTPUT_HANDLE
      CALL GetStdHandle@4
      ; EAX = handle

      ; Ecrire le message
      PUSH 0                         ; lpReserved
      PUSH OFFSET written            ; lpNumberOfCharsWritten
      PUSH msg_len                   ; nNumberOfCharsToWrite
      PUSH OFFSET msg                ; lpBuffer
      PUSH EAX                       ; hConsoleOutput
      CALL WriteConsoleA@20

      ; Quitter
      PUSH 0                         ; code de retour
      CALL ExitProcess@4
    main ENDP
    END main

  12.12 Resume : aide-memoire de conversion
  ---------------------------------------------------------------------------

  Etape   Action
  ------  ----------------------------------------------------------------
  1       Ajouter .386 (ou .386P) en premiere ligne
  2       Remplacer .MODEL SMALL par .MODEL FLAT, STDCALL
  3       Supprimer .STACK (geree par le SE)
  4       Supprimer MOV AX,@DATA / MOV DS,AX
  5       Remplacer AX->EAX, BX->EBX, CX->ECX, DX->EDX
  6       Remplacer SI->ESI, DI->EDI, BP->EBP, SP->ESP
  7       Remplacer DW par DD pour les pointeurs et offsets
  8       Remplacer les INT 21h par les API equivalentes
  9       Remplacer [BX+SI] par [EBX+ESI] (et profiter du SIB)
  10      Ajuster [BP+4] -> [EBP+8] dans les cadres de pile
  11      Remplacer PUSHF/POPF par PUSHFD/POPFD
  12      Remplacer PUSHA/POPA par PUSHAD/POPAD
  13      Remplacer CWD par CDQ, CBW par CWDE
  14      Remplacer JCXZ par JECXZ
  15      Supprimer LDS/LES (inutiles en FLAT)
  16      Utiliser MOVZX/MOVSX pour les extensions de taille
  17      Profiter de IMUL a 3 operandes
  18      Profiter du facteur d'echelle [reg + reg*n]
  19      Ajuster ADD SP,n -> ADD ESP,2n (parametres 32 bits)
  20      Declarer EXTRN pour les API Win32 utilisees

  12.13 Pieges et erreurs frequentes
  ---------------------------------------------------------------------------

  (a) Oublier le prefixe de taille

    En mode 32 bits, utiliser un registre 16 bits genere un prefixe
    de remplacement de taille operande (66h), ce qui ralentit le code.
    Privilegier les registres 32 bits :

      ; Lent (prefixe 66h)            ; Rapide (pas de prefixe)
      MOV AX, 0                       MOV EAX, 0
      ADD CX, 1                       ADD ECX, 1

  (b) Melanger tailles de pile

    En mode 32 bits, PUSH empile toujours 4 octets. Ne pas faire :

      PUSH AX            ; empile 2 octets -> desaligne la pile!
      PUSH EAX           ; correct : empile 4 octets

  (c) Oublier d'ajuster les offsets dans le cadre de pile

    ; 8086 : parametres a [BP+4], [BP+6], [BP+8]...
    ; 80386 : parametres a [EBP+8], [EBP+12], [EBP+16]...
    ; (decalage de 4 car l'adresse de retour fait 4 octets)

  (d) Utiliser des adresses segmentees en mode FLAT

    Les instructions FAR (CALL FAR, JMP FAR, RET FAR) ne sont plus
    necessaires en modele FLAT. Utiliser les versions NEAR :

      ; 8086                          ; 80386
      CALL FAR proc                   CALL proc
      JMP FAR label                   JMP label
      RETF                            RET

  (e) Interruptions DOS en mode protege

    INT 21h n'est pas disponible en mode protege Win32. Tout appel
    INT 21h provoquera une exception. Utiliser les API Win32.

  (f) Alignement des donnees

    En 80386, les acces memoire alignes sur 4 octets sont plus rapides.
    Aligner les variables DD avec ALIGN 4 :

      ALIGN 4
      compteur DD 0
      ALIGN 4
      resultat DD 0

==============================================================================
13. CONVERSION DE CODE ARM EN 80386
==============================================================================

  Cette section explique comment convertir du code assembleur ARM (ARMv4/v7,
  architecture RISC 32 bits) en code assembleur Intel 80386 (architecture
  CISC 32 bits). La conversion ARM -> x86 est plus complexe que la
  conversion 8086 -> 80386 car les deux architectures ont des philosophies
  fondamentalement differentes.

------------------------------------------------------------------------------
13.1 Principes generaux (RISC vs CISC)
------------------------------------------------------------------------------

  ARM est une architecture RISC (Reduced Instruction Set Computer) :
  - Instructions de taille fixe (32 bits en mode ARM, 16 bits en Thumb)
  - Architecture Load/Store : seules LDR/STR accedent a la memoire
  - 16 registres generaux visibles (R0-R15)
  - Execution conditionnelle sur toute instruction
  - Barrel shifter integre dans l'ALU
  - Pas d'acces memoire direct dans les operations arithmetiques

  Le 80386 est une architecture CISC (Complex Instruction Set Computer) :
  - Instructions de taille variable (1 a 15 octets)
  - Operations directement sur la memoire (MOV [mem], reg / ADD [mem], imm)
  - 8 registres generaux (EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP)
  - Pas d'execution conditionnelle (sauf CMOVcc sur Pentium Pro+)
  - Modes d'adressage complexes (base+index*scale+disp)
  - Instructions specialisees (LOOP, REP MOVS, ENTER/LEAVE...)

  Regles de base pour la conversion :
  (a) Mapper les registres ARM vers les registres x86 + memoire
  (b) Remplacer l'architecture Load/Store par des acces memoire directs
  (c) Convertir l'execution conditionnelle en branchements
  (d) Decomposer les operations avec barrel shifter
  (e) Adapter les conventions d'appel (AAPCS -> CDECL ou STDCALL)

------------------------------------------------------------------------------
13.2 Correspondance des registres
------------------------------------------------------------------------------

  ARM possede 16 registres de 32 bits. Le 80386 n'en a que 8.
  Il faut donc utiliser des variables memoire pour les registres excedentaires.

  Correspondance suggeree :

    ARM         80386         Commentaire
    -------     --------      ------------------------------------
    R0          EAX           Valeur de retour, accumulateur
    R1          ECX           Second argument, compteur
    R2          EDX           Troisieme argument
    R3          EBX           Quatrieme argument
    R4          ESI           Variable preservee (callee-saved)
    R5          EDI           Variable preservee (callee-saved)
    R6          [_arm_r6]     Sauvegarde en memoire
    R7          [_arm_r7]     Sauvegarde en memoire (frame pointer Thumb)
    R8          [_arm_r8]     Sauvegarde en memoire
    R9          [_arm_r9]     Sauvegarde en memoire
    R10         [_arm_r10]    Sauvegarde en memoire
    R11 (FP)    EBP           Frame pointer
    R12 (IP)    [_arm_r12]    Scratch register, en memoire
    R13 (SP)    ESP           Stack pointer
    R14 (LR)    [_arm_lr]     Link register -> utiliser CALL/RET
    R15 (PC)    EIP           Program counter (implicite)

  Declarations pour les registres en memoire :

    SECTION .data
    _arm_r6   DD 0
    _arm_r7   DD 0
    _arm_r8   DD 0
    _arm_r9   DD 0
    _arm_r10  DD 0
    _arm_r12  DD 0
    _arm_lr   DD 0

  Note : Le registre CPSR (Current Program Status Register) ARM correspond
  au registre EFLAGS du 80386, mais les drapeaux ne sont pas identiques :

    ARM CPSR      80386 EFLAGS     Signification
    ----------    ---------------  --------------------------
    N (bit 31)    SF (bit 7)       Signe / Negatif
    Z (bit 30)    ZF (bit 6)       Zero
    C (bit 29)    CF (bit 0)       Carry (retenue)
    V (bit 28)    OF (bit 11)      Overflow (debordement)

  Attention : le sens du carry pour la soustraction est inverse !
  ARM : C=1 si pas d'emprunt. x86 : CF=1 si emprunt.

------------------------------------------------------------------------------
13.3 Conversion des instructions de transfert de donnees
------------------------------------------------------------------------------

  (a) MOV registre a registre

    ARM :   MOV R0, R1
    80386 : MOV EAX, ECX

    ARM :   MOV R4, R5
    80386 : MOV ESI, EDI

    ARM :   MOV R6, R7          ; registres en memoire
    80386 : MOV EAX, [_arm_r7]
            MOV [_arm_r6], EAX

  (b) MOV avec immediat

    ARM :   MOV R0, #42
    80386 : MOV EAX, 42

    ARM :   MOV R0, #0
    80386 : XOR EAX, EAX        ; plus efficace que MOV EAX, 0

    Note : ARM limite les immediats a 8 bits + rotation. Pour les grandes
    valeurs, ARM utilise MOVW/MOVT (ARMv7) ou LDR Rd, =valeur (pseudo).
    Le 80386 accepte des immediats 32 bits directement :

    ARM :   MOVW R0, #0x1234
            MOVT R0, #0x5678    ; R0 = 0x56781234
    80386 : MOV EAX, 056781234h

  (c) MVN (Move Not)

    ARM :   MVN R0, R1          ; R0 = NOT R1
    80386 : MOV EAX, ECX
            NOT EAX

    ARM :   MVN R0, #0          ; R0 = 0xFFFFFFFF
    80386 : MOV EAX, 0FFFFFFFFh
    ou    : OR EAX, -1

  (d) Transferts conditionnels (voir section 13.11)

------------------------------------------------------------------------------
13.4 Conversion des instructions arithmetiques et logiques
------------------------------------------------------------------------------

  (a) Addition

    ARM :   ADD R0, R1, R2       ; R0 = R1 + R2
    80386 : MOV EAX, ECX
            ADD EAX, EDX

    ARM :   ADD R0, R0, #10      ; R0 = R0 + 10
    80386 : ADD EAX, 10

    ARM :   ADD R0, R1, R2, LSL #3  ; R0 = R1 + (R2 << 3)
    80386 : MOV EAX, EDX
            SHL EAX, 3
            ADD EAX, ECX
    ou    : LEA EAX, [ECX + EDX*8]  ; si le facteur est 1, 2, 4 ou 8

  (b) Soustraction

    ARM :   SUB R0, R1, R2       ; R0 = R1 - R2
    80386 : MOV EAX, ECX
            SUB EAX, EDX

    ARM :   RSB R0, R1, #0       ; R0 = 0 - R1 = -R1
    80386 : MOV EAX, ECX
            NEG EAX

    ARM :   RSB R0, R1, R2       ; R0 = R2 - R1
    80386 : MOV EAX, EDX
            SUB EAX, ECX

  (c) Addition/Soustraction avec carry

    ARM :   ADDS R0, R1, R2     ; R0 = R1 + R2, maj flags
            ADC R3, R4, R5      ; R3 = R4 + R5 + C
    80386 : MOV EAX, ECX
            ADD EAX, EDX        ; maj flags
            MOV EBX, ESI
            ADC EBX, EDI        ; EBX = ESI + EDI + CF

    Attention : le carry ARM est inverse pour SBC/SBC.
    ARM :   SBC R0, R1, R2      ; R0 = R1 - R2 - !C
    80386 : ; Inverser le carry d'abord
            CMC                  ; complement carry flag
            MOV EAX, ECX
            SBB EAX, EDX

  (d) Multiplication

    ARM :   MUL R0, R1, R2       ; R0 = R1 * R2
    80386 : MOV EAX, ECX
            IMUL EAX, EDX

    ARM :   MLA R0, R1, R2, R3   ; R0 = R1 * R2 + R3
    80386 : MOV EAX, ECX
            IMUL EAX, EDX
            ADD EAX, EBX

    ARM :   UMULL R0, R1, R2, R3 ; R1:R0 = R2 * R3 (64 bits non signe)
    80386 : MOV EAX, EDX
            MUL EBX              ; EDX:EAX = EAX * EBX

    ARM :   SMULL R0, R1, R2, R3 ; R1:R0 = R2 * R3 (64 bits signe)
    80386 : MOV EAX, EDX
            IMUL EBX             ; EDX:EAX = EAX * EBX (signe)

  (e) Division

    ARM n'a pas d'instruction de division (avant ARMv7-R/ARMv7-A avec UDIV/SDIV).
    Elle est souvent realisee par appel de fonction.

    ARM :   ; Appel runtime __aeabi_uidiv
            BL __aeabi_uidiv      ; R0 = R0 / R1
    80386 : XOR EDX, EDX
            DIV ECX               ; EAX = EAX / ECX, EDX = reste

    ARM :   SDIV R0, R1, R2       ; ARMv7, R0 = R1 / R2 (signe)
    80386 : MOV EAX, ECX
            CDQ                   ; extension signe EAX -> EDX:EAX
            IDIV EDX              ; attention: diviseur dans autre registre

  (f) Operations logiques

    ARM :   AND R0, R1, R2
    80386 : MOV EAX, ECX
            AND EAX, EDX

    ARM :   ORR R0, R1, R2
    80386 : MOV EAX, ECX
            OR EAX, EDX

    ARM :   EOR R0, R1, R2       ; XOR
    80386 : MOV EAX, ECX
            XOR EAX, EDX

    ARM :   BIC R0, R1, R2       ; R0 = R1 AND NOT R2
    80386 : MOV EAX, EDX
            NOT EAX
            AND EAX, ECX

    ARM :   ORN R0, R1, R2       ; R0 = R1 OR NOT R2 (Thumb-2)
    80386 : MOV EAX, EDX
            NOT EAX
            OR EAX, ECX

  (g) Comparaison et test

    ARM :   CMP R0, R1           ; flags = R0 - R1
    80386 : CMP EAX, ECX

    ARM :   CMN R0, R1           ; flags = R0 + R1
    80386 : MOV EDX, EAX         ; sauvegarder EAX
            ADD EDX, ECX         ; positionne les flags sans modifier EAX
    ou    : PUSH EAX
            ADD EAX, ECX
            POP EAX

    ARM :   TST R0, R1           ; flags = R0 AND R1
    80386 : TEST EAX, ECX

    ARM :   TEQ R0, R1           ; flags = R0 XOR R1
    80386 : MOV EDX, EAX
            XOR EDX, ECX         ; positionne les flags

  (h) Mise a jour des drapeaux (suffixe S)

    En ARM, les drapeaux ne sont mis a jour que si le suffixe S est present.
    En x86, la plupart des instructions mettent a jour les flags automatiquement.

    ARM :   ADD R0, R1, R2       ; PAS de maj flags
    80386 : ; Sauvegarder EFLAGS si necessaire
            PUSHFD
            MOV EAX, ECX
            ADD EAX, EDX
            POPFD                ; restaurer les flags

    ARM :   ADDS R0, R1, R2      ; maj flags
    80386 : MOV EAX, ECX
            ADD EAX, EDX         ; flags mis a jour automatiquement

    En pratique, on optimise en ne protegeant les flags que lorsque c'est
    strictement necessaire (c'est-a-dire quand une instruction suivante
    depend des flags d'une instruction precedente).

------------------------------------------------------------------------------
13.5 Conversion des decalages et rotations
------------------------------------------------------------------------------

  (a) Decalage logique a gauche

    ARM :   LSL R0, R1, #4       ; R0 = R1 << 4
    80386 : MOV EAX, ECX
            SHL EAX, 4

    ARM :   LSL R0, R1, R2       ; R0 = R1 << R2
    80386 : MOV EAX, ECX
            MOV CL, DL           ; x86 exige CL pour les decalages variables
            SHL EAX, CL

  (b) Decalage logique a droite

    ARM :   LSR R0, R1, #4
    80386 : MOV EAX, ECX
            SHR EAX, 4

  (c) Decalage arithmetique a droite

    ARM :   ASR R0, R1, #4
    80386 : MOV EAX, ECX
            SAR EAX, 4

  (d) Rotation a droite

    ARM :   ROR R0, R1, #4
    80386 : MOV EAX, ECX
            ROR EAX, 4

  (e) Rotation a droite etendue (avec carry)

    ARM :   RRX R0, R1           ; R0 = (C << 31) | (R1 >> 1)
    80386 : MOV EAX, ECX
            RCR EAX, 1           ; rotation via carry

  Note : En ARM, le decalage de 0 positions a un comportement special
  (LSL #0 = pas de decalage, LSR #0 = LSR #32, ASR #0 = ASR #32,
  ROR #0 = RRX). En x86, un decalage de 0 ne fait rien.

------------------------------------------------------------------------------
13.6 Conversion des branchements et conditions
------------------------------------------------------------------------------

  (a) Branchement inconditionnel

    ARM :   B label
    80386 : JMP label

  (b) Branchement avec lien (appel de fonction)

    ARM :   BL fonction           ; LR = adresse retour, PC = fonction
    80386 : CALL fonction         ; PUSH EIP+n, JMP fonction

  (c) Retour de fonction

    ARM :   BX LR                 ; PC = LR
    80386 : RET                   ; POP EIP

    ARM :   MOV PC, LR            ; ancien style retour
    80386 : RET

  (d) Branchements conditionnels

    Les codes condition ARM et x86 sont similaires mais pas identiques :

    ARM         80386       Condition
    --------    --------    ---------------------------------
    BEQ label   JE label    Egal (Z=1)
    BNE label   JNE label   Different (Z=0)
    BCS/BHS     JC label    Carry set / unsigned >= (C=1)
                            ATTENTION: pour CMP, utiliser JAE
    BCC/BLO     JNC label   Carry clear / unsigned < (C=0)
                            ATTENTION: pour CMP, utiliser JB
    BMI label   JS label    Negatif (N=1 / SF=1)
    BPL label   JNS label   Positif ou zero (N=0 / SF=0)
    BVS label   JO label    Overflow (V=1 / OF=1)
    BVC label   JNO label   No overflow (V=0 / OF=0)
    BHI label   JA label    Unsigned superieur (C=1 et Z=0)
    BLS label   JBE label   Unsigned inferieur ou egal
    BGE label   JGE label   Signe superieur ou egal
    BLT label   JL label    Signe inferieur
    BGT label   JG label    Signe superieur
    BLE label   JLE label   Signe inferieur ou egal
    BAL label   JMP label   Toujours (inconditionnel)

    ATTENTION au carry inverse pour les comparaisons non signees :
    Apres CMP en ARM, C=1 signifie "pas d'emprunt" (>=).
    Apres CMP en x86, CF=0 signifie "pas d'emprunt" (>=).
    Donc BCS (ARM) correspond a JAE/JNC (x86), pas a JC.
    Et BCC (ARM) correspond a JB/JC (x86), pas a JNC.

    Exemple :
    ARM :   CMP R0, R1
            BCS superieur_ou_egal   ; branche si R0 >= R1 (non signe)
    80386 : CMP EAX, ECX
            JAE superieur_ou_egal   ; JAE = JNC = CF=0

  (e) Branchement indirect

    ARM :   BX R0                 ; PC = R0
    80386 : JMP EAX

    ARM :   BLX R0                ; LR = PC+4, PC = R0
    80386 : CALL EAX

  (f) Table de branchement (switch)

    ARM :   ; Table de branchement avec TBB/TBH (Thumb-2)
            TBB [PC, R0]
    80386 : JMP DWORD PTR [table + EAX*4]

------------------------------------------------------------------------------
13.7 Conversion des acces memoire (Load/Store vs CISC)
------------------------------------------------------------------------------

  L'une des differences fondamentales est que ARM utilise une architecture
  Load/Store : seules les instructions LDR/STR accedent a la memoire.
  En x86, presque toute instruction peut acceder a la memoire directement.

  (a) Chargement (Load)

    ARM :   LDR R0, [R1]          ; R0 = mem[R1]
    80386 : MOV EAX, [ECX]

    ARM :   LDR R0, [R1, #8]      ; R0 = mem[R1 + 8]
    80386 : MOV EAX, [ECX + 8]

    ARM :   LDR R0, [R1, R2]      ; R0 = mem[R1 + R2]
    80386 : MOV EAX, [ECX + EDX]

    ARM :   LDR R0, [R1, R2, LSL #2]  ; R0 = mem[R1 + R2*4]
    80386 : MOV EAX, [ECX + EDX*4]

  (b) Stockage (Store)

    ARM :   STR R0, [R1]          ; mem[R1] = R0
    80386 : MOV [ECX], EAX

    ARM :   STR R0, [R1, #12]     ; mem[R1+12] = R0
    80386 : MOV [ECX + 12], EAX

  (c) Chargement/Stockage avec pre-indexation et write-back

    ARM :   LDR R0, [R1, #4]!     ; R1 = R1 + 4, R0 = mem[R1]
    80386 : ADD ECX, 4
            MOV EAX, [ECX]

  (d) Post-indexation

    ARM :   LDR R0, [R1], #4      ; R0 = mem[R1], R1 = R1 + 4
    80386 : MOV EAX, [ECX]
            ADD ECX, 4

  (e) Acces sur un octet (byte)

    ARM :   LDRB R0, [R1]         ; R0 = zero_extend(mem8[R1])
    80386 : MOVZX EAX, BYTE PTR [ECX]

    ARM :   LDRSB R0, [R1]        ; R0 = sign_extend(mem8[R1])
    80386 : MOVSX EAX, BYTE PTR [ECX]

    ARM :   STRB R0, [R1]         ; mem8[R1] = R0[7:0]
    80386 : MOV BYTE PTR [ECX], AL

  (f) Acces sur un demi-mot (halfword, 16 bits)

    ARM :   LDRH R0, [R1]         ; R0 = zero_extend(mem16[R1])
    80386 : MOVZX EAX, WORD PTR [ECX]

    ARM :   LDRSH R0, [R1]        ; R0 = sign_extend(mem16[R1])
    80386 : MOVSX EAX, WORD PTR [ECX]

    ARM :   STRH R0, [R1]         ; mem16[R1] = R0[15:0]
    80386 : MOV WORD PTR [ECX], AX

  (g) Chargement d'adresse (ADR / LDR =)

    ARM :   ADR R0, label          ; R0 = adresse de label (PC-relative)
    80386 : LEA EAX, [label]
    ou    : MOV EAX, OFFSET label

    ARM :   LDR R0, =0x12345678   ; pseudo-instruction, charge constante
    80386 : MOV EAX, 012345678h

  (h) Acces memoire multiples (LDM/STM) - voir section 13.9

------------------------------------------------------------------------------
13.8 Conversion des modes d'adressage
------------------------------------------------------------------------------

  ARM supporte plusieurs modes d'adressage pour LDR/STR.
  Le 80386 supporte le mode [base + index*scale + deplacement].

  (a) Offset immediat

    ARM :   LDR R0, [R1, #offset]
    80386 : MOV EAX, [ECX + offset]      ; si offset dans -2^31..+2^31

  (b) Offset registre

    ARM :   LDR R0, [R1, R2]
    80386 : MOV EAX, [ECX + EDX]

  (c) Offset registre avec decalage

    ARM :   LDR R0, [R1, R2, LSL #2]     ; echelle 1,2,4,8 -> SIB
    80386 : MOV EAX, [ECX + EDX*4]

    ARM :   LDR R0, [R1, R2, LSL #3]     ; echelle 8
    80386 : MOV EAX, [ECX + EDX*8]

    Pour les decalages non supportes par SIB (ex: LSL #5) :
    ARM :   LDR R0, [R1, R2, LSL #5]
    80386 : MOV EAX, EDX
            SHL EAX, 5
            MOV EAX, [ECX + EAX]

  (d) Offset negatif

    ARM :   LDR R0, [R1, #-4]
    80386 : MOV EAX, [ECX - 4]

    ARM :   LDR R0, [R1, -R2]
    80386 : MOV EAX, ECX
            SUB EAX, EDX
            MOV EAX, [EAX]

  (e) Adressage PC-relatif

    ARM utilise frequemment l'adressage relatif au PC.
    En x86-32, l'adressage est absolu (sauf en PIC/PIE).

    ARM :   LDR R0, [PC, #offset]
    80386 : MOV EAX, [label]              ; adresse absolue

------------------------------------------------------------------------------
13.9 Conversion des instructions de pile (PUSH/POP, STMFD/LDMFD)
------------------------------------------------------------------------------

  (a) PUSH et POP simples

    ARM :   PUSH {R0}             ; equivalent STR R0, [SP, #-4]!
    80386 : PUSH EAX

    ARM :   POP {R0}              ; equivalent LDR R0, [SP], #4
    80386 : POP EAX

  (b) PUSH/POP multiples

    ARM :   PUSH {R4-R7, LR}     ; sauvegarde plusieurs registres
    80386 : PUSH ESI              ; R4
            PUSH EDI              ; R5
            PUSH [_arm_r6]        ; R6
            PUSH [_arm_r7]        ; R7
            ; LR est gere par CALL en x86

    ARM :   POP {R4-R7, PC}      ; restaure et retourne
    80386 : POP [_arm_r7]         ; R7
            POP [_arm_r6]         ; R6
            POP EDI               ; R5
            POP ESI               ; R4
            RET                   ; PC = adresse retour sur la pile

    Note : ARM empile du registre le plus eleve au plus bas.
    L'ordre de POP est inverse. En x86, chaque PUSH/POP est individuel.

  (c) STMFD/LDMFD (Full Descending = pile standard)

    ARM :   STMFD SP!, {R0-R3}   ; empiler R0-R3
    80386 : PUSH EBX              ; R3 d'abord (adresse la plus haute)
            PUSH EDX              ; R2
            PUSH ECX              ; R1
            PUSH EAX              ; R0

    ARM :   LDMFD SP!, {R0-R3}   ; depiler R0-R3
    80386 : POP EAX               ; R0
            POP ECX               ; R1
            POP EDX               ; R2
            POP EBX               ; R3

  (d) Transfert de blocs (STMIA/LDMIA pour copie memoire)

    ARM :   STMIA R0!, {R4-R7}   ; mem[R0..R0+15] = R4-R7, R0 += 16
    80386 : MOV [EAX], ESI
            MOV [EAX+4], EDI
            MOV EDX, [_arm_r6]
            MOV [EAX+8], EDX
            MOV EDX, [_arm_r7]
            MOV [EAX+12], EDX
            ADD EAX, 16

------------------------------------------------------------------------------
13.10 Conversion des appels de fonctions et conventions d'appel
------------------------------------------------------------------------------

  (a) Convention ARM (AAPCS - ARM Architecture Procedure Call Standard) :
      - Arguments dans R0-R3, surplus sur la pile
      - Valeur de retour dans R0 (et R1 pour 64 bits)
      - R0-R3, R12 : caller-saved (volatile)
      - R4-R11 : callee-saved (non volatile)
      - LR (R14) : adresse de retour
      - SP (R13) doit etre aligne sur 8 octets

  (b) Convention x86 CDECL :
      - Tous les arguments sur la pile (droite a gauche)
      - Valeur de retour dans EAX (EDX:EAX pour 64 bits)
      - EAX, ECX, EDX : caller-saved
      - EBX, ESI, EDI, EBP : callee-saved
      - L'appelant nettoie la pile (ADD ESP, n)

  (c) Exemple de conversion d'appel

    ARM :   ; appel : result = mafonction(a, b, c, d, e)
            LDR R0, [SP, #arg_a]   ; premier argument dans R0
            LDR R1, [SP, #arg_b]   ; deuxieme dans R1
            LDR R2, [SP, #arg_c]   ; troisieme dans R2
            LDR R3, [SP, #arg_d]   ; quatrieme dans R3
            LDR R4, [SP, #arg_e]
            PUSH {R4}              ; cinquieme sur la pile
            BL mafonction
            ADD SP, SP, #4         ; nettoyer argument 5
            ; resultat dans R0

    80386 : ; CDECL : tous les arguments sur la pile, droite a gauche
            PUSH DWORD PTR [arg_e]  ; cinquieme argument
            PUSH DWORD PTR [arg_d]  ; quatrieme
            PUSH DWORD PTR [arg_c]  ; troisieme
            PUSH DWORD PTR [arg_b]  ; deuxieme
            PUSH DWORD PTR [arg_a]  ; premier
            CALL mafonction
            ADD ESP, 20             ; nettoyer 5 * 4 octets
            ; resultat dans EAX

  (d) Prologue et epilogue de fonction

    ARM :   mafonction:
              PUSH {R4-R7, LR}      ; sauvegarder registres + LR
              SUB SP, SP, #16       ; espace pour variables locales
              ; ... corps ...
              ADD SP, SP, #16
              POP {R4-R7, PC}       ; restaurer et retourner

    80386 : mafonction:
              PUSH EBP
              MOV EBP, ESP
              PUSH ESI               ; R4
              PUSH EDI               ; R5
              SUB ESP, 16            ; variables locales
              ; ... corps ...
              ADD ESP, 16
              POP EDI
              POP ESI
              MOV ESP, EBP
              POP EBP
              RET

  (e) Acces aux parametres et variables locales

    ARM :   ; parametre 1 (dans R0 a l'entree, souvent sauvegarde sur pile)
            STR R0, [SP, #0]       ; var locale 1
            LDR R1, [SP, #0]       ; relecture

    80386 : ; parametre 1 = [EBP + 8], param 2 = [EBP + 12]...
            MOV EAX, [EBP + 8]    ; premier parametre
            MOV [EBP - 4], EAX    ; variable locale 1
            MOV ECX, [EBP - 4]    ; relecture

------------------------------------------------------------------------------
13.11 Conversion des instructions conditionnelles ARM
------------------------------------------------------------------------------

  Une particularite majeure d'ARM est que presque toute instruction peut
  etre executee conditionnellement avec un suffixe de condition.
  Le 80386 n'a pas cette capacite (sauf SETcc et CMOVcc sur P6+).

  (a) Instruction conditionnelle simple

    ARM :   MOVEQ R0, #1          ; si Z=1, R0 = 1
    80386 : JNE skip_moveq
            MOV EAX, 1
    skip_moveq:

    ARM :   ADDNE R0, R0, #1      ; si Z=0, R0++
    80386 : JE skip_addne
            ADD EAX, 1
    skip_addne:

  (b) Paire conditionnelle (if-then-else)

    ARM :   CMP R0, R1
            MOVGT R2, R0          ; if (R0 > R1) R2 = R0
            MOVLE R2, R1          ; else          R2 = R1
    80386 : CMP EAX, ECX
            JLE else_branch
            MOV EDX, EAX
            JMP endif_branch
    else_branch:
            MOV EDX, ECX
    endif_branch:

  (c) Bloc conditionnel IT (Thumb-2)

    ARM :   CMP R0, #0
            ITE NE                 ; If-Then-Else
            MOVNE R1, #1           ; Then
            MOVEQ R1, #0           ; Else
    80386 : CMP EAX, 0
            JE is_zero
            MOV ECX, 1
            JMP done_ite
    is_zero:
            MOV ECX, 0
    done_ite:

  (d) Operations conditionnelles sur les drapeaux

    ARM :   CMP R0, R1
            ADDGT R2, R2, #1      ; si R0 > R1 (signe), R2++
            SUBLT R2, R2, #1      ; si R0 < R1 (signe), R2--
    80386 : CMP EAX, ECX
            JLE not_gt
            ADD EDX, 1
    not_gt:
            CMP EAX, ECX          ; re-comparer (flags perdus par ADD)
            JGE not_lt
            SUB EDX, 1
    not_lt:

  (e) SETcc comme alternative (pour les booleens)

    ARM :   CMP R0, R1
            MOVHI R2, #1          ; R2 = (R0 > R1) ? 1 : 0
            MOVLS R2, #0
    80386 : CMP EAX, ECX
            SETA DL               ; DL = 1 si au-dessus (non signe)
            MOVZX EDX, DL

    Note : SETcc n'est disponible que sur 80386+.
    CMOVcc est disponible a partir du Pentium Pro (pas sur 80386).

------------------------------------------------------------------------------
13.12 Conversion du barrel shifter ARM
------------------------------------------------------------------------------

  Le barrel shifter ARM permet d'appliquer un decalage ou une rotation
  au second operande de nombreuses instructions ALU, sans cout supplementaire.
  En x86, il faut decomposer l'operation en instructions separees.

  (a) ADD avec decalage

    ARM :   ADD R0, R1, R2, LSL #3    ; R0 = R1 + (R2 << 3)
    80386 : MOV EAX, EDX
            SHL EAX, 3
            ADD EAX, ECX

    Optimisation avec LEA quand le facteur est 1, 2, 4 ou 8 :
    ARM :   ADD R0, R1, R2, LSL #2    ; R0 = R1 + R2*4
    80386 : LEA EAX, [ECX + EDX*4]

  (b) SUB avec decalage

    ARM :   SUB R0, R1, R2, LSR #4    ; R0 = R1 - (R2 >> 4)
    80386 : MOV EAX, EDX
            SHR EAX, 4
            MOV EDX, ECX              ; sauver R1
            SUB EDX, EAX
            MOV EAX, EDX

  (c) AND avec decalage

    ARM :   AND R0, R1, R2, ROR #8    ; R0 = R1 AND (R2 ROR 8)
    80386 : MOV EAX, EDX
            ROR EAX, 8
            AND EAX, ECX

  (d) CMP avec decalage

    ARM :   CMP R0, R1, LSL #2        ; flags = R0 - (R1 << 2)
    80386 : MOV EDX, ECX
            SHL EDX, 2
            CMP EAX, EDX

  (e) MOV avec decalage (= decalage simple)

    ARM :   MOV R0, R1, LSL #4        ; R0 = R1 << 4
    80386 : MOV EAX, ECX
            SHL EAX, 4

  (f) Decalage par registre

    ARM :   ADD R0, R1, R2, LSL R3    ; R0 = R1 + (R2 << R3)
    80386 : MOV EAX, EDX
            PUSH ECX                  ; sauver R1 (ECX)
            MOV CL, BL               ; decalage dans CL (R3 = EBX)
            SHL EAX, CL
            POP ECX
            ADD EAX, ECX

  Conseil : le barrel shifter ARM est souvent utilise pour multiplier par
  des constantes. Reconnaitre ces patterns :
    LSL #1 = *2,  LSL #2 = *4,  LSL #3 = *8,  LSL #4 = *16
    ADD Rd, Rn, Rn, LSL #1 = Rn * 3
    ADD Rd, Rn, Rn, LSL #2 = Rn * 5
    RSB Rd, Rn, Rn, LSL #3 = Rn * 7

  En x86, utiliser LEA ou IMUL :
    ARM : ADD R0, R1, R1, LSL #2    ; R0 = R1 * 5
    x86 : LEA EAX, [ECX + ECX*4]    ; EAX = ECX * 5

    ARM : RSB R0, R1, R1, LSL #3    ; R0 = R1 * 7
    x86 : IMUL EAX, ECX, 7          ; EAX = ECX * 7

------------------------------------------------------------------------------
13.13 Exemple complet de conversion
------------------------------------------------------------------------------

  Programme ARM qui calcule la somme des entiers de 1 a N :

  --- Code ARM (AAPCS, appel systeme Linux) ---

    .text
    .global _start

    _start:
        MOV R0, #10           ; N = 10
        BL sum_1_to_n         ; appel de la fonction
        ; R0 contient le resultat
        MOV R7, #1            ; syscall exit
        SWI #0                ; appeler le noyau

    sum_1_to_n:
        PUSH {R4, LR}         ; sauvegarder R4 et LR
        MOV R4, R0             ; R4 = N
        MOV R0, #0             ; somme = 0
    .loop:
        CMP R4, #0
        BLE .done
        ADD R0, R0, R4         ; somme += compteur
        SUB R4, R4, #1         ; compteur--
        B .loop
    .done:
        POP {R4, PC}           ; restaurer R4, retourner

  --- Code 80386 equivalent (MASM, Win32 ou Linux) ---

    .386
    .MODEL FLAT, STDCALL

    .DATA
        resultat DD 0

    .CODE
    _start:
        PUSH 10                  ; N = 10 (argument sur pile)
        CALL sum_1_to_n
        ADD ESP, 4               ; nettoyer pile (CDECL)
        MOV [resultat], EAX      ; sauver resultat
        ; ... appeler ExitProcess ou sys_exit ...
        PUSH 0
        CALL ExitProcess

    sum_1_to_n PROC
        PUSH EBP
        MOV EBP, ESP
        PUSH ESI                 ; sauvegarder ESI (= R4)
        MOV ESI, [EBP + 8]      ; ESI = N (premier parametre)
        XOR EAX, EAX            ; somme = 0
    boucle:
        CMP ESI, 0
        JLE fin
        ADD EAX, ESI             ; somme += compteur
        DEC ESI                  ; compteur--
        JMP boucle
    fin:
        POP ESI                  ; restaurer ESI
        MOV ESP, EBP
        POP EBP
        RET
    sum_1_to_n ENDP

    END _start

  Points de conversion illustres dans cet exemple :
  - BL -> CALL / BX LR -> RET
  - Arguments R0 -> pile [EBP+8]
  - Registre preserve R4 -> ESI
  - PUSH {R4, LR} -> PUSH EBP / MOV EBP, ESP / PUSH ESI
  - POP {R4, PC} -> POP ESI / MOV ESP, EBP / POP EBP / RET
  - SWI #0 (syscall ARM) -> CALL ExitProcess (Win32) ou INT 80h (Linux)

13.14 Resume : aide-memoire de conversion ARM -> 80386
------------------------------------------------------------------------------

  Etape 1  : Identifier les registres utilises et etablir la correspondance
             R0-R5 -> EAX,ECX,EDX,EBX,ESI,EDI ; R6+ -> memoire
  Etape 2  : Convertir les LDR/STR en MOV avec adressage memoire
  Etape 3  : Remplacer les instructions conditionnelles par des branchements
  Etape 4  : Decomposer les operations avec barrel shifter
  Etape 5  : Convertir les acces memoire multiples (LDM/STM) en PUSH/POP
             ou series de MOV
  Etape 6  : Adapter les conventions d'appel (R0-R3 -> pile)
  Etape 7  : Convertir BL -> CALL, BX LR -> RET
  Etape 8  : Remplacer les appels systeme (SWI/SVC -> INT 80h ou API Win32)
  Etape 9  : Gerer les differences de carry (ARM C inverse pour SUB)
  Etape 10 : Gerer les flags : ARM ne met pas a jour sans suffixe S,
             x86 met a jour automatiquement
  Etape 11 : Adapter les immediats (ARM: 8 bits + rotation, x86: 32 bits)
  Etape 12 : Convertir les pseudo-instructions ARM (ADR, LDR =value)
  Etape 13 : Adapter l'alignement de la pile (ARM: 8 octets, x86: 4 ou 16)
  Etape 14 : Verifier l'endianness (ARM peut etre big ou little endian,
             x86 est toujours little endian)
  Etape 15 : Convertir les instructions SIMD/NEON en SSE si necessaire
             (hors portee de ce document 80386)

  Table de correspondance rapide des instructions :

    ARM                 80386
    ------------------  ------------------------------------------
    MOV Rd, Rn          MOV reg, reg
    MOV Rd, #imm        MOV reg, imm
    MVN Rd, Rn          MOV reg, reg / NOT reg
    ADD Rd, Rn, Op2     MOV + ADD (ou LEA)
    SUB Rd, Rn, Op2     MOV + SUB
    RSB Rd, Rn, Op2     MOV + SUB (operandes inverses)
    MUL Rd, Rn, Rm      IMUL reg, reg
    AND Rd, Rn, Op2     MOV + AND
    ORR Rd, Rn, Op2     MOV + OR
    EOR Rd, Rn, Op2     MOV + XOR
    BIC Rd, Rn, Op2     MOV + NOT + AND
    CMP Rn, Op2         CMP
    TST Rn, Op2         TEST
    LDR Rd, [Rn, off]   MOV reg, [reg + off]
    STR Rd, [Rn, off]   MOV [reg + off], reg
    LDRB Rd, [Rn]       MOVZX reg, BYTE PTR [reg]
    STRB Rd, [Rn]       MOV BYTE PTR [reg], reg8
    LDRH Rd, [Rn]       MOVZX reg, WORD PTR [reg]
    STRH Rd, [Rn]       MOV WORD PTR [reg], reg16
    PUSH {regs}         serie de PUSH
    POP {regs}          serie de POP
    B label             JMP label
    BL fonction         CALL fonction
    BX LR               RET
    Bcc label           Jcc label (adapter condition)
    SWI / SVC           INT 80h (Linux) / CALL API (Win32)
    NOP                 NOP

13.15 Pieges et differences fondamentales
------------------------------------------------------------------------------

  (a) Nombre de registres

    ARM a 16 registres generaux, x86 n'en a que 8. Il faut utiliser
    la pile et des variables memoire pour les registres ARM supplementaires.
    Cela peut degrader les performances. Sur x86-64, il y a 16 registres.

  (b) Carry inverse pour la soustraction

    C'est le piege le plus courant. Apres une soustraction :
    - ARM : C=1 signifie "pas d'emprunt" (resultat >= 0 en non signe)
    - x86 : CF=1 signifie "emprunt" (resultat < 0 en non signe)

    Pour SBC/ADC, il faut utiliser CMC (Complement Carry) avant SBB,
    ou restructurer la logique.

  (c) Mise a jour des drapeaux

    En ARM, les drapeaux sont mis a jour uniquement avec le suffixe S
    (ADDS, SUBS, etc.). En x86, la plupart des instructions arithmetiques
    et logiques mettent a jour les flags automatiquement.

    Cela signifie qu'en x86, des instructions intermediaires peuvent
    corrompre les flags. Il faut parfois sauvegarder EFLAGS avec PUSHFD
    ou restructurer le code.

  (d) Architecture Load/Store vs CISC

    En ARM, on ne peut pas faire ADD R0, R1, [memoire]. Il faut d'abord
    charger la valeur. En x86, on peut : ADD EAX, [memoire].
    Lors de la conversion ARM -> x86, on peut souvent fusionner un LDR
    suivi d'une operation en une seule instruction x86 :

    ARM :   LDR R1, [R2]
            ADD R0, R0, R1
    80386 : ADD EAX, [EDX]       ; une seule instruction

  (e) Immediats

    ARM limite les immediats a 8 bits avec rotation de 2 bits (soit 256
    valeurs par rotation). Les grandes constantes necessitent MOVW/MOVT
    ou un literal pool (LDR Rd, =constante).
    x86 accepte des immediats 32 bits directement.

  (f) Endianness

    ARM peut fonctionner en little-endian ou big-endian.
    x86 est toujours little-endian. Si le code ARM source est en mode
    big-endian, il faut convertir les acces memoire multi-octets
    avec BSWAP (disponible a partir du 80486) ou une sequence manuelle
    de decalages.

  (g) Alignement memoire

    ARM (avant ARMv7) exige un alignement strict : LDR sur une adresse
    alignee a 4 octets, LDRH sur 2 octets. Un acces non aligne provoque
    une exception ou un comportement indefini.
    x86 autorise les acces non alignes (avec une penalite de performance).
    Lors de la conversion, on peut relacher les contraintes d'alignement,
    mais il est preferable de maintenir l'alignement pour la performance.

  (h) Instructions inexistantes en x86

    Certaines instructions ARM n'ont pas d'equivalent direct en x86 :
    - CLZ (Count Leading Zeros) : utiliser BSR (Bit Scan Reverse) puis
      soustraire de 31. BSR n'est pas identique (BSR est indefini si
      l'operande est 0, CLZ retourne 32).
    - REV (Byte Reverse) : utiliser BSWAP (80486+)
    - RBIT (Reverse Bits) : pas d'equivalent, implementer en logiciel
    - UBFX/SBFX (Bit Field Extract) : utiliser SHR + AND (ou SAR + AND)
    - PKH (Pack Halfword) : decomposer en operations AND/OR/shift

  (i) Appels systeme

    ARM Linux utilise SWI 0 (ou SVC 0) avec le numero de syscall dans R7.
    x86 Linux utilise INT 80h avec le numero dans EAX.

    ARM :   MOV R7, #4           ; sys_write
            MOV R0, #1           ; fd = stdout
            LDR R1, =message     ; buffer
            MOV R2, #13          ; longueur
            SWI #0
    80386 : MOV EAX, 4           ; sys_write
            MOV EBX, 1           ; fd = stdout
            MOV ECX, OFFSET message
            MOV EDX, 13          ; longueur
            INT 80h

    En Win32, utiliser les API (WriteFile, etc.) via CALL.

  (j) Coprocesseur flottant

    ARM utilise VFP/NEON (registres S0-S31, D0-D31).
    Le 80386 utilise le coprocesseur x87 (registres ST(0)-ST(7)) avec
    une pile flottante, ou SSE sur les processeurs plus recents.
    La conversion des operations flottantes est hors portee de ce document.
