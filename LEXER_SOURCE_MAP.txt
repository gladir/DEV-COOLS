================================================================================
KEY IMPLEMENTATION NOTES FOR CSCW32
================================================================================

1. lex_peek_tok is used for single-token lookahead
   - Only one lookahead token supported
   - lex_has_peek flag indicates if valid

2. All global state - no object-oriented structure
   - Suitable for Win32 port with global variables
   - Consider wrapping in a Win32 DLL structure

3. Character position tracking:
   - lex_pos: 1-based position in lex_line
   - lex_lineno: 1-based line number
   - lex_colno: 1-based column in current line

4. Token information fields:
   - kind: TTokenKind (enum)
   - val: string representation (up to 128 chars)
   - int_val: numeric value (for TK_INT tokens)
   - line, col: source position for error reporting

5. Multi-character operator handling:
   - Explicitly lists all two-char operators
   - No smart lookahead table
   - Check with LexPeekChar(1) in LexReadOp

6. Comment handling:
   - Line comments (//) stop at EOL
   - Block comments (/* */) can span lines
   - Nested block comments NOT supported (in_block flag is simple boolean)

7. String handling:
   - No raw/verbatim strings (@"...")
   - Escape sequences decoded during scanning
   - Truncated to MAX_TOKSTR if longer

8. Number parsing:
   - Decimal and hex (0x/0X prefix)
   - Integer suffixes (u, l, ul, lu) parsed but discarded
   - No floating-point support
   - Returned as both text (val) and numeric (int_val)
